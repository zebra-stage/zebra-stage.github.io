Motion Screen Blanking

Motion Screen Blank 1.5

Overview

Motion Screen Blank is an Android app designed for Zebra devices used in motor vehicles. Using GPS and other technologies, the app detects when the device is in motion and automatically removes meaningful content from the screen, helping to prevent driver distraction and reducing the potential for injury. MSB is a "headless" app with no user interface available to the user or vehicle operator.

Main Features

  • Launches automatically on re/boot
  • Immediately begins detecting motion upon launch
  • Turns off screen when motion is detected
  • Turns on screen when motion is no longer detected
  • Launchable through an intent
  • Detects motion while device is charging
  • Installation and management secured through Zebra Secure Storage Manager
  • Remotely configurable through StageNow or EMM system
  • Configurable features:
    • Service enable/disable
    • Delay screen blanking after motion detected (default = XX sec.)
    • Delay screen restoration after absence of motion detected (default = XX sec.)

Requirements

  • The Zebra Motion Screen Blank app (.apk):
    • Download from Zebra Support Portal
    • Download from Google Play
  • At least one supported Zebra device running Android 14 (or later)
  • StageNow or an EMM system with Zebra OEMConfig

Installation

  1. Install the app on a supported device
  2. Push the desired configuration .json file to device
  3. Trigger intent to launch the app through StageNow or EMM system

After successful installation of the ZebraMotionScreenBlank APK, configuration file has to be pushed to SSM Configuration file is a json format file

Usage

Launch automatically on reboot (register for boot complete intent) AFTER being launched one time manually (or through intent)

Enable Screen Blanking

The .JSON file below DOES THIS AND THAT


{
" motion_screen_blank_service ": “false"
 "key_blank_waiting_time": “2000",
 "key_restore_waiting_time": “4000",
}

Impact of the config on the app behavior:

1 . motion_screen_blank_service: Ω Default_value:motion_screen_blank_service=false

a.if motion_screen_blank_service true then MotionScreenBlankService will start.

b. if motion_screen_blank_service

        false MotionScreenBlankService will be stopped.

2 . key_blank_waiting_time : Default_value:key_restore_waiting_time=2000L the value of this will be the time in milli-seconds to wait before blanking the screen

3 . key_restore_waiting_time : Default_value:key_blank_waiting_time=4000L

the value of this will be the time in milli-seconds to wait before un-blanking the screen

After successfully pushing config to SSM ,we need to trigger intent to start the ZebraMotionScreenBlank service.

Mode: Send Intent when xml Processed. Action: Broadcast Android Action Name: com.zebra.motionscreenblank.ACTION_FIRST_LAUNCH Package Name : com.zebra.motionscreenblank Class Name : com.zebra.motionscreenblank.MotionScreenBlankBroadcastReceiver

After triggering intent wait for 20 seconds a notification from Motion Detection Service is obtained as Motion detection is running in the background. Connect to power source to start motion detection.
