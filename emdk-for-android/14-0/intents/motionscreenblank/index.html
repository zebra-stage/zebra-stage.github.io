Motion Screen Blanking

Motion Screen Blank 1.5

Overview

Motion Screen Blank is an Android app designed for Zebra devices used in motor vehicles. Using GPS and other technologies, the app detects when the device is in motion and automatically removes meaningful content from the screen, helping to prevent driver distraction and reducing the potential for injury. MSB is a "headless" app with no user interface available to the user or vehicle operator; it's installed and managed remotely.

The Motion Screen Blank image shown when vehicle is in motion. Click image to enlarge; ESC to exit.

Main Features

  • Launches automatically on re/boot
  • Immediately begins detecting motion upon launch
  • Turns screen off when motion is detected
  • Turns screen on when motion is no longer detected
  • Launchable through an intent
  • Detects motion whenever device is connected to a power source
  • Installation and management secured through Zebra Secure Storage Manager
  • Remotely configurable through StageNow or an EMM system with Zebra OEMConfig
  • Configurable features:
    • Service enable/disable
    • Delay screen blanking after motion detection (default = XX sec.)
    • Delay screen restoration after absence of motion is detected (default = XX sec.)

Requirements

  • The Zebra Motion Screen Blank app (.apk):

Installation

  1. Install the app on a supported device (details to come).
  2. Push the desired configuration .json file to device.
  3. Trigger an intent through StageNow or an EMM system to launch the app.

Usage

After installation, the Motion Screen Blank app must be launched at least once, either manually or by sending an intent. After the initial launch, MSB runs automatically whenever the device starts up.

Enable Screen Blanking

The .JSON file below configures the app to be active on the device, to blank the screen after two seconds of motion and restore the screen when motion ceases for four seconds. 


{
" motion_screen_blank_service ": “true"
 "key_blank_waiting_time": “2000",
 "key_restore_waiting_time": “4000",
}

Parameters

  • motion_screen_blank_service - Used to start/stop Motion Screen Blank service
    • true: start service
    • false: stop service (default)
  • key_blank_waiting_time - Sets the delay (in ms) between motion detection and screen blanking (default = 2000L)
    • Range: (INFO NEEDED)
  • key_restore_waiting_time - Sets the delay (in ms) between lack-of-motion detection and screen is restored (default = 4000L)
    • Range: (INFO NEEDED)

Send Intent

After deployment of the app and config file, broadcast an intent to start the service.

  • Mode: Send intent when XML is processed
  • Action: Broadcast
  • Package name: com.zebra.motionscreenblank
  • Android Action name: com.zebra.motionscreenblank.ACTION_FIRST_LAUNCH
  • Class name: com.zebra.motionscreenblank.MotionScreenBlankBroadcastReceiver

About 20 seconds after broadcasting the intent, a notification should indicate that the service is running in the background. Connect to power source to start motion detection.
