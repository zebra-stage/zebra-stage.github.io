入门

StageNow 3.0

简介

本部分提供有关 StageNow Android 部署解决方案的信息，包括部署、其目的和用户的说明。

部署

部署是设备管理员或操作员将设备配置和/或将固件或软件部署到设备以准备好生产使用的过程。这需要配置、部署和管理系统与设备。

可以在集中位置执行部署，企业中的所有设备都一起配置和部署，在某个特定存储或站点中配置和部署该站点上的所有设备，或在现场/针对每个用户为该站点或为该用户配置单个设备。

StageNow 部署解决方案

StageNow 是 Zebra 下一代 Android 部署解决方案，支持 KitKat 和 Jelly Bean，并且基于 MX 4.3 平台构建。它允许创建简单的配置文件，并轻松部署带简单条码或标签读取的设备。

StageNow 解决方案包括以下组件：

  • 部署工具提供了一个在部署工作站（主机）中访问的用户界面。除了其他部署操作以外，部署管理员通过配置设备组件使用此工具创建部署配置文件。部署工具包括存储在部署工具中输入内容的部署数据库和部署服务器，例如，设置模板和部署配置文件。部署数据是持续的。
  • 部署客户端驻留在设备上，并为部署操作员提供启动部署的用户界面。操作员使用部署模式，例如打印和扫描条码，以将部署材料传送到设备

支持的设备

StageNow 支持所有 KitKat 和 Jelly Bean Android 设备：

  • TC70 (Android KitKat 4.4.2)

  • TC55（非 GMS）(Android Jelly Bean 4.1.2)

  • MC40 (Android Jelly Bean 4.1.1)

StageNow 用户

以下是 StageNow 用户：

  • 部署管理员是“部署工具”的主要用户，并负责系统和客户端设备的配置、部署和管理。管理员创建指示要执行部署的配置文件，收集部署内容（例如应用程序），并将此内容上传到“部署工具”中。
  • 部署操作员识别要部署的设备，并使用“部署工具”查看和选择管理员发布的配置文件。然后，操作员确定要使用的部署材料来部署设备（例如条码），执行部署，并将设备交付给最终用户

部署方法/方式

以下是通过 StageNow 可用的部署方法。有关详情，请参阅第 1 页（共 4 页）的“设备部署”。

  • 条码配置文件 - 操作员打印包含配置文件信息的条码，并使用该设备扫描以配置该设备。
  • NFC 标签 - 操作员将配置文件数据导出至 NFC 标签，然后在特定距离内使用设备读取标签以将该配置文件部署到设备。
