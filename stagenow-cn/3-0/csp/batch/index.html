关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

批处理设置用来从另一个配置文件执行一个配置文件（可能包含任意数量的设置）。这可以被视为一种子例程调用。在 StageNow 中，批处理设置在内部（筹备管理员未明确引用）用于生成条码和 NFC 标签，以及实施从部署服务器读取的部署部分的执行。

批处理设置也可以被筹备管理员明确用来从另一个配置文件中利用一个配置文件的功能。这可用于创建可重复使用的“组件”作为配置文件，然后可通过在批处理设置中嵌入或引用它们从其他配置文件中使用它们。例如，一个筹备管理员可以创建和测试一个配置文件，演示并证明它提供了一些功能。之后，可将此配置文件导出为 XML 或另存为二进制文件（例如，用于编程到一个 NFG 标签中）。配置文件（XML 或二进制文件）之后可以嵌入到一个批处理设置中。然后，筹备管理员可在另一个配置文件中使用该批处理设置（通过重复使用已保存的设置），以便在创建的配置文件中放置该批处理设置的位置执行所嵌入的配置文件的整个功能。使用该批处理设置的筹备管理员不能看见在该批处理设置中嵌入的配置文件，尽管如此还是能够利用该嵌入式配置文件的功能。

批处理设置也可以引用设备文件系统中包含配置文件（XML 或二进制文件）的文件，而不是将批处理设置中嵌入到批处理设置中。这种使用情况需要“批处理文件”（包含配置文件的文件）存在于设备文件系统中，然后才能执行引用它的批处理设置，这样才会出现正确的结果。此类“批处理文件”可使用“文件”设置或通过其他某些“带外”方法传输到设备。StageNow 的“部署部分”功能将使用文件和批处理设置的组合自动在内部实施，无需管理员明确创建文件设置或批处理设置。但筹备管理员可以使用此类设置构建更高级的或自定义的筹备部署方案。

主要功能

  • 处理设备文件系统中包含配置文件的文件
  • 处理在批处理设置中嵌入的配置文件
  • 处理 XML 或二进制文件形式的配置文件
  • 处理可根据需要加密的配置文件
