关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

Battery Manager (BatteryMgr) 允许配置用来确定何时必须停用电池的阈值。

Zebra Android 设备可支持三类电池：常规（非智能）、智能和赠送的。常规电池仅提供有关电池电量的信息。智能电池增加了可唯一标识每个电池的能力，能够跟踪它的来源和使用历史记录。除了这些功能之外、赠送的电池还增加了提供详细的充电监控和健康信息以及更准确的“电池电量侦测”数据。

智能和赠送的电池的一个主要功能是能够确定电池何时接近其使用寿命结束并且应停用。对于智能电池，停用完全基于电池的充电历史记录。对于赠送的电池，停用决定可基于有关电池状况的更丰富信息做出。在任一情况下，有关何时必须停用电池的决定均是可配置的，应考虑在不同环境中对电池的需求。

BatteryMgr 允许配置用来确定何时应停用电池的阈值。配置完毕后，这些值（与其他电池参数一起）由电池管理软件用来确定在智能电池或赠送的电池中存储的停用标记值，之后便随电池一起携带。

支持的阈值：

  • 电池使用停用阈值
    • 如果电池使用次数大于或等于电池使用停用阈值，则将电池视为要“停用”。
    • 电池使用停用阈值适用于智能电池以及 TC50 和 TC75 设备的设备。
    • 注意：电池使用次数是电池的累计充电量除以额定容量。
  • 健康度停用阈值
    • 如果电池健康度小于或等于健康度停用阈值时，则将电池视为要“停用”。
    • 健康度停用阈值适用于赠送的电池。
    • 注意：健康度是在“design_capacity”的放电率条件下“present_capacity”与“design_capacity”的比值。

使用 BatteryMgr 配置的阈值存储在 batteryusage_decommission_threshold 或 percent_decommission_threshold 文件中，在后续的重启或企业重置后仍保留在设备上。恢复出厂设置将擦除所有此类用户数据。尝试对智能电池或赠送的电池之外的电池设置停用阈值将会在结果 XML 文档中返回错误。

主要功能

  • 为智能电池设置电池使用停用阈值
  • 为赠送的电池设置电池健康度停用阈值

使用停用阈值

用于在支持智能电池的设备上设置电池使用停用阈值。如果智能电池的“电池使用次数”大于或等于当前配置的电池使用停用阈值，则将该电池视为“已准备停用”。此参数的默认值为 400，表示一旦经历过大约 400 个“充电/放电”周期后就应该停用。

注意：此参数可在支持智能电池的 Zebra Android 设备上使用，无论智能电池当前是否安装在设备上。这使管理员可以在安装智能电池之前为目前正在使用常规电池的设备配置停用阈值。如果在不支持赠送的电池的设备上使用此参数，则会在结果 XML 文档中返回错误。

智能电池保留电池在一段时间内累计的充电/放电周期总数。这种累计也被称为累计充电。累计充电量数值较高的电池的剩余使用时间比累计充电量数量值低的电池短是合理的预期。

“电池使用次数”的计算方法是智能电池的累计充电量除以其额定容量。因此，我们试图使用电池使用次数来评估电池经历绝对数量的充电和放电后对电池健康的影响情况。

重要说明：目前不确定完整充电周期与部分充电周期是否对电池健康度有影响。累计充电量类似的两个电池将有类似的电池使用次数。但是，如果这些电池有不同的充电/放电模式，比较它们的“电池使用次数”可能无法揭示有关其相对健康度的有用信息。唯一确定的是，给定电池健康度会随着计算的“电池使用次数”的增加而降低。因为，认为“电池使用次数”高的电池没有“电池使用次数”低的电池健康度好是合理的。通过设置一个基于非常高的“电池使用次数”的阈值，可以识别出最可能需要更换的电池。

参数值输入规则：

  • 0 - 65535 之间的整数
  • 设置为 0（或 XML 中缺少此参数）导致没有更改，或表示“不适用”

参数名称：SetBatteryUsageDecommissionThreshold

需要：

  • OSX：4.4+
  • MX：4.4+
  • Android API：16+

健康度停用阈值

用于在支持赠送的电池的设备上设置电池健康度停用阈值。如果赠送的电池的“电池健康度”小于当前配置的电池健康度停用阈值，则将该赠送的电池视为“已准备停用”。此参数的默认值为 75，表示智能电池在实际容量小于原始额定容量的 75% 后就应该停用。

赠送的电池可以确定在特定放电条件下它们的“实际容量”。基于这一点，赠送的电池根据实际容量相比“额定容量”的百分比来估计它们的电池健康度。随着时间的推移，实际容量和电池健康度将消减，有理由预期“电池健康度”值较低的电池的剩余使用寿命比“电池健康度”值较高的电池短。

注意：此参数可在支持赠送的电池的 Zebra Android 设备上使用，无论赠送的电池当前是否安装在设备上。这使管理员可以在安装赠送的电池之前为目前正在使用智能电池的设备配置停用阈值。如果在不支持赠送的电池的设备上使用此参数，则会在结果 XML 文档中返回错误。

参数值输入规则：

  • 0 - 100 之间的整数
  • 设置为 0（或 XML 中缺少此参数）导致没有更改，或表示“不适用”

参数名称：SetPercentDecommissionThreshold

需要：

  • OSX：4.4+
  • MX：4.4+
  • Android API：16+
