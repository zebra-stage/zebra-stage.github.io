关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

Ethernet Manager 用于管理通过配备以太网的通讯座连接到网络的设备的设置。此 CSP 对于 Wi-Fi 通信或无线以太网适配器设置没有影响，它们是通过 Wi-Fi Manager 进行控制的。

主要功能

  • 启用/禁用通过配备以太网的通讯座进行的网络连接
  • 配置通讯座使用 DHCP
  • 手动设置通讯座的 IP 地址设置
  • 启用/禁用代理的使用
  • 设置代理服务器端口和代理主机名
  • 指定绕过代理的主机名和/或 IP 地址

以太网启用/禁用

这是设备放置在配备以太网的通讯座后通过以太网连接网络的能力的开/关键。

此参数对于 Wi-Fi 通信或无线以太网适配器设置没有影响。

参数名称：EthernetState

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会更改以太网状态；任何以前选择的设置都将保留。

OSX：6.2+

MX：6.2+

Android API：23+
1 打开 启用通过以太网通讯座端口的网络连接。

OSX：6.2+

MX：6.2+

Android API：23+
2 关闭 禁用通过以太网通讯座端口的网络连接。

OSX：6.2+

MX：6.2+

Android API：23+

配置 IP 地址（复选框）

控制是否可以给配备了以太网的通讯座分配 IP 地址。当启用时（值=1），可以自动从 DHCP 服务器获取 IP 地址（默认设置），或由管理员使用相关参数手动分配。

参数名称：ConfigIpAddr

选项 名称 说明需要
0 （未选中） 阻止手动分配或通过 DHCP 分配 IP 地址。

OSX：6.2+

MX：6.2+

Android API：23+
1 （选中） 允许手动分配或通过 DHCP 分配 IP 地址

OSX：6.2+

MX：6.2+

Android API：23+

使用 DHCP（复选框）

控制配备以太网的通讯座是使用 DHCP 自动获取它的 IP 地址还是由管理员使用相关参数手动分配。当禁用时（值=0），必须使用以下相关参数指定设备的 IP 地址、默认网关、子网掩码和至少一个 DNS 服务器。

在以下条件下显示：ConfigIPAddress 为“已选中”

参数名称：UseDHCP

选项 名称 说明需要
0 （未选中） 允许手动配置 IP 地址设置。

OSX：6.2+

MX：6.2+

Android API：23+
1 （选中） 设置设备从 DHCP 服务器获取 IP 设置。

OSX：6.2+

MX：6.2+

Android API：23+

设置 IP 地址

用于指定以太网通讯座的 IP 地址。必须是通讯座所连接的网络上唯一的地址。

参数值输入规则：

  • 包含有效 IPv4 地址的字符串。例如：“192.168.0.100”

在以下条件下显示：“配置 IP 地址”框选中并且“使用 DHCP”框未选中

参数名称：IpAddress

需要：

  • OSX：6.2+
  • MX：6.2+
  • Android API：23+

设置默认网关

用于指定以太网通讯座默认网关的 IP 地址，通过它路由与外部网络之间的数据包。仅当设备必须访问其本地子网之外的网络或从这些网络访问该设备时，才需要此参数。

参数值输入规则：

  • 包含有效 IPv4 地址的字符串。例如：“192.168.0.1”

在以下条件下显示：“配置 IP 地址”框选中并且“使用 DHCP”框未选中

参数名称：网关

需要：

  • OSX：6.2+
  • MX：6.2+
  • Android API：23+

设置子网掩码

用于指定以太网通讯座的子网掩码，它定义设备将在上面运行的网络细分。

参数值输入规则：

  • 包含有效 IPv4 地址的字符串。例如：“255.255.255.0”

在以下条件下显示：“配置 IP 地址”框选中并且“使用 DHCP”框未选中

参数名称：网络掩码

需要：

  • OSX：6.2+
  • MX：6.2+
  • Android API：23+

设置主要 DNS

用于指定以太网通讯座的主要域名服务器。

参数值输入规则：

  • 包含有效 IPv4 地址的字符串。例如：“8.8.8.8”

在以下条件下显示：“配置 IP 地址”框选中并且“使用 DHCP”框未选中

参数名称：DNS1

需要：

  • OSX：6.2+
  • MX：6.2+
  • Android API：23+

设置辅助 DNS

用于指定以太网通讯座的辅助域名服务器。

参数值输入规则：

  • 包含有效 IPv4 地址的字符串。例如：“8.8.4.4”

在以下条件下显示：“配置 IP 地址”框选中并且“使用 DHCP”框未选中

参数名称：DNS2

需要：

  • OSX：6.2+
  • MX：6.2+
  • Android API：23+

以太网代理启用/禁用

控制设备在通过配备以太网的通讯座连接到网络时是否使用以太网代理。当选择“无”时（值=1），所有以太网事务处理都直接通过设备执行。当选择“手动”时（值=2），必须指定代理主机名和代理主机端口，事务处理将通过该服务器路由。

参数名称：SetEthernetProxySettings

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会更改以太网代理设置；任何以前选择的设置都将保留。

OSX：6.2+

MX：6.2+

Android API：23+
1 无 禁用以太网代理的使用。

OSX：6.2+

MX：6.2+

Android API：23+
2 手动 启用以太网代理的使用。

OSX：6.2+

MX：6.2+

Android API：23+

设置代理主机名

用于指定用来传递网络通信的代理服务器的主机名。

参数值输入规则：

  • 1 到 64 个字符的字符串，指定代理主机名。例如："www.MyProxyHostServer.com"

在以下条件下显示：以太网代理设置为“手动”

参数名称：ProxyHostName

需要：

  • OSX：6.2+
  • MX：6.2+
  • Android API：23+

设置代理端口

用于指定用来访问代理服务器的端口号（2 位数字），通过该代理服务器来传递网络通信。

参数值输入规则：

  • 两位数的整数，用来指定正在使用的代理服务器的端口号

在以下条件下显示：以太网代理设置为“手动”

参数名称：ProxyPort

需要：

  • OSX：6.2+
  • MX：6.2+
  • Android API：23+

设置绕过代理

用于指定设备直接访问的主机名和/或 IP 地址，绕过“代理主机名”参数中指定的代理服务器。可以用任意组合指定多个名称和/或 IP 地址，使用逗号分隔。通常用于指定内部服务器和/或作为一种手段来防止指定代理服务器上流量过载。绑定给任何主机而未使用此参数特别排除在外的流量将通过指定的代理服务器进行路由。

Zebra 建议在常规部署之前，对工作设备上的绕过代理设置进行测试。

参数值输入规则：

  • 1 到 256 个字符的字符串，指定一个或多个主机名和/或 IP 地址，使用逗号分隔。例如：“www.MyBypassProxy.com,192.168.1.100, our.internalServer”

在以下条件下显示：以太网代理设置为“手动”

参数名称：BypassProxy

需要：

  • OSX：6.2+
  • MX：6.2+
  • Android API：23+
