关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

UI Manager (UiMgr) 可用来控制各种各样的 UI 配置设置，如语言和区域设置的配置、设备通知下拉列表、快速设置控件、默认输入设备以及剪贴板行为。

Android 设备包含各种系统应用程序，用来访问分布在整个操作系统中的 UI 元素。在某些情况下，对于企业中部署的设备，Zebra 客户和/或合作伙伴可能不喜欢这些“标准”UI 元素中的一项或多项。UiMgr 提供对这些 UI 元素的控制以及禁用有可能妨碍从事特定商业任务的设备用户的工作效率的 Android 行为或功能的选项。

安全警告：UiMgr 允许禁止“网络可能被监控...”消息。这一重要保护措施旨在警告设备用户有潜在的安全风险。除非在设备上安装 CA 证书之前已正确审查了证书，否则管理员不应禁止此消息，并且应确保后来不会在消息被禁止时安装不受信任的证书。

主要功能

  • 允许/禁止使用以下各项：
    • 快速设置，用于访问 Wi-Fi、蓝牙和其他设置
    • 剪贴板功能，允许设备用户在应用程序之间复制信息
    • 自动更正功能，它存储通常输入的字词，使它们能够输入以代替错误键入的类似字词
    • Home 键功能，用来退出当前应用程序并显示主页屏幕应用程序
    • 蓝牙配对弹出功能，它会请求设备用户参与，即使应用程序已提供了所有需要的配对信息
    • “网络可能受到监控...”消息
    • “最近应用程序”按钮
    • 导航栏
  • 配置本地化设置以：
    • 针对所需的语言和区域设置当前区域设置
    • 设置默认输入方法以增强本地化数据输入
  • 清除剪贴板，将放弃先前可能已复制的数据

自动更正使用启用/禁用

控制在设备上是否可访问“自动更正”功能。“自动更正”可用于保存经常输入的字词或行业特定的字词或短语，并使用它们来更正可能错误键入的字词。此类功能可能会带来安全风险，因为它会将敏感信息暴露给后来无权查看或使用该信息的设备用户。

注意：此参数不会打开或关闭“自动更正”功能或配置其任何功能。在“自动更正”处于启用状态时，系统管理员或设备用户仍必须使用“系统设置”面板来打开或关闭该功能以及配置它的功能。当禁用时，“自动更正”功能不能在设备上使用。

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会更改是否允许使用“自动更改”功能；任何以前选择的设置都将保留。

1 启用 允许使用“自动更正”功能，允许系统管理员或设备用户使用“系统设置”面板打开和关闭功能以及配置它的设置。

MX：4.3+

设置当前区域设置

允许选择设备的区域设置。一个区域设置是语言以及使用该语言的区域的组合。语言使用两个小写字母进行标识；区域使用两个大写字母进行标识 。区域设置使用下划线（“_”）字符将两对字符连起来。.例如，“en_US”是为“美国英语”指定的区域设置，“fr_CA”是为“加拿大法语”指定的区域设置。

在更改区域设置时，还可能希望将默认输入法更改为适合输入与选定区域设置关联的语言字符的输入法。将某个特定输入法设置为默认输入法时，不会阻止设备用户手动选择不同的输入法，它只是确定哪种输入法在每次调用该参数时自动弹出。

控制在以编程方式请求更改或由设备用户手动请求更改时是否更改默认输入法（也就是软键盘）。将某个特定输入法设置为默认输入法时，不会阻止设备用户手动选择不同的输入法，它只是确定哪种输入法在每次调用该参数时自动弹出。在更改当前区域设置时，还可能希望将默认输入法更改为适合输入与选定区域设置关联的字符的输入法。

参数名称：InputMethodAction

选项 名称 说明需要

输入法包名称

用于要设为默认输入法（也就是软键盘）的用户定义输入法的 Android 包名称。

注意 - 指定的包名称必须与已安装的输入法的包名称匹配。如果试图指定与已安装的输入法不匹配的包名称，将会在结果 XML 文档中返回错误。

参数值输入规则：

  • 包含当前已安装并且实施有效输入法的 Android 应用程序的有效包名称的字符串。例如：com.mycompany.myinputmethod
  • 最小长度为 1 个字符的字符串
