关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

USB Manager (UsbMgr) 控制可在设备上使用哪些 USB 功能。

Android 设备一般都有一个或多个通用串行总线 (USB) 端口，可用于各种用途。给定的 USB 端口可能实施“主机”模式，用于连接各种 USB 外围设备（例如鼠标、键盘、USB 存储设备）到 Android 设备。给定的 USB 端口也可能实施“外围设备”模式，用于将 Android 设备连接到一个主机，如 PC。给定的 USB 端口还可以实施移动 (OTG) 支持，可根据连接的电缆进入“主机”或“外围设备”模式。

主要功能

  • 允许或阻止使用以下各项：
    • USB 的所有模式和功能
    • 外部 USB 存储设备
    • 通过 (Android Debug Bridge) ADB 连接到 PC
    • PC 通过 USB 对设备存储的所有访问
    • PC 通过 USB　使用　USB 大容量存储 (UMS) 访问设备存储
    • PC 通过 USB　使用媒体传输协议　(MTP) 访问设备存储
    • PC 通过 USB　使用图片传输协议　(PTP) 访问设备存储

USB 模块使用

控制是否允许使用 USB 模块。设置此参数将导致启用或禁用所有 USB 功能。

参数名称：UsbModuleUsage

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）将导致不更改是否可以使用 USB 模块。此值还会导致提供其他参数，以允许控制单个 USB　功能。

OSX：1.3+

MX：4.2+
1 启用 此值将导致允许使用 USB 模块。此值还会导致提供其他参数，以允许控制单个 USB　功能。

OSX：1.3+

MX：4.2+
2 禁用 此值将导致阻止对　USB 模块的所有使用。此值还会阻止呈现其他参数，因为在阻止对　USB　模块的所有使用时，也就不能使用个别的　USB 功能。

OSX：1.3+

MX：4.2+

外部存储启用/禁用

控制是否允许使用 USB 外部存储设备。允许时，可能必须通过支持主机模式的　USB　端口、使用 OTG 适配器电缆或 OTG 兼容存储设备通过 OTG USB 端口连接 USB　外部存储设备。使用　USB 外部存储可能带来安全风险，因为它可能被用作从设备提取敏感数据或向设备注入恶意数据或应用程序的路径。

在以下条件下显示：“USB 模块使用”值不是“禁用”

参数名称：UsbExternalStorageUsage

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）将导致不更改是否可以使用 USB 外部存储。

OSX：4.1+

MX：4.2+
1 启用 此值将导致允许使用 USB 外部存储。

OSX：4.1+

MX：4.2+
2 禁用 此值将导致阻止使用 USB 外部存储。

OSX：4.1+

MX：4.2+

ADB 启用/禁用

控制在设备上是否可以启用 Android Debug Bridge (ADB) 模式。允许时，可以通过支持“外围设备”模式的 USB 端口或使用 USB“移动”电缆将 PC 连接到设备。一旦连接，ADB 命令行工具可能允许 PC 与 Android 设备之间通信，用于复制文件、安装应用程序、运行 shell 命令等。使用 ADB 可能会带来安全风险，因为它可能被用作从设备中提取敏感数据、向设备注入恶意数据或应用程序、删除或修改设备上重要应用程序的操作的路径。

在以下条件下显示：“USB 模块使用”值不是“禁用”

参数名称：UsbADBUsage

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会对设备上的设置执行更改；任何以前的设置将被保留。

OSX：1.3+

MX：4.2+
1 启用 允许在设备上启用 ADB 模式。

OSX：1.3+

MX：4.2+
2 禁用 阻止在设备上启用 ADB 模式。

OSX：1.3+

MX：4.2+

USB 存储启用/禁用

控制是否使用允许使用通过 USB 从 PC 直接访问设备存储的方法。这控制对 USB 大容量存储 (UMS)、媒体传输协议 (MTP) 和图片传输协议 (PTP) 的使用。不控制对 ADB 模式的使用，该模式单独控制。

允许这些模式时，可以通过支持“外围设备”模式的 USB 端口或使用“移动”(OTG) USB 端口将 PC 连接到设备。一旦连接，此 PC 就可以浏览设备文件系统中暴露的部分，将文件复制到设备或从中复制文件，删除文件等。使用这些模式可能带来安全风险，因为它可能被用作从设备提取敏感信息、向设备注入恶意数据或应用程序、从设备中删除重要信息的途径。

注意：此参数是在 B 组设备上启用/禁用 MTP 和 PTP 模式的唯一方法。

在以下条件下显示：“USB 模块使用”值不是“禁用”

参数名称：UsbAllAccessDeviceStorageUsage

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会对设备上的设置执行更改；任何以前的设置将被保留。

OSX：4.1+

MX：4.2+
1 启用 允许通过 USB 从 PC 访问设备存储。此值还会阻止呈现其他参数，因为在允许使用所有模式时，也就不能控制单个模式。

OSX：4.1+

MX：4.2+
2 禁用 阻止通过 USB 从 PC 访问设备存储。此值还会阻止呈现其他参数，因为在阻止使用所有模式时，也就不能控制单个模式。

OSX：4.1+

MX：4.2+

通过 UMS 访问存储

控制是否允许使用 USB 大容量存储 (UMS)。当允许此模式时，可以通过支持“外围设备”模式的 USB 端口或 OTG USB 端口将 PC 连接到设备。一旦连接，此 PC 就可以浏览设备文件系统中暴露的部分，将文件复制到设备或从中复制文件，删除文件、甚至是对整个存储卷进行格式化。使用此模式可能带来安全风险，因为它可能被用作从设备提取敏感数据、向设备注入恶意数据或应用程序或者从设备中删除信息的路径。

在以下条件下显示：“USB 模块使用”值不是“禁用”*并且*“通过 USB　从　PC　访问设备存储”不是“启用”或“禁用”

参数名称：UsbUMSAccessDeviceStorageUsage

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）将导致不更改是否可以通过 USB UMS　从　PC　中访问设备存储。

OSX：4.1+

MX：4.2+
1 启用 此值将导致允许通过 USB UMS 从 PC 访问设备存储。

OSX：4.1+

MX：4.2+
2 禁用 此值将导致阻止通过 USB UMS 从 PC 访问设备存储。

OSX：4.1+

MX：4.2+

注意：B 组设备上不支持此参数；改用“USB 存储启用/禁用”参数。

OSX：4.1+

OSX：4.1+
