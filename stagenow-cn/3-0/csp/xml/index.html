关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

XmlMgr 指定 MXMS 在处理请求 XML 文档时将使用的错误处理模式。

MXMS 处理包含一个或多个在 CSP 中定义的功能的请求 XML 文档。每个功能在请求 XML 文档中都用一个顶层特征标记表示，其类型与该 CSP　功能匹配。例如，XML　中“XmlMgr”CSP 的功能开头为：

<characteristic type="XmlMgr" ...

MXMS 执行功能，将它们传递到对应的　CSP，按照它们在请求　XML　文档中的出现顺序，将特征传递到对应的　CSP。当某个 CSP 执行某项功能时，CSP 将返回结果 XML。CSP 返回特征标记表明该功能成功。CSP　返回特征-错误标记表明该功能失败。MXMS 按照功能的执行顺序汇总由 CSP 返回的结果 XML，以创建最终的结果 XML。

为了讨论 MXMS 错误处理的目的，能够完全执行的任何功能都视为成功，不能完全执行的功能都视为失败。错误由 MXMS 根据生效的错误处理模式进行处理。MXMS 有一个默认的错误处理模式，在每个被处理的新请求 XML 文档的开头始终生效。该错误处理模式可保留为默认设置，也可在一个请求 XML 文档中被更改一次或多次。

当默认错误处理模式“执行所有到结束”生效时，MXMS 将执行所有功能，无论它们成功还是失败，一直到 XML 的末尾。MXMS 将返回结果 XML，其中包含有特征标记和特征-错误标记，它们加起来的数量与请求 XML 文档中的特征标记数量相同。此错误处理模式基本上会尝试执行所有功能，最后报告所发生的情况。此模式是默认设置，因为它适合大多数一般的情况。但是，如果一个功能依赖于另一个，此模式可能导致“多米诺效应”，一个失败导致整个序列失败。

MXMS 还支另一种错误处理模式，即“执行直到出错，然后停止”，在此模式下，它在第一次失败时即停止执行功能，然后返回只包含执行成功的功能的特征标记以及执行失败的功能的特征-错误标记的结果 XML。当功能依赖于彼此时，使用此模式可能有利，因为它能够避免一次失败导致整个序列失败的“多米诺效应”，还会让执行尽快终止，而不用尝试执行注定要失败的功能。

XmlMgr 允许您指定在处理请求 XML 文档时 MXMS 应使用的错误处理模式。如果在请求 XML 文档中没有 XmlMgr 功能，则使用默认错误处理模式处理整个文档。如果请求 XML 文档中存在 XmlMgr 功能，MXMS 将使用 XmlMgr 功能指定的错误处理模式来执行随后的功能，直到出现下一个 XmlMgr 功能或到达请求 XML 文档的结尾（以先达到的条件为准）。

主要功能

  • 指定要使用的 XML 处理模式
    • 执行所有功能并返回所有成功和失败
    • 执行功能直到第一个错误，然后停止并返回所有成功和第一个失败
    • 执行功能直到第一个错误，然后跳至下一个“Catch”或结束
    • 停止跳过并恢复执行功能

XML 处理模式

用于指定 MXMS 用于处理请求 XML 文档的错误处理模式。

参数名称：ProcessingMode

选项 名称 说明需要
1 执行所有到结束 设置该错误处理模式时，MXMS 将尝试按照功能在请求 XML 文档中的出现顺序执行其中的所有功能，而不管每个功能是成功还是失败。然后，结果 XML 将按照尝试执行功能的顺序返回所有成功和所有失败。

MX：4.2+
2 执行直到出错，然后停止 设置该错误处理模式时，MXMS 将尝试按照功能在请求 XML 文档中的出现顺序执行每项功能，只要这些功能成功就会继续，但是只要出现第一次失败就会停止，并且忽略请求 XML 中剩余的功能。然后，结果 XML 将按照尝试执行功能的顺序返回所有成功和第一个失败。

MX：4.2+
3 尝试直到出错，然后跳至 Catch 设置该错误处理模式时，MXMS 将尝试按照功能在请求 XML 文档中的出现顺序执行每项功能，只要这些功能成功就会继续，在出现第一次失败时就会停止，并且开始跳过功能。将一直跳过功能，直到遇到 ProcessingMode 参数值为“4”（“捕获错误，执行其余”）的 XmlMgr 功能，或者到达请求 XML 文档的结尾。如果已到达“Catch”，MXMS 将继续执行随后的功能，并且错误处理模式设置为“1”（“执行所有到结束”）。然后，结果 XML 将按照尝试执行功能的顺序返回所有成功和第一个失败。

MX：4.2+
4 捕获错误，执行其余 表示如上所述的“Catch”点的位置。
