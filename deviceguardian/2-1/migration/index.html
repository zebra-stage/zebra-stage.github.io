Migration

Device Guardian + Access Management 2.1

All references to Device Guardian also apply to Device Guardian Access Management within this guide, unless otherwise specified.

Overview

Device Guardian has been rebranded from Device Tracker, featuring updated user interfaces for both the web portal and Android client. Existing Device Tracker users who wish to migrate to Device Guardian can follow the instructions provided in this migration guide.

The migration from Device Tracker to Device Guardian is separated into two distinct phases to ensure a smooth and manageable transition:

  1. Pre-Server Upgrade - Update Device Guardian to ensure all devices meet the minimum requirements.
  2. Post-Server Upgrade - Migrate the device-side client software from Device Tracker to Device Guardian.

Updated Timeline and Key Changes:

  • Extended Deadline: Customers have until June 1, 2026, to plan, test, and deploy the Device Guardian client to their devices. This extends the original deadline of December 31, 2025, communicated earlier.
  • Server Upgrade Scheduling: Once the pre-server upgrade preparations are complete, customers can schedule their server upgrade directly with Zebra.
  • License Management Enhancements:
    • Zebra is currently updating all Device Tracker servers with a new license management component.
    • Existing Device Tracker licenses will be automatically transitioned to Device Guardian licenses.
    • This update is transparent, will not affect system functionality, and enables customers to purchase Device Guardian SKUs instead of Device Tracker SKUs for adding new licenses.
Play Store Deployments: Deploying via the Play Store requires an additional LifeGuard (LG) or hotfix update after the upgrade. If your EMM supports sideloading, Zebra recommends using the Sideloading method until the next LG update becomes available for the device.

Key Enhancements

The following outlines the enhancements and key points of the upgrade:

  • Device Compatibility: Device Guardian supports devices running Android 11 and higher. Mixed deployments of Android 8, 10 and 11 or higher are not supported. All devices must be upgraded to Android 11 or later with a compatible LifeGuard Update. For compatible LifeGuard updates, refer to the Zebra support portal.
  • Client Release: The new Device Guardian client has been released with package name com.zebra.mdna.dg and version 1.0.
  • Configuration Management: Device Guardian client settings are now managed through Enterprise Mobility Management (EMM) configurations.
  • Device Tracker Compatibility: Device Tracker client versions earlier than v6.3 are not compatible with the new Device Guardian Server.
  • Identity Guardian Compatibility: Device Guardian is compatible with Identity Guardian v2.0 or later.
  • New APIs: APIs are now available to retrieve and share data between applications, including Identity Guardian.
  • Device Tracker Deprecation: No new features for Device Tracker will be released; only bug fixes will be provided until December 31, 2025.

No changes:

  • Existing purchased license counts remain unchanged.

Migration Benefits

  • Improved user interface and experience
  • Faster performance and access to new features
  • Platform is designed for future enhancements
  • Ongoing support and updates

Pre-Server Upgrade

To prepare for your upgrade from Device Tracker to Device Guardian, it is essential to review the following details to understand your specific requirements.

Device Compatibility:

  • Unsupported Devices - Devices running Android 8 or 10 are not supported. Options for unsupported devices:
    • Upgrade Hardware - Transition to hardware compatible with Device Guardian.
    • Remain on Device Tracker - Note that no new features will be added to Device Tracker.
  • Supported Devices - Devices upgradable to Android 11 or higher can migrate to Device Guardian.

Software Requirements:

  • Devices running Device Tracker v6.2 or later are supported by Device Guardian.
  • If running an earlier version, upgrade to Device Tracker v6.2 or higher.

Identity Guardian CheckOut/CheckIn:

  • Devices using Device Tracker v6.1 with native CheckOut/CheckIn must migrate to Identity Guardian v2.0 or higher, as CheckOut/CheckIn is now managed through Identity Guardian.

Preparation Completion: Once these requirements are met, contact Zebra Support to schedule your server upgrade.

Post-Server Upgrade

After the server upgrade, the migration involves transitioning the device client from Device Tracker to Device Guardian.

Important Notes:

  • Play Store Deployment - The Play Store deployment of the Device Guardian client requires either a hotfix or an updated LifeGuard (LG) update or both; see Play Store Hotfix/LifeGuard Update below.
    • Alternative Mitigation - If your EMM (Enterprise Mobility Management) supports APK sideloading, use the Deployment via Sideloading method until the next LG update is released.
  • Configuration Management - Device Guardian client settings are now managed via EMM Managed Configuration.

Managing Play Store Updates

By default, Device Guardian and other Google Play apps are set to update automatically. However, if Identity Guardian is in use, administrators managing Android devices must prevent automatic updates before upgrading the OS.

Options for Managing Updates:

  • Disable Automatic Updates - Submit the XML to disable auto updates via your EMM platform.
  • Enable Automatic Updates - Submit the XML to enable auto updates via your EMM platform.
  • Alternative Methods - Refer to the blog post Preventing Play Store Apps from Updating Automatically.

Deployment via Play Store

Follow the steps below for devices with Device Tracker v6.2 or higher.

Preparation:

  1. For devices upgrading to Android 13 or higher, disable Play Store automatic updates.
  2. Upgrade the OS to meet the minimum requirements for Device Guardian.
  3. If applicable, deploy the necessary hotfix after the OS upgrade (available through Zebra Support).
  4. For CheckOut/CheckIn users, prepare the deployment of Identity Guardian v2.0 or higher via EMM.

Deployment Steps:

  1. Uninstall Device Tracker.
  2. Deploy the Device Guardian client and configuration via EMM.
  3. Deploy the Identity Guardian client and configuration via EMM.
  4. (Optional) Re-enable Play Store updates for devices on Android 13 or higher.

Deployment via Sideloading

Follow the steps below for devices with Device Tracker v6.2 or higher.

Preparation:

  1. For devices upgrading to Android 13 or higher, disable Play Store automatic updates.
  2. Upgrade the OS to meet the minimum requirements for Device Guardian.
  3. For CheckOut/CheckIn users, prepare the deployment of Identity Guardian v2.0 or higher via EMM.

Deployment Steps:

  1. Uninstall Device Tracker.
  2. Deploy the Device Guardian client APK and configuration via EMM.
  3. Deploy the Identity Guardian client and configuration via EMM.
  4. (Optional) Re-enable Play Store updates for devices on Android 13 or higher.
