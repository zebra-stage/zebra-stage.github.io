
> **All references to Device Guardian within this guide also apply to Device Guardian Access Management, unless explicitly stated otherwise.**

## Overview

This section provides a quick-start guide for using Device Guardian (DG) and Device Guardian Access Management (DGAM), along with configuration, administration, and usage information.

---

## Getting Started

### Preparation

Ensure all [requirements](../setup/#requirements) are met, including network requirements.

### Device Setup

For both DG and DGAM:

- **[Install DG and Enroll Devices via EMM](../setup) -** For DGAM, this process also applies to kiosks.
- **Install Identity Guardian for Single Sign-On (SSO) -** To enable [SSO](../config/#singlesignonsso) for user accountability, install [Identity Guardian](/identityguardian) v2.0 or higher on devices and configure it with [SSO Authentication Configuration](/identityguardian/latest/mc/#ssoauthenticationconfiguration). This step is optional for DG, but required for DGAM.

**For DGAM Only:**

- **[Grant Required Permissions](../setup/#devicerequirements) -** Ensure all necessaary permissions are enabled for both kiosks and devices.
- **[Import DataWedge Profile](../setup/#importdatawedgeprofile) -** Import the profile to enable barcode scanning on devices to register to a kiosk.
- **Assign the [Kiosk](../setup/#managekiosks) to a Site -** After enrolling the kiosk to the server, assign it to the appropriate site for monitoring and management.
- **[Register Devices to Kiosk](../setup/#registerdevicetokiosk) -** Enables localized monitoring of devices directly through the kiosk.

### Device Guardian Portal Setup

Follow these setup steps:

1. **[Enroll Devices](../config/#deviceenrollment) –** Enroll devices to your organization's cloud server to enable device-server communication.
2. **[Add Sites](../config/#addsites) –** Create a site entry for each location.
3. **[Add Access Points (APs)](../config/#addaccesspoints) –** Register APs and assign each to its respective site. Ensure that the AP's friendly name (e.g., department, area) is user-friendly to facilitate device searches.
4. **[Add Devices](../config/#adddevices) -** Input device information into the server.
5. **[Add Users](../config/#adduser) –** Add users for administrators and managers. End users do not need to be added. If using [single sign-on (SSO)](../config/#singlesignonsso) with role-based mapping, this step is not required.

**For DGAM Only:**

In addition to the above steps:
1. **[Add Kiosks](../config/#managekiosks) -** Register kiosks to enable localized device monitoring and management.
2. **[Add Kiosk Devices](../config/#managekioskdevices) -**  Add devices associated with kiosks to ensure proper tracking and accountability.

## Next Steps

After setting up Device Guardian, consider various configuration and administration options for optimal device tracking and device recovery. This section offers further guidance on these considerations.

### Configuration

- **Automate Site Assignments -** Streamline [site assignments](../config/#siteassignment) by automatically assigning devices to a site based on their connected AP or IP address range.
- **Implement User Accountability -** Integrate Device Guardian with [SSO](../config/#singlesignonsso) to associate devices with users via login credentials, providing admins real-time visibility into device usage.
- **Map-Based Location Tracking -** Enable [Map Based Locationing](../config/#mapbasedlocationing) to retrieve GPS coordinates and visualize their locations on a map.
- **Automate Workflow Processes -** Configure Device Guardian to automatically update device states based on predefined conditions or thresholds, [streamlining workflows](../config/#automation) and reducing manual oversight.
- **Nearby Lost Device Alerts -** Activate [Lost Device Nearby notifications](../config/#lostdevicenearby) to alert users when a missing device is detected in close proximity. Notifications can include audio, vibration, LED, or Android alerts to prompt recovery actions.
- **Offline Device Tracking-** Use [Secondary Bluetooth Low Energy (BLE)](../secondaryble/) to track device locations even when powered off or with critically low battery (below 5%).
- **Customize Audio Alerts -** Personlize [audio alert](../config/#playsound) for lost devices by selecting tones, adjusting volume, and setting duration or intervals.
- **Track Bluetooth Accessories -** Locate lost [Bluetooth accessory scanners](../config/#bluetoothscanners) with tracking features. Enable **Virtual Tethering** to alert users when a paired scanner moves out of range from the host device.

**For DGAM Only:**
- **[Device Auto Assignment](../config/#deviceautoassignment) -** Automatically assign devices to the nearest kiosk within Bluetooth range.
- **[One Device Per User](../config/#onedeviceperuser) -** Restrict users to logging into a single device at a time when managed by a kiosk.

### Device & Accessory Recovery

- **Find a Device:**
    1. **Mark Device to Find -** Admins or managers [mark a lost device](../use/#markdevicetofind) as **To Be Found,** adding it to the **Missing** device list for recovery actions.
    2. **Search Device -** A user selects a device in the **Missing** state and performs the search process to [find the device](../use/#finddevice), using the proximity meter and sending audio alerts.
- **Find a Device [via Kiosk](../use/#throughkiosk) - _For DGAM Only_:**
    1. **Scan QR Code -** Scan the missing device's QR Code displyed on the kiosk.
    2. **Search Device -** Use the proximity meter and audio alerts to find the device.
- **Nearby Lost Device Detection -** Receive automatic notifications when a [lost device is detected nearby.](../use/#lostdevicenearby) User can initiate the recovery process directly from the alert.
- **Locate Lost Bluetooth Accessories -** Use the [finding process](../use/#bluetoothscanners) to locate lost Zebra Bluetooth scanners.
- **Map Device Location -** View GPS coordinates and mapped locations of devices. This feature is accessible to [managers](../dashboard/#mobiledevices-1) and [admins](../dashboard/#mobiledevices) via the web portal.
- **Track Powered-Off or Critically Low Battery Devices -** Use [secondary BLE](../secondaryble) to [find devices](../use/#secondaryble) that are powered off or have critically low battery.

### Administration

- **Web Portal Dashboard -** [Admins](../dashboard/#administratordashboard) and [managers](../dashboard/#managerdashboard) can access the web portal dashboard, offering a centralized to view of devices, APs and sites across the organization.
- **Productivity Dashboard -** [Graphically displays trends](../dashboard/#productivity) of lost and found devices, as well as average lost duration and recovery time, at both company-wide and individual site levels. This is feature is accessible only to admins.
- **DGAM Dashboard - _For DGAM Only:_** Offers a centralized view of devices managed by kiosks. 
- **Daily Summary Report -** The [End of Day Device Summary](../dashboard/#endofdaydevicesummary) provides a snapshot of device counts by state (e.g., To Be Found, Found, Low Battery) at that moment, aiding daily accountability. Reports can be emailed to designiated recipients with options for scheduled report time, report criteria, and attachments.
- **Events By Site Report -** [Reports event changes](../dashboard/#eventsbysite) within a site based on device state transitions (e.g., Disconnected, Found, Low Battery). Administrators can view data across all sites, while managers can view data for their assigned site.
