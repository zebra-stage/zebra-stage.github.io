Migration from ZAMS

Device Guardian & Access Management 2.2

All references to Device Guardian within this guide also apply to Device Guardian Access Management, unless explicitly stated otherwise.

Overview

This guide outlines the steps to upgrade, configure, and use Device Guardian (DG) and Device Guardian Access Management (DGAM) when migrating from Zebra Access Management System (ZAMS).

Zebra Guardian Portfolio

Zebra Guardian offers a streamlined experience at every stage of device use for frontline workers, starting with secure access and ending with safe storage and readiness for the next shift. These four integrated solutions work seamlessly together to minimize lost devices, maximize productivity, and ensure devices are always available and secure:

  • Identity Guardian: Device Access - Securely access devices with personalized credentials, including facial biometrics and single sign-on.
  • Device Guardian: Find/Prevent Lost Devices - Quickly find misplaced devices and proactively prevent loss, protecting both your investment and productivity.
  • Device Guardian Access Management: Device Status - Monitor device status and availability in real time, ensuring devices are always charged and ready for use.
  • Guardian Cabinets: Device Storage and Readiness - Smart storage solutions control device access and automate readiness for any shift from start to finish.

Requirements

All system, device, network, web portal, and license requirements must be met as detailed in the main Requirements guide.

Hardware Compatibility: The device must be compatible with the following software:

  • Identity Guardian
  • Device Guardian and Device Guardian Access Management
  • Zebra DNA (if using Zebra DNA as your EMM)

The administrator performing this setup should be proficient with deploying Managed Configurations in your EMM. Refer to the Skills Matrix.

Migration Stages

This guide outlines the stages required to migrate from ZAMS to DGAM, involving coordinated actions from both the Customer and Zebra.

  • For complete details, refer to the full Migration User Guide.
  • After completing these migration steps, proceed to the Quick Start guide for DGAM deployment and configuration.

I. Onboarding

Role Action
Customer Purchases or renews the DGAM license.
Zebra Provisions a new DGAM cloud instance. Sends an onboarding email containing the portal URL, login credentials, APK download link, and user guide.
Outcome Customer is onboarded and has everything needed to begin the migration.

II. Migration Planning

Role Action
Customer Aligns with Zebra on a migration time window. Prepares the EMM for device re-enrollment.
Zebra Confirms the migration slot. Prepares ZAMS data for migration.
Outcome The migration plan is finalized to ensure minimal business disruption.

III. Portal Data Migration

Role Action
Zebra Migrates Sites and Admin Users data from ZAMS to the new DGAM instance.
Customer Logs into the DGAM portal using the onboarding credentials and reviews the migrated data to confirm accuracy.
Outcome The customer's ZAMS data is successfully migrated and available in DGAM.

IV. ZAMS Application Uninstall

This stage prepares devices for the new DGAM installation by removing the old ZAMS application. Choose the path that matches your deployment method.

Path Method & Instructions
Path A:
Using StageNow		 Scan the appropriate StageNow barcode:
   • Below Android 13 devices: Scan Uninstall_ZAMS_Device
   • For Android 13 & above devices: Scan A13_Uninstall_ZAMS_Device
Path B:
Using EMM		 Run the ZamsDevice_Uninstall.xml script via your EMM.
Outcome The ZAMS application is removed from the devices, preparing them for DGAM.
