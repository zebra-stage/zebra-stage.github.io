Common Criteria

Common Criteria

Overview

The following is a generalized list of the requirements and procedures to bring a Zebra device into Common Criteria compliance.

For detailed procedures, please refer to the Zebra Administrative Guidance Documentation (.pdf).

Prerequisites

The following are required to bring a Zebra device into CC state:

  • One or more supported devices running Android 10 and with support for Federal Information Processing Standard (FIPS)
  • Windows computer with Zebra StageNow 4.2 (or later) installed (Download now).
  • LifeGuard patch for CC compliance
  • FDE to FBE conversion patch

Supported Devices

The following Zebra devices running Android 10 (kernel 4.4.205) have been brought to a CC-compliant state by Zebra engineers:

  • MC9300
  • TC52
  • TC57

Basic Procedure

Before Beginning

Target device(s) must be running the latest Android 10 build. More about LifeGuard updates.

To bring a Zebra device into CC compliance:

1. Create a StageNow Profile to:

  • Apply the LifeGuard patch for CC compliance
  • Apply the FDE-to-FBE patch to migrate device from full-disk to file-based encryption

2. Generate and scan barcodes to execute Profiles and apply patches on device.
3. Create a second StageNow Profile to set device according to CC specifications:

  • Disable external storage
  • Enable Trusted Staging
  • Protect above settings from changes
  • Download and install Device Policy Controller on device
  • Launch DPC app and enroll as Device Owner

4. Disable "add new user" function in DPC app.
5. Configure additional device settings:

  • Require lock screen
  • Disable "Smart Lock"
  • Disable debugging features
  • Disable "app installation from unknown sources"
  • Enable and route all network traffic through a VPN
  • Enable audit logging

Device is CC complaint when all steps above are completed successfully.

For detailed procedures, please refer to the Zebra Administrative Guidance Documentation (.pdf).
