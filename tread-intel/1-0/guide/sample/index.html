Sample App

Zebra Tread Intel 1.0

Overview

This sample application is only intended for educational purposes, demonstrating the use of the Tread Intel Intent APIs.

This sample app demonstrates the use of Zebra Tread Intel Intent APIs to measure tire tread depths. Developed using a Java environment, it serves as a guide for developing your own application.

The Tread Intel API is built on the Android intent architecture, minimizing library dependencies and integration time for app developers. For more information, refer to the API reference guide.

Requirements

Prerequisites:

  • The Zebra Tread Intel Snap-On Reader must be attached to the supported mobile computer, such as the Zebra TC7x or TC5x series devices. For supported devices, see the Zebra Support Portal for the supported devices list.
  • The Zebra Tread Intel Android Framework must be installed on the mobile computer.

The following requirements must be met:

  • Minimum Android SDK version: The minimum supported version is 30.

  • Grant required permissions: Make sure the MANAGE_EXTERNAL_STORAGE permission is enabled to allow full access to external storage.

  • Add Queries Tag: For devices running Android 13 and higher, add the <queries> tag in the AndroidManifest.xml of your Android application project. This tag defines the package name for Zebra Tread Intel and is required to allow the Zebra Tread Intel API to communicate with your application.

    <queries>
    <package android:name="com.zebra.intentapis" />
</queries>

  • Add Constants: Create a Java class named IntentAPIConstantUtils within your Android application project and add the code below. This helps manage and maintain consistent values throughout the project.

Usage

This sample app enables users to measure tire tread depth and operates in two modes: demo mode and test mode. Build and deploy the app to the device, then follow the instructions below.

Demo Mode

Demo mode is designed to showcase the app's ability to read tire tread depth. In this mode, data is not saved, and there are no options to enter a Tester ID, Tire ID, or enter plunger measurements.

  1. By default, the app opens in demo mode, with the Demo tab highlighted in the bottom menu. The initial main screen shows the Measure button as deactivated.
  2. When the snap-on reader is attached to the device, it automatically connects, activating the Measure button. If it remains inactive, ensure the snap-on reader is properly attached to the device.
  3. Tap the Measure button and swipe the device across the tire from edge to edge. When the read is successful, measurements are displayed along with their corresponding confidence levels. If the confidence level below 60, a prompt will appear to re-swipe.

Test Mode

Test mode is intended to facilitate the evaluation of the solution by saving data from each read session, including the tester ID, tire ID, and plunger measurements. The plunger adjustments enable the entry of manual measurements using the plunger, allowing for comparison with reader measurements to help assess accuracy. The Tire ID resets after each measurement. Although the Confidence Level is not displayed visually, it is recorded in the log.

  1. Tap the Test icon in the bottom menu. When the app detects the snap-on attachment, the Measure button is activated.

  2. Enter the Tester ID and Tire ID. Adjust plunger values for the inner, center, and outer tire regions as needed. Enter these values based on manual measurements (using a plunger) for comparison with the reader's results.

  3. Tap the Measure button and swipe the device across the tire from edge to edge. When the read is successful, measurements are displayed, and the data is saved in the internal storage directory. If the confidence level is below 60, a prompt appears to re-swipe. The following data is stored in the internal storage directory (e.g. /storage/emulated/0/Download/):

    • TreadReaderLogFile.csv - Contains cumulative data of all reader activity, includes data when readers are switched.
    • Ttinput.png - Contains data that may be requested by Zebra engineers for evaluation if further analysis is needed.
    • Folder in format of serialno_testerID_tireID - Contains data that may be requested by Zebra engineers for evaluation if further analysis is needed.

Versions

The Versions tab in the app displays the software component versions of the reader, as well as the reader serial number and the scan count, which is the total number of scans performed by the reader.
