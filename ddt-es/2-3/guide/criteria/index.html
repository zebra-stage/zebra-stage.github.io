Criterios de pruebas

Device Diagnostic Tool 2.3

Aspectos generales

Device Diagnostic Tool comprueba el funcionamiento de las funciones de hardware del dispositivo según criterios específicos, con los resultados siguientes:

  • Pass (Superada) - si se cumplen los criterios
  • Failed (No superada) - si no se cumplen los criterios
  • Information (Información) - muestra datos informativos (no se aplica superación/no superación)

  - Volume down (Bajar volumen)		 Se supera cuando se detecta la pulsación del botón de cada uno de los botones.

Audio Sí Grabar y reproducir el archivo Se requiere intervención manual del usuario. El usuario graba manualmente el audio y luego escucha la grabación. La superación/no superación de la prueba queda a la discreción del usuario. En esta prueba se deben usar los altavoces del dispositivo - no se acepta el uso de auriculares. Esta prueba debe llevarse a cabo en un entorno con el menor ruido de fondo posible para que la prueba sea correcta. Ajuste el dispositivo al volumen máximo durante la reproducción para obtener los mejores resultados.
Nota: Esta prueba no se puede ejecutar si el dispositivo no tiene micrófono, p. ej., MC18.


El texto de color rojo indica información confidencial que no se muestra. En su lugar, se muestra “valid” (válido)o “invalid” (no válido) para indicar si la información se detecta en el formato adecuado. “Valid” (Válido) significa que se recuperó el valor adecuado. “Invalid” (No válido) indica que el valor recuperado es nulo, está vacío o no coincide con el patrón de dirección MAC esperado (solo se aplica a la dirección MAC).

(i) indica datos informativos que se consideran datos adicionales, que se muestran si show_extra_data se establece en “true” en el archivo configuration.xml.
