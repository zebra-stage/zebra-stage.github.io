CIM Service Provider for Linux

Overview

CIM Service Provider is the UPOS systems management interface for Zebra scanners for Linux.

Latest version of CIM Service Provider supports the following Linux distributions:

  • SUSE Linux Enterprise Server 15 SP3
  • SUSE Linux Enterprise Server 12 SP5

Dependencies

Following dependencies are required to execute CIM Service Provider on host environment.

  • sfcb
  • cmpi base library
  • cmpi common library
  • wbemcli - for deployment verification only
  • zebra-scanner-corescanner (v4.4.1-19 or higher) - https://www.zebra.com/us/en/support-downloads/software/developer-tools/scanner-sdk-for-linux.html
  • zebra-scanner-devel (v4.4.1-19 or higher) - https://www.zebra.com/us/en/support-downloads/software/developer-tools/scanner-sdk-for-linux.html
  • zebra-scanner-javapos (v4.4.1-25 or higher) - https://www.zebra.com/us/en/support-downloads/software/developer-tools/scanner-sdk-for-linux.html

Deployment of the CIM Service Provider

Installation Instructions

  1. Extract "Zebra_CimServiceProvider_vX.XX_C98.tar.gz" or "Zebra_CimServiceProvider_vX.XX_C11.tar.gz" as per the selected Linux distribution.

  2. Execute script file "zebra_service_provider_installer.sh" with super-user permission.

    NOTE: If the message "No init script found - you need to start sfcbd manually." is printed at the end of execution, please start the sfcb service manually.

Uninstallation Instructions

  1. Execute script file "zebra_service_provider_uninstaller.sh" with super-user permission.

Verification

  1. Following command can be used to enumerate an instance. The class name is "UPOS_BarcodeScanner".

wbemcli ei http://localhost:5988/root/cimv2:UPOS_BarcodeScanner
