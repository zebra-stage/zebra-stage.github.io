Getting Started

Overview

This section provides step-by-step instructions on developing jpos application from the Zebra JPOS driver package with IntelliJ IDEA.

Note: The demo app in this guide is intended for tutorial purposes only and should not be used in production environments.

JPOS driver package includes following libs

  • Lib : Service Class Library and other JPOS dependent libraries.
  • jpos-rfidscanner-test.jar : Contains sample app to demonstarte.
  • Jpos.xml :Jpos configuration file contain the logical device name.
  • javaPosTest.bat : Batch file to run the sample app.
  • jpos_trace.properties where user can config the log file configuaration like logFilePath,logFileName etc. and jpos_trace.properties should be located inside the Lib folder.

Creating Project using IntelliJ IDEA

This tutorial will aid you creating a new application project using IntelliJ IDEA.

Creating the Project

Open IntelliJ desktop application , if you have a previous project open , close the project by clicking Close Project from the File menu.

  1. Select Start a new IntelliJ project under Quick Start section.

  2. In the new project wizard, you need to assign an Application Name, specify the location, select the language as Java, and choose a build system depending on the project type. Here, we are selecting IntelliJ as the build system and then selecting the JDK. After that, click the ‘Create’ button to proceed.

    3. Adding the Zebra JPOS service lib and other JPOS dependent Libraries.

  3. Copy the lib folder from the Zebra jpos package and add into the project where src folder is located and add jpos.xml file in the same directory.

  4. Importing the libraries in the project. Select Project Structure from the File menu. In the Project Structure window select Libraries click on + it will show the New project Library select Java and add the location of the lib folder which was added. after adding the path click on Apply and OK.

    5. Your project is now ready to use the JPOS API's.

Configuring the jpos.xml file

In the jpos.xml file user can configure the com port of the device and antenna power of the reader.

  • Set the COM Port in jpos.xml, if discovery is not required.
    • 
    <!-- if this property(comPort) is not set com port will be discovered -->
    <prop name="comPort" value="COM4"/>
  • Configure the antenna power, if not antenna power will set to default 27dBm.
    • 
    <!-- if antennaPower is not set, it will set to default 27dBm -->
    <prop name="antennaPower" value="200"/>
