About SmartDocs


The "SmartDocs" bar can customize this page to show only the features present on a particular Zebra device.

  1. Use drop-downs to select the OSX, MX and/or Android version(s) on the device. Unsupported features will turn gray based on selection(s).
  2. With at least one version selected when viewing a CSP (i.e. "App Manager"), click forward and back buttons to scroll to the next and previous supported feature of that CSP (not supported on Compatibility matrix).
  3. To clear all selections press the SmartDocs button.

OSX, MX and Android version information for a device can be found in the Android Settings panel or by querying the device through ADB, EMDK or the MX CSP. More info

  • The OSX layer provides extensions to the base operating system to implement functionality not offered by Android. The root OSX version number always matches the root number of the Android version that it extends. More info
  • The MX Management System provides a uniform interface into privileged and unprivileged Android APIs on the device. Some features require a specific pairing of MX and OSX versions. More info
  • The Android version is sometimes enough to determine a feature set. Others require a specific MX version plus a minimum version of Android.

Overview

The Bug Report Manager (BugReportMgr) provides a framework for reporting the time and relevant technical data associated with perceived issues on Zebra mobile computers. Bug reports can be triggered programmatically through an intent or manually by an administrator or device user using the Power Key menu. Reports can be stored in internal memory or a removable memory card, sent to an e-mail address or uploaded to the Zebra Cloud for further analysis.

Main Functionality

  • Bug reporting Enable/Disable
  • Report generation from Power Key menu Enable/Disable
  • Allow device user to provide additional information
  • Store reports in internal (public) flash memory
  • Store reports in removable flash memory
  • Store reports in private area of flash memory
  • Send reports to a specified email address
  • Upload reports to the Zebra Cloud
  • Capture audio with bug reports
  • Trigger bug reports with an intent (or key-presses on MC40)

Bug Report Handling

Used to configure the handling of bug reports on the device. Default handling includes the following parameters:

  • Storage - internal flash memory, local path /storage/sdcard0/BugReports/
  • Email

Parm Name: BugReportProfile

Option Name Description Requires
0 Do nothing This value (or the absence of this parm from the XML) will cause no change to Bug Report Manager settings; any previously selected setting will be retained.

MX: 6.3+

Android API Level: 23+

1 old Sets Bug Report Manager to use default handling of bug reports.

MX: 6.3+

Android API Level: 23+

2 new Sets Bug Report Manager to use custom handling of bug reports.

MX: 6.3+

Android API Level: 23+

Dialog Action

Option Name Description Requires

MX: 6.3+

Android API Level: 23+

MX: 6.3+

Android API Level: 23+

Android API Level: 23+

MX: 6.3+

Android API Level: 23+

6 AddVoiceRecordButton Displays a button that the user can press to record a message to accompany the bug report.

MX: 6.3+

Android API Level: 23+

7 LayoutEnd Displays a message that the questionnaire is finished.

MX: 6.3+

Dialog Timeout Enable/Disable

Used to control whether the dialog for collecting supplemental bug report information will be dismissed automatically or displayed indefinately until filled out or dismissed by the user. If the timeout parameter is used, the dialog will be automatically dismissed when the interval specified in the Dialog Timeout parameter has passed (default = 60 seconds).

Parm Name: SetDialogTimeout

Option Name Description Requires
0 Do nothing This value (or the absence of this parm from the XML) will cause no change to Bug Report Manager settings; any previously selected setting will be retained.

MX: 6.3+

Android API Level: 23+

1 false Causes the user-data collection dialog to be displayed indefinately until filled out or dismissed by the user.

MX: 6.3+

Android API Level: 23+

2 true Causes the user-data collection dialog to be dismissed automatically if not filled out or dismissed by the user.

MX: 6.3+

Android API Level: 23+

Set Dialog Timeout

Used to specify the timeout interval (in ms) for the supplemental information collection dialog, after which the dialog will be automatically dismissed and the report sent. Default is 60,000 ms (60 seconds), which is reset when the user touches the dialog.

Parm value input rules:

  • String from 1 - characters containing the timeout interval (in ms)

Shown if: SetDialogTimeout parameter is set to "True"

Parm Name: DialogTimeout

Requires:

  • MX: 6.3+
  • Android API Level: 23+

Panic Button Enable/Disable

Used to specify whether to display a "Panic Button" in Android's "Power Off" Menu, which appears after long-pressing the power key on the device. Bug reports generated using this feature will be stored and/or sent using the handling preferences specified in Bug Report Profile parameter.

Note: The Power Off Menu is controlled by the Power Key Manager.

Parm Name: EnableBRInPwrMenu

Option Name Description Requires
0 Do nothing This value (or the absence of this parm from the XML) will cause no change to Bug Report Manager settings; any previously selected setting will be retained.

MX: 6.3+

Android API Level: 23+

1 true Displays a "Panic Button" on the Power Key menu and enables a bug report to be generated by pressing it.

MX: 6.3+

Android API Level: 23+

2 false Hides the "Panic Button" from the Power Key menu.

MX: 6.3+

Android API Level: 23+

Storage Enable/Disable

Used to specify whether to store bug reports in device flash memory. If enabled, bug reports will be stored in the location specified in the BugReportFilePath parameter.

Shown if: BugReportProfile parameter is set to "New"

MX: 6.3+

Android API Level: 23+

MX: 6.3+

Android API Level: 23+

2 true Enables bug reports to be stored in device memory.

MX: 6.3+

Android API Level: 23+

Examples

Install An Application


<!-- Silently install Clock.apk from the /enterprise/usr/persist folder -->
<wap-provisioningdoc>
    <characteristic version="4.2" type="AppMgr">
        <parm name="Action" value="Install" />
        <parm name="APK" value="/enterprise/usr/persist/Clock.apk" />
    </characteristic>
</wap-provisioningdoc>
