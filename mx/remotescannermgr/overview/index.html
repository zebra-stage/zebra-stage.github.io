The Remote Scanner Manager (RemoteScannerMgr, or RSM) is used to monitor, configure and apply firmware updates to supported Zebra wired and wireless scanners from a Zebra device running Android 5.1 (Lollipop) or later.

Main Functionality

  • Update the firmware of a connected scanner
  • Perform the following functions on a paired scanner:
    • Apply a Scan123 config file (downloaded separately; learn more)
    • Cycle the power on the scanner
    • Initiate a device-location LED/sound sequence
    • Disconnect the scanner
    • Unpair the scanner
  • Enter the serial number of a device on which to perform an action

Supported Devices

Remote Scanner Manager supports most wired scanners plus the following Bluetooth scanners:

  • CS6080
  • DS2278
  • DS3678*
  • DS8178
  • LI3678
  • RS507
  • RS5000
  • RS5100
  • RS6000
  • RS6100

Scanners in BOLD are supported only with SDM660-platform devices in SSI mode running Android 11 or later. See Zebra devices by platform.

* Support requires MX 6.3 (or later) on the host device.

Programmatic Usage

The features of RSM can be accessed programmatically, enabling firmware updates to be built into a new or existing Android app running on a mobile computing device connected to a scanner. The functions required of the app are listed below, followed by a code snippet with examples of some functions.

  • Firmware update prep steps:
    • Download firmware from Zebra Support Portal to admin host
    • Create a config file with 123Scan Utility
  • Required functions for updating firmware:
    • Push the firmware and required configuration file(s) to the host device using FileMgr.
    • Create a 123Scan config file using 123Scan.
    • Push the 123Scan config file to the scanner using RemoteScannerMgr.
    • Display a “Start Update” button to initiate firmware update using RemoteScannerMgr.
    • User presses the "Start Update" button to begin the scanner update.
  • Optional functions for before, during and/or after firmware update:
    • Get the current firmware version, serial number, model number, 123Scan config version/name from the connected scanner using OEMInfo.†
    • Get the battery level of the scanner using OEMInfo.†
    • Publish progress of update/configuration progress and/or pass/fail status using OEMInfo,† if desired.

Notes

  • Scanner must remain connected to the host while the update and configuration are in progress.
  • Scanner cannot be used for scanning until the process is completed.
  • Remove the "Start Update" button from view if the scanner is disconnected before the update begins.
  • If the list of parameters list changes in the new firmware, all parameters are reset to their default values.
  • Firmware of some early Zebra Ring Scanners erased the host MAC address during firmware update, resulting in a failure to reconnect after reboot. This issue has been addressed on devices running Android 11 and later.
  • Device Central must be installed if using OEMInfo.
  • If DataWedge is used for scanning, its parameters overwrite those of 123Scan whenever scanner is paired.

† OEMInfo requires the Zebra Device Central app on the device.

Sample code is for demonstration purposes only. DO NOT USE in production environments.
