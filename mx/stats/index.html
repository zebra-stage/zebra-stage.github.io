About SmartDocs

This document contains "SmartDocs" capabilities. This will allow you to easily see only the content that is relevant to the Zebra device you are working with.

MX Features will be available on a device based on a combination of factors:

  • MX Version: The version of MX on the device can be retrieved by following these instructions.
  • OSX Version: The version of OSX on the device can be retrieved by following these instructions. Most features require a combination of OSX and MX in order to operate.
  • Android Version: Some features only require a specific version of MX and a minimum version of Android. The version of Android on the device can be retrieved by checking the system settings application.

After identifying the version information for your device, simply select them in the SmartDocs bar. Any section that does not apply to your device will be greyed out. You can clear all settings by hitting the SmartDocs button. Clicking the back and next buttons on the SmartDocs bar will navigate to the sections that are relevant to your device.

Overview

Stats controls whether the device will collect statistics on XML submitted to the MX Management Framework (MXMF), and can set the name and location of the statistics file, change its reporting interval, and clear saved data. Collected statistics are reported to the Analytics Manager for processing and distribution.

The MXMF works through Configuration Service Providers (CSPs), which implement and control Android capabilities by submitting to the framework XML documents containing the desired device settings and behaviors. Stats controls whether these XML submissions are tracked, where they're stored on the device and the minimum wait time between reports.

Main Functionality

  • Enable/Disable usage statistics
  • Set the Reporting Interval minimum wait time
  • Set the name and location of the statistics file
  • Clear data saved in the statistics file

Statistics Enable/Disable

This parm controls whether to collect statistics about XML submissions to the MX Management Framework. The collection file can contain a maximum of 10,000 XML nodes.

Parm Name: EnableStats

Option Name Description Requires
true Enable Enables the device to collect statistics on XML submitted to the MXMF.

MX: 5.1+

false Disable Disables the collection of statistics on XML submitted to the MXMF.

MX: 5.1+

Set Statistics File Name

Use this parm to specify the name of the file for storage of MX usage statistics. Max length: 255 characters. If no file name is specified, stats will be stored using the name mxStats.xml.

Shown if: Statistics gathering is enabled (EnableStats = True)

Parm Name: StatsFileName

Requires:

  • MX: 5.1+

Set Path for Statistics File Storage

Specify the local (on-device) path of the file for storage of MX usage statistics. Max length: 255 characters. If no path is specified, the default path is /storage/sdcard0/Stats/.

Shown if: Statistics gathering is enabled (EnableStats = True)

Parm Name: SetStatsFilePath

Requires:

  • MX: 5.1+

Display the Path of the Statistics File

Displays the path of the stored MX usage statistics file. If no custom path was specified, the default path is /storage/sdcard0/Stats/.

Shown if: Statistics gathering is enabled and a path was specified

Parm Name: StatsFilePath

Requires:

  • MX: 5.1+

Clear Statistics File

Use this parm to clear the statistics file on the device. If a custom name and/or path was specified, the stats file of that name/path will be cleared. Otherwise, the default file myStats.XML in the default path /storage/sdcard0/Stats/ will be cleared.

Parm Name: ClearStats

Option Name Description Requires
0 Do not change This value (or the absence of this parm from the XML) will cause no change; the statistics file will not be cleared.

MX: 5.1+

true Clear Stats file This will clear the Statistics file.

MX: 5.1+

Reporting Interval

Use this parm to set the minimum wait time (in minutes) after which the statistics file will be reported to Analytics Manager for processing and distribution. Data will be sent only when a new activity has occurred after the Report Interval has been satisfied. For example, if the Report Interval is 1440 (24 hours) and a new activity occurs 10 hours after the last report was sent, the new activity will not be reported until another new activity occurs at least 14 later. However, if new activity occurs 25 hours after the last report, that activity will be reported immediately.

The default Report Interval value is 2880 (48 hours). A setting of zero (0) disables reporting but stat collections will continue until the cap of 10,000 XML nodes has been reached.

Parm value rules
  • Accepts a numeric String representing wait time (in minutes) between reports
  • Minimum length is 1 character; maximum length is 255 characters
  • Default value is 2880 (48 hours)
  • A setting of zero (0) disables reporting (but collection continues to 10,000 node cap)
  • Submission of non-numeric characters will place an error in the Return XML

Shown if: Statistics gathering is Enabled (EnableStats = True)

Parm Name: ReportInterval

Requires:

  • MX: 5.1+

Examples

Configure Statistics Gathering

Input

Standard Configuration (showing all default values)
    
    <wap-provisioningdoc>
        <characteristic type="Stats" version="4.4" >
            <parm name="EnableStats" value="True"/>
            <parm name="StatsFileName" value="mxStats.xml"/>
            <parm name="SetStatsFilePath" value="1"/>
            <parm name="StatsFilePath" value="/storage/sdcard0/Stats/"/>
            <parm name="ReportInterval" value="2880"/>
        </characteristic>
    </wap-provisioningdoc>

If the operation was successful, the Result XML (output) will be identical to the Input.

Input

Clear the Stats File
    <wap-provisioningdoc>
        <characteristic type="Stats" version="4.4" >
            <parm name="EnableStats" value="True"/>
            <parm name="StatsFileName" value="mxStats.xml"/>
            <parm name="ClearStats" value="True"/>
            <parm name="ReportInterval" value="2880"/>
        </characteristic>
    </wap-provisioningdoc>

If the operation was successful, the Result XML (output) will be identical to the Input.
