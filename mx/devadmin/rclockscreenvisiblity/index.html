Used to select whether the Android Lock Screen is visible on the remote console if the Lock Screen appears on a device while being remotely controlled. The remote console is the rendering of the device being controlled as seen on the DNA Cloud console or that of a third-party EMM system.

Platform Note

This feature is supported ONLY on the following platforms and requires the following Android- and platform-specific OSX versions.

  • Android 14, 15:
    • Supported on all devices and platforms
  • Android 13:
    • Platform - Minimum OSX ver:
    • 4490 - QCT4490.130.13.15.4
    • 6375 - QCT6375.130.13.22.15
    • 6490 - QCT6490.130.13.20.16
    • SDM660 - SDM660.130.13.20.18
  • Android 11:
    • Platform - Minimum OSX ver:
    • 6375 - SD6375.110.30.41
    • 6490 - QCT6490.110.11.14.32
    • SDM660 - QCT.110.11.27.46

See Zebra device models by platform

Behavior Notes

Proper lock screen operation during remote-control sessions relies on the conditions listed below.

  • The remote client app must support Android Direct Boot mode, and be designed to run before the device is unlocked after any reboot.
  • Zebra Key Touch Injection (KTI) 15.0 (or later) is required, which also supports Direct Boot.
  • Lock-screen visibility must be set to "Allow" (Option 1, below).
