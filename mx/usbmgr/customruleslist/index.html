Used to enter one or more custom rules by which to control USB peripheral usage when the device is in USB Host Mode. Allowed peripherals can be categorized by device class (sometimes referred to as device "type"), vendor ID (VID) and/or product ID (PID). When a device is in USB Host Mode, a given USB peripheral is usable only if there is at least one rule that allows it, and unusable if there are NO rules that allow it.

More than one rule can apply to the same peripheral. For example, if a rule allowing all printers existed with a rule to allow all devices from Zebra (VID = 0A5F), Zebra printers would remain usable if either of those rules were deleted.

All connected USB peripherals NOT explicitly allowed are rendered unusable by apps. Such peripherals continue to receive power (via USB) and in some cases could appear to be enabled, but are effectively disabled from communicating with apps.

Example XML File

To use the Custom Rules List parameter, create a file using the format below to specify the device class(es), vendor(s) and/or product(s) to be allowed. Then paste the file's XML-formatted contents into the Custom Rules List field in the relevant StageNow or EMM device-configuration Profile. 


<?xml version="1.0" encoding="UTF-8"?>
<usbconfig mode="whitelist">
  <class>AINPUT_SOURCE_DPAD</class>
  <class>AINPUT_SOURCE_JOYSTICK</class>
  <class>USB_CLASS_AUDIO</class>
    <vid vendorid="1504">
    <pid>5889</pid>
    </vid> 
</usbconfig>
