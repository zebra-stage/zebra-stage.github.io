Verwendung von Device Diagnostic Tool

Device Diagnostic Tool 2.3

Überblick

Device Diagnostic Tool unterstützt zwei Betriebsmodi:

  1. Administratormodus – Administratoren können auf App-Einstellungen zugreifen, Testkonfigurationen ändern und einzelne Tests ausführen. Dies ist der Standardmodus.
  2. Benutzermodus – Benutzer können nur Tests ausführen. Der Zugriff auf App-Einstellungen oder Testkonfigurationen sowie die Auswahl einzelner Tests ist ihnen nicht möglich. Zum Aktivieren des Benutzermodus setzen Sie den Parameter admin_mode auf „false“ (falsch) (siehe Abschnitt Konfiguration).

Starten der App

Es gibt verschiedene Möglichkeiten zum Starten von Device Diagnostic Tool:

  1. Manuell über die Benutzeroberfläche – der Benutzer startet Device Diagnostic Tool manuell und führt dann die Tests aus. Siehe Abschnitt Benutzeroberfläche unten.
  2. Per Fernzugriff über ein EMM-System (Enterprise Mobility Management) – über folgenden Befehl wird Device Diagnostic Tool im Hintergrund gestartet und die Protokollfunktion aktiviert:
    adb shell am broadcast -n "com.symbol.selfdiagnostics/com.symbol.selfdiagnostics.SESReceiver"

Es darf nur eine der genannten Methoden zum Starten von Device Diagnostic Tool verwendet werden. Andernfalls kann ein unerwartetes Verhalten der App die Folge sein.

Benutzeroberfläche

Beim ersten Starten der App werden auf der Hauptseite die verfügbaren Tests aufgelistet: Hauptseite
Wenn bereits Tests ausgeführt wurden, ist neben dem Namen des Tests das Datum der letzten Ausführung angegeben.

Tippen Sie auf der Hauptseite auf Run Tests (Tests ausführen), um alle Tests auszuführen. Welche Tests ausgeführt werden, richtet sich nach der Auswahl auf der Seite Configure Tests (Tests konfigurieren). Bei den folgenden Tests wird der Benutzer aufgefordert, zusätzliche Aktionen auszuführen: Scanner Test (Scannertest), Button Test (Tastentest), Touch Screen Test (Touchscreen-Test) und Audio Test (Audio-Test). Es wird empfohlen, sämtliche Tests im normalen Gerätebetrieb auszuführen, und nicht, während es sich in einer Dockingstation befindet oder über ein USB-Kabel mit einem Computer verbunden ist. Nach Abschluss der Tests werden die Ergebnisse angezeigt:

   Pass (Bestanden) – der Test wurde ausgeführt und das Ergebnis entsprach den Testkriterien.
   Fail (Fehlgeschlagen) – der Test wurde ausgeführt und das Ergebnis entsprach nicht den Testkriterien.
   Information (Informationen) – Daten wurden abgerufen und werden angezeigt.

Im Administratormodus ist es auch möglich, einzelne Testkategorien auszuwählen und dann auf Run Tests (Tests ausführen) zu tippen, um nur die ausgewählten Tests auszuführen.
Bei Test failed (Test fehlgeschlagen) und Test timed-out (Test-Timeout) wird das gleiche rote Symbol angezeigt.

Die Option „Help“ (Hilfe) kann auf der Hauptseite über das Menü oben rechts aufgerufen werden. Sie öffnet das Support-Portal für Device Diagnostic Tool.

Scanner Test (Scannertest)

Der Benutzer wird aufgefordert, einen Barcode zu scannen. Ergebnisse:

  • Scanner Test (Scannertest) – Anzeige der Barcode-Daten
  • Label Type (Etikettentyp) – Anzeige des gescannten Barcode-Typs oder Decoders

Hinweis: Bei der Ausführung des Scannertests auf dem TC55 muss der Scan-Auslöser länger gedrückt gehalten werden, damit das Auslösen erkannt wird.


Button Test (Tastentest)

Der Benutzer wird aufgefordert, die physischen Tasten am Gerät zu drücken: Scan-Auslöser (links oder rechts), Push-to-Talk, Lautstärke (lauter/leiser). Ergebnisse:

  • Button Test (Tastentest) – test successful (Test erfolgreich), test failed (Test fehlgeschlagen) oder test timed-out (Test-Timeout)

Hinweis: Wenn die Funktionszuweisung für Tasten geändert wurde, kann es beim Tastentest zu unerwartetem Verhalten kommen.


Touch Screen Test (Touchscreen-Test)

Der Benutzer wird aufgefordert, jede Rasterzelle auf dem Bildschirm zu berühren.

  • Touch Screen Test (Touchscreen-Test) – test successful (Test erfolgreich), test failed (Test fehlgeschlagen) oder test timed-out (Test-Timeout)

Bluetooth Tests (Bluetooth-Tests)

Überprüfung der Funktionsfähigkeit des Bluetooth-Funkmoduls mit Bereitstellung von relevanten Bluetooth-Informationen Ergebnisse:

  • Name – Anzeige des Bluetooth-Namens
  • Radio Power Cycle (Funkmodul-Einschalttest) – test successful (Test erfolgreich) oder test failed (Test fehlgeschlagen). Der Status des Funkmoduls vor diesem Test wird beibehalten.
  • Functional/Non-functional (Funktionsfähig/Nicht funktionsfähig) functional (funktionsfähig) oder non-functional (nicht funktionsfähig)
  • Discoverable/Connectable (Sichtbar/Verbindung möglich) – connectable (Verbindung möglich), discoverable (sichtbar) oder none (keine)

Wifi Tests (WLAN-Tests)

Überprüfung der Funktionsfähigkeit des WLAN-Funkmoduls mit Bereitstellung von relevanten WLAN-Informationen Ergebnisse:

  • MAC Address (MAC-Adresse) – valid (gültig) oder invalid (ungültig)
  • Network Test (Netzwerktest) – connected (verbunden) oder not connected (nicht verbunden); der Test schlägt fehl, wenn keine Netzwerkverbindung besteht. Der Status des Funkmoduls vor diesem Test wird beibehalten. Wenn der Test erfolgreich ist, werden die folgenden Werte angezeigt:
  • strength (Stärke) – Anzeige der Signalstärke
  • essid – Anzeige der ESSID. Ab Android O muss aufgrund von Android-Einschränkungen in den Geräteeinstellungen der Service Location (Standort) aktiviert sein. Ist der Service Location (Standort) nicht aktiviert, wird der Benutzer aufgefordert, ihn zu aktivieren. Beim Starten des Tests ohne aktivierten Service Location (Standort) wird für ESSID „Location not enabled“ (Standort nicht aktiviert) ausgegeben.
  • ip – Anzeige der IP-Adresse
  • bssid – valid (gültig) oder invalid (ungültig)
  • speed (Geschwindigkeit) – Anzeige der Verbindungsgeschwindigkeit

Battery Tests (Akkutests)

Überprüfung des Akkustatus mit Bereitstellung von relevanten Akkuinformationen Ergebnisse:

  • Part Number (Artikelnummer) – Anzeige der Artikelnummer
  • Serial Number (Seriennummer) – Anzeige der Seriennummer
  • Manufacture Date (Herstellungsdatum) – Anzeige des Herstellungsdatums des Akkus
  • Decommission Status (Außerbetriebnahme-Status) – Anzeige des Akkustatus:
    • Good (Gut) – der Zustand des Akkus ist gut.
    • Decommissioned (Außerbetriebnahme) – der Akku hat den festgelegten Schwellenwert von derzeit 400 Ladezyklen erreicht und sollte möglichst bald durch einen neuen Akku ersetzt werden.
    • Unknown (Unbekannt) – beim Abrufen der Informationen zum Akkuzustand ist ein Problem aufgetreten.
  • Voltage (Spannung) – Anzeige der Stromspannung
  • Current (Stromstärke) – Anzeige der Stromstärke
  • Temperature (Temperatur) – Anzeige der Temperatur
  • Level (Akkustand) – Anzeige des Akkuladestands in Prozent (%)
  • Current Capacity (Aktuelle Kapazität) – Anzeige der im Akku verfügbaren Energie in mAh (Milliamperstunden). Dies wird nur mit PowerPrecision+-Akkus unterstützt.

WWAN Tests (WWAN-Tests)

Überprüfung der Funktionsfähigkeit des WWAN-Funkmoduls mit Bereitstellung von relevanten WWAN-Informationen Ergebnisse:

  • Sim State (SIM-Status) – der Flugmodus muss deaktiviert sein. Einer der folgenden Werte wird angezeigt:
    • Present (Vorhanden) – eine SIM-Karte ist vorhanden.
    • Absent (Nicht vorhanden) – es ist keine SIM-Karte vorhanden.
  • Voice State (Sprachstatus) – einer der folgenden Werte wird angezeigt:
    • Voice in service (Sprachdienst aktiviert)
    • Voice out of service (Sprachdienst deaktiviert)
    • Voice Emergency only (Sprache nur bei Notfall)
    • Voice power off (Sprache ausgeschaltet)
    • Unknown voice (Unbekannter Sprachstatus)
  • Data State (Datenstatus) – einer der folgenden Werte wird angezeigt:
    • Data connected (Datenverbindung vorhanden)
    • Data disconnected (Keine Datenverbindung)
    • Data connecting (Datenverbindung wird hergestellt)
    • Data suspended (Datenverbindung ausgesetzt)
    • Unknown data (Unbekannter Datenstatus)
  • WWAN Type (WWAN-Typ) – Anzeige des Netzwerktyps: LTE, 2G, 3G, 4G oder Not Available (Nicht verfügbar)
  • Signal Strength (Signalstärke) – einer der folgenden Werte wird angezeigt:
    • Not applicable (Nicht zutreffend) – keine SIM-Karte vorhanden
    • Unknown (Unbekannt) – das Gerät befindet sich möglicherweise im Flugmodus.
    • Signal strength (Signalstärke) – Anzeige der Signalstärke in dBm
  • Phone Number (Telefonnummer) – valid (gültig) oder invalid (ungültig), je nachdem, ob die Rufnummer vergeben ist oder nicht.
  • Device ID (Geräte-ID) – valid (gültig) oder invalid (ungültig), je nachdem, ob die Geräte-ID vergeben ist oder nicht.

Audio Test (Audio-Test)

Überprüfung der Funktionsfähigkeit von Mikrofon und Lautsprecher des Geräts Ergebnisse:

  • Audio Test (Audio-Test) – test successful (Test erfolgreich) oder test failed (Test fehlgeschlagen)
