Verwaltete Konfigurationen

Device Diagnostic Tool 2.7

Überblick

Device Diagnostic Tool (DDT) kann mithilfe von verwalteten Konfigurationen per Fernzugriff konfiguriert werden. Diese von Google und der Android-Community entwickelten Spezifikationen dienen dazu, installierte Apps zu konfigurieren und diese Konfigurationen über ein EMM-System (Enterprise Mobility Management) bereitzustellen. Ein Schema definiert, welche DDT-Funktionen dem EMM-System zur Verfügung stehen, und stellt die erforderlichen Informationen für die entsprechende datengesteuerte Oberfläche des EMM-Systems bereit. Über diesen Mechanismus wird die DDT-Fernkonfiguration nahtlos in das EMM-System integriert. Sobald neue Merkmale verfügbar sind, veröffentlicht Zebra ein neues Schema. DDT und das Schema von Zebra sind im Google Play Store erhältlich und müssen gemeinsam verwendet werden, damit DDT mit den verwalteten Konfigurationen in der Lage ist, Diagnosevorgänge durchzuführen.

Voraussetzungen

Voraussetzungen für die Verwendung von verwalteten Konfigurationen mit DDT:

  • Zebra-Gerät mit Android 10 oder Android 11
  • Download von DDT mit dem Schema von Zebra aus dem Google Play Store
  • DDT ist auf dem Gerät installiert und wurde bisher mindestens einmal gestartet.
  • Ein EMM-System zur Gerätebereitstellung
  • FTP-Server zum Hochladen von DDT-Testdaten
    Empfehlung: Um Risiken durch die Offenlegung von Anmeldedaten möglichst zu vermeiden, empfiehlt Zebra, Testdaten nur auf einen dedizierten Server hochzuladen.

Verwendung von verwalteten Konfigurationen

Im folgenden Abschnitt erhalten Sie allgemeine Hinweise zur Verwendung von verwalteten Konfigurationen über ein EMM-System. Das genaue Vorgehen ist vom verwendeten EMM-System abhängig. Sobald DDT im EMM-System dem App-Katalog zur Installation hinzugefügt wurde, wird das entsprechende Schema automatisch abgerufen und anhand der definierten Daten in die EMM-Seite für verwaltete Konfigurationen eingebunden. Auf dieser Seite kann der Administrator die gewünschten verwalteten Konfigurationen auswählen. Bevor diese Konfigurationen auf ein Gerät übertragen werden können, muss auf diesem Gerät DDT mindestens einmal gestartet worden sein. Andernfalls akzeptiert das Gerät diese Konfigurationen nicht. In der folgenden Anleitung wird auch erläutert, wie DDT mithilfe von OEMConfig per Fernzugriff gestartet werden kann.

Allgemeine Hinweise zur Verwendung von verwalteten Konfigurationen:

  1. Fügen Sie Device Diagnostic Tool dem App-Katalog hinzu. Suchen Sie auf dem EMM-System im Google Play Store Device Diagnostic Tool 2.7 oder höher, und fügen Sie es dem App-Katalog hinzu. Dadurch wird automatisch das entsprechende Schema für die App abgerufen, sodass die App zur Bereitstellung auf dem Gerät über das EMM-System verfügbar ist.

  2. Konfigurieren Sie App-Einschränkungen mithilfe von verwalteten Konfigurationen gemäß der Beschreibung im Schema. Im folgenden Abschnitt App-Einschränkungen werden die verfügbaren Optionen beschrieben.

  3. Übertragen Sie eine Richtlinie zur Bereitstellung der App und der zuvor erstellten verwalteten Konfigurationen auf das Gerät.

  4. Starten Sie DDT mithilfe von OEMConfig per Fernzugriff – zuvor muss DDT wie in Schritt 3 beschrieben installiert worden sein:

    • Falls noch nicht geschehen, installieren Sie OEMConfig auf dem Gerät.

    • In OEMConfig öffnen Sie „Device Administration Configuration“ (Konfiguration der Geräteadministration).

    • Wählen Sie Action > Submit XML (Aktion > XML übermitteln) aus.

    • Geben Sie in das Eingabefeld für Submit XML (XML übermitteln) den folgenden Text ein, um DDT zu starten:

      <wap-provisioningdoc><characteristic type="Intent"><parm name="Action" value="StartActivity"/><parm name="ActionName" value="android.intent.action.MAIN"/><parm name="Package" value="com.symbol.selfdiagnostics"/><parm name="Class" value="com.symbol.selfdiagnostics.activities.MainActivity"/><characteristic type="Extra"><parm name="ExtraType" value="string"/><parm name="ExtraName" value="LAUNCH_DDT"/><parm name="ExtraValue" value="1"/></characteristic></characteristic></wap-provisioningdoc>

      Hinweis: Zeilenschaltungen müssen entfernt werden.

    • Wenden Sie die OEMConfig-Konfiguration auf das Gerät an.

App-Einschränkungen

DDT bietet zwei Betriebsmodi: Admin-Modus und Benutzermodus. Derzeit sind die Planung von regelmäßigen Tests und die Konfiguration von Protokoll-Uploads nur im Admin-Modus möglich.

Admin-Modus

Im Admin-Modus gelten für DDT folgende App-Einschränkungen:

  1. Testplan – Erstellen oder Ändern eines Testplans mit Zeitplan, Aktivität und Ergebnissen.
  2. Testprotokoll-Aufbewahrung – Festlegen, ob Testprotokolle nach dem automatischen Upload auf den Server auf dem Gerät gespeichert oder von dort gelöscht werden sollen.
  3. Upload-Einstellung – Festlegen, ob Protokolle basierend auf einem festgelegten Zeitplan oder nur nach einem Testfehler hochgeladen werden sollen.

Optionen für App-Einschränkungen

In diesem Abschnitt werden die verfügbaren Optionen für die einzelnen App-Einschränkungen beschrieben.

Testplan

Mit „Testplan“ wird ein Testplan erstellt oder geändert, der den wöchentlichen Zeitplan sowie den auszuführenden Test enthält und in dem der Server konfiguriert ist, auf den die Testergebnisse hochgeladen werden sollen.
