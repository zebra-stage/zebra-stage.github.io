Uso de Device Diagnostic Tool

Device Diagnostic Tool 2.3

Aspectos generales

Device Diagnostic Tool admite dos modos de funcionamiento:

  1. Admin mode - (Modo de administrador) el administrador puede acceder a los ajustes de la aplicación, modificar las configuraciones de las pruebas y ejecutar cada una de las pruebas individuales. Se trata del ajuste predeterminado.
  2. User mode - (Modo de usuario) el usuario solo puede ejecutar pruebas, (Modo de usuario) el usuario solo puede ejecutar pruebas. No tiene acceso a los ajustes de la aplicación, a las configuraciones de prueba ni a pruebas individuales. Para establecer el modo de usuario, deshabilite admin_mode como se indica en el apartado Configuración.

Inicio de la aplicación

Ejecute Device Diagnostic Tool con uno de los métodos siguientes:

  1. Manually Turing The User Interfaces - (Uso manual de la interfaz de usuario) el usuario inicia Device Diagnostic Tool y ejecuta las pruebas manualmente. Consulte el apartado Interfaz de usuario a continuación.
  2. Remotely with an EMM (Enterprise Mobility Management) system (De manera remota, con un sistema EMM [Enterprise Mobility Management]) - inicie Device Diagnostic Tool en segundo plano y genere un registro con el comando:
    adb shell am broadcast -n "com.symbol.selfdiagnostics/com.symbol.selfdiagnostics.SESReceiver"

Solo debe usarse uno de los métodos para ejecutar Device Diagnostic Tool; de lo contrario, puede producirse un comportamiento inesperado.

Interfaz de usuario

Al iniciar por primera vez la aplicación, la pantalla principal muestra las pruebas disponibles para su ejecución: Main screen (Pantalla principal)
Si ya se han realizado pruebas, el nombre de la prueba va seguido de la fecha en que se llevó a cabo la última prueba.

Toque Run Tests (Ejecutar pruebas) en la pantalla principal para ejecutar todas las pruebas. La ejecución de pruebas se basa en las pruebas seleccionadas en la pantalla Configure Tests (Configuración de pruebas). Una vez iniciada la prueba, el sistema pide al usuario que realice acciones adicionales durante la ejecución de las pruebas siguientes: Scanner Test (Prueba del escáner), Button Test (Prueba de botones), Touch Screen Test (Prueba de la pantalla táctil) y Audio Test (Prueba de audio). Se recomienda realizar todas las pruebas mientras el dispositivo esté en uso normal, es decir, que no esté acoplado a una base ni conectado a un ordenado a través de USB. Una vez realizada las pruebas, se muestran los resultados:

   Pass - (Superación) la prueba se ejecutó y se superó. El resultado superó los criterios de la prueba.
   Fail - (No superación) la prueba se ejecutó y no cumplió los criterios de la prueba.
   Information - (Información) los datos se recuperan y se muestran.

En el modo de administrador también se pueden realizar pruebas individuales tocando la categoría de pruebas individuales y luego Run Tests (Ejecutar pruebas).
Test failed (Prueba no superada) y test timed-out (Agotado el tiempo de espera de la prueba) tienen como resultado el mismo icono rojo de riesgo.

La opción Help (Ayuda) está disponible al tocar el menú superior derecho de la pantalla principal. Enlaza con el portal de asistencia de Device Diagnostics Tool.

Prueba del escáner

El sistema pide al usuario que escanee un código de barras. Resultados:

  • Scanner Test – (Prueba del escáner) muestra los datos del código de barras
  • Label Type – (Tipo de etiqueta) muestra el tipo de código de barras o el descodificado escaneado

Note: Al realizar la prueba del escáner en TC55, el usuario debe realizar una pulsación larga en el botón del escáner para que se detecte.


Prueba de botones

El sistema pide al usuario que pulse los botones físicos del dispositivo: scan trigger (left or right) (disparador de escaneado [izquierdo o derecho]), push-to-talk, volume up (subir volumen) y volume down (bajar volumen). Resultados:

  • Button Test – (Prueba de botones) test successful (prueba correcta), test failed (prueba no superada) o test timed-out (agotado el tiempo de espera de la prueba)

Nota: Si los botones de tecla físicos se reasignan, la prueba de botones no funciona tal como se espera.


Prueba de la pantalla táctil

El sistema pide al usuario que toque cada uno de los cuadro de cuadrícula en la pantalla

  • Touch Screen Test – (Prueba de la pantalla táctil) test successful (prueba correcta), test failed (prueba no superada) o test timed-out (agotado el tiempo de espera de la prueba)

Pruebas de Bluetooth

Se comprueba si la radio Bluetooth funciona y se devuelve información relacionada con Bluetooth: Resultados:

  • Name – (Nombre) muestra el nombre Bluetooth
  • Radio Power Cycle – (Ciclo de potencia de radio) test successful (prueba correcta) o test failed (prueba no superada). Se mantiene el estado de la radio anterior a esta prueba.
  • Functional/Non-functional – (Operativa/No operativa) functional (operativa) o non-functional (no operativa)
  • Discoverable/Connectable - (Detectable/Conectable) connectable (conectable), discoverable (detectable) o none (ninguno)

WiFi Tests (Pruebas de WiFi)

Se comprueba el funcionamiento de la radio WiFi y se devuelve información relacionada con WiFi. Resultados:

  • MAC Address – (Dirección MAC) valid (válida) o invalid (no válida)
  • Network Test – (Prueba de red) muestra connected (conectada) o not connected (no conectada) Se produce un fallo si la WiFi no está conectada a ninguna red. Se conserva el estado de la radio anterior a esta prueba. Si la prueba es correcta, se muestran los siguientes valores:
  • strength – (intensidad) se muestra la intensidad de la señal
  • essid – se muestra el ESSID. Debido a restricciones de Android, para mostrar el ESSID en Android O y versiones superiores, es preciso que el servicio Location (Localización) esté habilitado en Configuración de dispositivo. Si el servicio Location (Localización) no está habilitado, el sistema pide al usuario que lo habilite. Si el usuario procede a ejecutar la prueba sin habilitar Location (Localización), el ESSID devuelve "Location not enabled" (Localización no habilitada)".
  • ip – muestra la dirección IP
  • bssid – valid (válido) o invalid (no válido)
  • speed – (velocidad) muestra la velocidad de conexión

Pruebas de batería

Se comprueba el estado de la batería y se devuelve información relacionada con la batería Resultados:

  • Part number – (Número de referencia) devuelve el número de referencia
  • Serial number – (Número de referencia) muestra el número de referencia
  • Manufacture date – (Fecha de fabricación) devuelve la fecha de fabricación de la batería
  • Decommission status – (Estado de retirada del servicio) muestra el estado de salud de la batería como:
    • Good – (Bueno) la batería se encuentra en buen estado
    • Decommissioned – (Retirado del servicio) la batería ha alcanzado el umbral designado, actualmente 400 ciclos de carga, lo que indica que la batería debe sustituirse cuanto antes por otra nueva.
    • Unknown – (Desconocido) indica problemas al recuperar la información del estado de salud de la batería
  • Voltage – (Tensión) muestra la tensión
  • Current – (Corriente) muestra la corriente
  • Temperature – (Temperatura) muestra la temperatura
  • Level - muestra el porcentaje (%) de batería restante
  • Current Capacity - (Capacidad de corriente) muestra la cantidad de energía almacenada en la batería, en mAh (miliamperios hora). Solo compatible con baterías PowerPrecision+.

Pruebas de WWAN

Se comprueba el funcionamiento de la radio WWAN y se devuelve información relacionada con WWAN. Resultados:

  • Sim State – (Estado de la SIM), debe deshabilitarse el modo Avión. Muestra uno de los valores siguientes:
    • Present – (Presente) la tarjeta SIM está presente
    • Absent – (Ausente) la tarjeta SIM no está presente
  • Voz State - (Estado de la voz) muestra uno de los valores siguientes:
    • Voice in service (Voz en servicio)
    • Voice out of service (Voz fuera de servicio)
    • Voice Emergency only (Solo emergencia de voz)
    • Voice power off (Apagado de voz)
    • Unknown voice (Voz desconocida)
  • Data State – (Estado de los datos) muestra uno de los valores siguientes:
    • Data connected (Datos conectados)
    • Data disconnected (Datos desconectados)
    • Data connecting (Datos en conexión)
    • Data suspended (Datos suspendidos)
    • Unknown data(Datos desconocidos)
  • WAN Type – (Tipo de WAN) muestra el tipo de red, como, por ejemplo: LTE, 2G, 3G, 4G, o Not Available (No disponible)
  • Signal Stregth– (Intensidad de la señal) muestra uno de los valores siguientes:
    • Not applicable - (No aplicable), ausencia de tarjeta SIM
    • Unknown – (Desconocido), puede que el dispositivo esté en modo Avión
    • Signal strength - (Intensidad de la señal) en dBm
  • Phone Number – (Número de teléfono) valid (válido) o invalid (no válido), en función de que el proveedor del servicio exponga o no el dispositivo.
  • Device ID – (ID de dispositivo) valid (válido) o invalid (no válido), en función de que se exponga o no el ID de dispositivo

Audio Test (Prueba de audio)

Se comprueba el funcionamiento del micrófono y el altavoz del dispositivo Resultados:

  • Audio Test – (Prueba de audio) test successful (prueba correcta) o test failed (prueba no superada)
