Criterios de pruebas

Device Diagnostic Tool 2.4

Descripción general

Device Diagnostic Tool comprueba la operación de las funciones de hardware del dispositivo según criterios específicos, con los resultados siguientes:

  • Superado - si se cumplen los criterios
  • No superado - si no se cumplen los criterios
  • Información - muestra datos informativos (no se aplica superado/no superado)
Prueba Requiere intervención del usuario Propiedades/Acción Criterios de superado/no superado
Escáner Sí Escanear un código de barras La prueba se supera cuando se escanea correctamente un código de barras.
Botón Sí Pulsación del botón:
  - Disparador de escaneado (izquierdo o derecho)
  - Push-to-talk
  - Subir volumen
  - Bajar volumen		 Supera la prueba cuando se detecta la pulsación de cada uno de los botones.
Pantalla táctil Sí Cuadrícula táctil La prueba se supera cuando se detecta funcionalidad táctil en cada cuadro de cuadrícula en la pantalla.
Bluetooth No Información devuelta:
  - Nombre
  - Reiniciar radio
  - Funcional/
No funcional
  - Conectable/
Detectable		 La prueba se supera si la radio Bluetooth se puede reiniciar y la información se devuelve correctamente.
WLAN No Información devuelta:
  -Dirección MAC
  - Prueba de red
  - Intensidad de la señal (i)
  - Dirección IP (i)
  -BSSID (i)
  - Velocidad (i)		 La prueba se supera cuando los resultados de la prueba de red son correctos.
WWAN No Información devuelta:
  - Estado de SIM
  - Estado de voz
  - Estado de datos
  - Tipo de WAN (i)
  - Intensidad de la señal (i)
  -Número de teléfono (i)
  -ID de dispositivo (i)		 La prueba se supera cuando la tarjeta SIM se encuentra presente. La prueba se supera automáticamente en los dispositivos no WAN.
Batería No Información devuelta:
  - Número de pieza
  - Número de serie
  - Fecha de fabricación
  - Decommission Status (Estado de retirada del servicio)
  - Tensión (i)
  - Corriente (i)
  - Temperatura (i)
  - Nivel (i)
  - Capacidad de corriente (i)		 La prueba se supera cuando el estado de salud de la batería es bueno y se recupera correctamente la información de la batería. Nota: Capacidad de corriente solo se aplica a las baterías PowerPrecision+.
Audio Sí Grabar y reproducir el archivo Se requiere intervención manual del usuario. El usuario graba manualmente el audio y luego escucha la grabación. La prueba se supera/no se supera a discreción del usuario. En esta prueba se deben usar los altavoces del dispositivo. No se acepta el uso de auriculares. Esta prueba debe llevarse a cabo en un entorno con el menor ruido de fondo posible para que sea correcta. Ajuste el dispositivo al volumen máximo durante la reproducción para obtener los mejores resultados.
Nota: Esta prueba no se puede ejecutar si el dispositivo no tiene micrófono, p. ej., MC18.


El texto de color rojo indica información confidencial que no se muestra. En su lugar, se muestra “válido" o “no válido" para indicar si la información se detecta en el formato adecuado. “Válido" significa que se recuperó el valor adecuado. “No válido" indica que el valor recuperado es nulo, está vacío o no coincide con el patrón de dirección MAC esperado (solo se aplica a la dirección MAC).

(i) indica datos informativos que se consideran datos adicionales, que se muestran si show_extra_data se establece en “true” en el archivo configuration.xml.
