Configuraciones gestionadas

Device Diagnostic Tool 2.7

Aspectos generales

Device Diagnostic Tool (DDT) puede configurarse de forma remota a través de Configuraciones gestionadas, especificaciones desarrolladas por Google y la comunidad Android para configurar aplicaciones instaladas y desplegarlas a través de EMM (Enterprise Mobility Management). Un esquema define las funciones de DDT que están disponibles para EMM, suministrando la información necesaria para presentar a la correspondiente interfaz de usuario basada en datos de EMM. Este mecanismo integra la configuración remota de DDT en EMM para un funcionamiento armonizado y permite a Zebra liberar un nuevo esquema en cuanto dispone de nuevas funciones. DDT y el esquema de Zebra están disponibles en Google Play Store y deben utilizarse juntos para sacar el máximo partido de DDT con sus configuraciones gestionadas para realizar operaciones de diagnóstico.

Requisitos

Requisitos para el uso de configuraciones gestionadas con DDT:

  • Dispositivo Zebra con Android 10 o Android 11
  • DDT con el esquema de Zebra descargado desde Google Play Store
  • DDT está instalado en el dispositivo y se debe iniciar al menos una vez
  • Un EMM para despliegue del dispositivo
  • Servidor FTP para cargar los datos de prueba de DDT
    Se recomienda: Para minimizar los riesgos de seguridad y evitar la revelación de credenciales de inicio de sesión, Zebra recomienda cargar datos de prueba a un servidor dedicado.

Utilizar configuraciones gestionadas

Esta sección ofrece instrucciones generales para utilizar configuraciones gestionadas a través de un EMM La interfaz de usuario puede variar según el EMM En el EMM, después de añadir DDT al catálogo de aplicaciones del dispositivo para la instalación de la aplicación, se recuperará automáticamente el esquema correspondiente y se presentará a la interfaz de usuario de configuraciones gestionadas de EMM de acuerdo con los datos definidos. A través de esta interfaz de usuario el administrador puede seleccionar las configuraciones gestionadas deseadas. Antes de insertar las configuraciones gestionadas en los dispositivos, DDT debe iniciarse al menos una vez para que acepte estas configuraciones. En las instrucciones se indica cómo puede iniciarse DDT de forma remota con OEMConfig.

Instrucciones generales para utilizar configuraciones gestionadas:

  1. Añada Device Diagnostic Tool al catálogo de aplicaciones. En el EMM, busque Device Diagnostic Tool 2.7 o superior en Google Play Store y añádalo al catálogo de aplicaciones. De este modo se recuperará automáticamente el esquema correspondiente para la aplicación y esta pasará a estar disponible para su despliegue en los dispositivos a través del EMM.

  2. Configurar restricciones de la aplicación utilizando configuraciones gestionadas descritas en el esquema. Consulte la sección Restricciones de la aplicación siguiente, en la que se indican las opciones disponibles.

  3. Inserte una directiva que despliegue la aplicación DDT y las configuraciones gestionadas creadas en los dispositivos.

  4. Inicie DDT de forma remota utilizando OEMConfig - DDT debe instalarse en el paso 3 antes de ejecutar este paso:

    • Instale OEMConfig en el dispositivo, si no se ha instalado anteriormente.

    • En OEMConfig, abra Device Administration Configuration.

    • Seleccione Action > Submit XML.

    • En el campo de entrada Submit XML, escriba el texto siguiente para iniciar DDT:

      <wap-provisioningdoc><characteristic type="Intent"><parm name="Action" value="StartActivity"/><parm name="ActionName" value="android.intent.action.MAIN"/><parm name="Package" value="com.symbol.selfdiagnostics"/><parm name="Class" value="com.symbol.selfdiagnostics.activities.MainActivity"/><characteristic type="Extra"><parm name="ExtraType" value="string"/><parm name="ExtraName" value="LAUNCH_DDT"/><parm name="ExtraValue" value="1"/></characteristic></characteristic></wap-provisioningdoc>

      Nota: Los saltos de línea deben eliminarse.

    • Aplique la configuración de OEMConfig a los dispositivos.

Restricciones de la aplicación

Hay dos modos de funcionamiento para DDT: modo de administrador y modo de usuario. Actualmente solo se admite el modo de administrador con configuración gestionada para programar pruebas periódicas y configurar cargas de registros.

Modo de administrador

En el modo de administrador, las restricciones de la aplicación DDT son las siguientes:

  1. Plan de pruebas - Cree o modifique un plan de pruebas que incluya Programación, Actividad y Entrega.
  2. Retención de registro de pruebas - Seleccione si se mantienen o descartan los planes de pruebas en el dispositivo después de la carga automática en el servidor.
  3. Preferencia de carga - Seleccione si los registros se cargan basándose en la programación especificada o solo después de un error de prueba.

Plan de pruebas crea o modificar un plan de prueba que incluye programación de prueba semanal, especificando la prueba que se realizará y configurando el servidor para recibir los resultados de la prueba.

Retención de registro de prueba

Retención de registro de prueba especifica la acción que se realizará para todos los archivos de planes de pruebas en el dispositivo después de cargarlos en el servidor designado.

Nombre de la restricción Descripción Valor
Retención de registro de prueba Conserve o descarte los archivos de planes de pruebas después de cargarlos en el servidor • Mantener después de cargar
• Eliminar después de cargar

Preferencia de carga permite seleccionar si los registros se cargan basándose en la programación especificada o solo después de un error de prueba.

Preferencia de carga Seleccione si los registros se cargan basándose en una programación especificada o solo después de un error de prueba • Todos los registros de prueba
