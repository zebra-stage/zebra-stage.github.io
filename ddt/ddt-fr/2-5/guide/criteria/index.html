Critères des tests

Device Diagnostic Tool 2.5

Présentation

Device Diagnostic Tool vérifie le fonctionnement des fonctionnalités matérielles des équipements sur la base de critères spécifiques. Les résultats sont les suivants :

  • Réussite : si le résultat répond au critère défini
  • Échec : si le résultat ne répond pas au critère défini
  • Information : affiche des informations (Réussite/Échec ne s'applique pas au test)
Test Intervention de l'utilisateur requise Propriétés/action Critères de réussite/d'échec
Scanner Oui Lecture d'un code-barres Le test réussit si la lecture d'un code-barres aboutit.
Bouton Oui Pression sur un bouton :
  - Déclencheur de lecture (gauche ou droite)
  - Talkie-walkie
  - Augmentation du volume
  - Réduction du volume		 Le test réussit si la pression est détectée sur chacun des boutons.
Écran tactile Oui Grille tactile Le test réussit si le toucher est détecté dans chaque section de la grille à l'écran.
Bluetooth Non Informations renvoyées :
  - Nom
  - Mise sous/hors tension de radio
  - Opérationnelle/
Non opérationnelle
  - Connexion possible/
Identifiable		 Le test réussit si la radio Bluetooth peut être mise sous/hors tension et que des informations sont renvoyées.
Réseau local sans fil Non Informations renvoyées :
  -Adresse MAC
  - Test du réseau
  - Intensité du signal (i)
  - ESSID (i)  - Adresse IP (i)
  -BSSID (i)
  - Vitesse (i)		 Le test réussit si le test du réseau aboutit.
Réseau étendu sans fil Non Informations renvoyées :
  - État de carte SIM
  - État de voix
  - État des données
  - Type de réseau étendu (i)
  - Intensité du signal (i)
  -Numéro de téléphone (i)
  -ID d'équipement (i)		 Le test réussit si la carte SIM est insérée. Il réussit automatiquement sur les équipements non-WAN.
Batterie Non Informations renvoyées :
  - Numéro de référence
  - Numéro de série
  - Date de fabrication
  - Decommission Status (État de mise hors service)
  - Tension (i)
  - Alimentation (i)
  - Température (i)
  - Niveau de charge (i)
  - Capacité actuelle (i)		 Le test réussit si l'état de la batterie est satisfaisant et que les informations associées sont renvoyées.
Remarque : Capacité actuelle s'applique aux batteries PowerPrecision+ uniquement.
Audio Oui Enregistrement et lecture d'un fichier L'intervention manuelle de l'utilisateur est nécessaire. L'utilisateur enregistre manuellement un fichier audio, puis écoute l'enregistrement. Il lui appartient de décider si le test réussit ou échoue. Les haut-parleurs de l'équipement doivent être utilisés dans le cadre de ce test, car l'utilisation d'écouteurs n'est pas acceptable. Ce test doit être réalisé dans un environnement avec un minimum de bruits de fond. Pour obtenir des résultats optimaux, le volume de l'équipement doit être réglé sur la valeur maximale.
Remarque : il est impossible de réaliser ce test si l'équipement n'intègre pas de microphone (le MC18, par exemple).
Carte SD Non Informations renvoyées :
  - Présence de carte SD
  - Lecture
  - Écriture
  - Espace total (i)
  - Espace disponible (i)		 Les tests réussissent si la carte SD est présente et qu'il est possible de lire et d'écrire sur la carte. Un avertissement est généré si la carte SD n'est pas installée ou lorsque la lecture ou l'écriture échoue.


Le texte en rouge représente des informations sensibles qui ne sont pas affichées. Le texte « Valide » ou « Non valide » s'affiche pour indiquer si les informations sont détectées dans le format correct. « Valide » indique que la valeur extraite est appropriée. « Non valide » indique que la valeur extraite est nulle, vide ou ne correspond pas au schéma d'adresse MAC prévu (ne s'applique qu'à l'adresse MAC).

(i) indique des informations considérées comme complémentaires, qui s'affichent si show_extra_data (afficher_données_complémentaires) est réglé sur « true » dans le fichier configuration.xml.

Vous trouverez une description de chaque test à la section Configurer les tests.
