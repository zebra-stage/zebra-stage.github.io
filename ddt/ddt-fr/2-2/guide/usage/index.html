Utilisation de Device Diagnostic Tool

Device Diagnostic Tool 2.2

Présentation

Device Diagnostic Tool prend en charge deux modes de fonctionnement :

  1. Mode administrateur : l'administrateur peut accéder aux paramètres de l'application, modifier la configuration des tests et exécuter un test spécifique. Ce mode est activé par défaut.
  2. Mode utilisateur : l'utilisateur est autorisé à exécuter les tests uniquement. Il n'a accès ni aux paramètres de l'application, ni à la configuration des tests. Pour activer le mode utilisateur, désactivez admin_mode (mode_admin) comme indiqué à la section Configuration.

Lancement de l'application

Exécutez Device Diagnostic Tool comme suit, au choix :

  1. Manuellement à l'aide de l'interface utilisateur : vous démarrez Device Diagnostic Tool et exécutez manuellement les tests. Reportez-vous à la section Interface utilisateur ci-dessous.
  2. À distance à l'aide d'un système EMM (Enterprise Mobility Management) : démarrez Device Diagnostic Tool en arrière-plan et générez un fichier journal à l'aide de la commande :
    adb shell am broadcast -n "com.symbol.selfdiagnostics/com.symbol.selfdiagnostics.SESReceiver"

N'utilisez que l'une des méthodes pour exécuter Device Diagnostic Tool. Si vous utilisez simultanément les deux méthodes, un comportement inattendu risque de se produire.

Interface utilisateur

Lorsque vous démarrez l'application pour la première fois, l'écran principal affiche les tests disponibles : Écran principal
Si un test a déjà été réalisé, son nom est suivi de la date de sa dernière exécution.

Touchez Run Tests (Exécuter les tests) sur l'écran principal pour exécuter tous les tests. Sont exécutés les tests sélectionnés sur l'écran Configure Tests (Configurer les tests). Une fois la procédure démarrée, vous êtes invité à effectuer des actions complémentaires lors de l'exécution des tests suivants : Scanner Test (Test du scanner), Button Test (Test des boutons), Touch Screen Test (Test d'écran tactile) et Audio Test (Test audio). Il est recommandé d'exécuter tous les tests dans le cadre d'une utilisation normale de l'équipement (en d'autres termes, évitez qu'il ne soit placé dans la station d'accueil ou connecté par USB à un ordinateur). Une fois les tests terminés, les résultats sont affichés :
   Pass (Réussite) : le test a été exécuté et est réussi. Le résultat est conforme aux critères de test.
   Fail (Échec) : le test a été exécuté et n'est pas conforme aux critères définis.

   Information : des données ont été extraites et affichées.

En mode administrateur, vous pouvez également exécuter un test spécifique. Pour ce faire, touchez la catégorie appropriée, puis Run Tests (Exécuter les tests).
Si les valeurs test failed (le test a échoué) ou test timed-out (le délai imparti au test a été dépassé) sont renvoyées, la même icône rouge indiquant un danger s'affiche.

Scanner Test (Test du scanner)

L'utilisateur est invité à lire un code-barres. Résultats :

  • Scanner Test (Test du scanner) : affiche les données du code-barres
  • Label Type (Type d'étiquette) : affiche le type de codes-barres ou le décodeur lu

Remarque : si l'utilisateur exécute le test du scanner sur un TC55, il doit appuyer sur le bouton de lecture pour qu'il soit détecté.

Button Test (Test des boutons)

L'utilisateur est invité à appuyer sur les boutons physiques de l'équipement : déclencheur de lecture (gauche ou droite), talkie-walkie et contrôle du volume (+/-). Résultats :

  • Button Test (Test des boutons) : test successful (le test a réussi), test failed (le test a échoué) ou test timed-out (le délai imparti au test a été dépassé)

Remarque : si les boutons physiques sont remappés, le test des boutons ne fonctionne pas comme prévu.

Touch Screen Test (Test d'écran tactile)

L'utilisateur est invité à toucher chaque section de la grille à l'écran.

  • Touch Screen Test (Test d'écran tactile) : test successful (le test a réussi), test failed (le test a échoué) ou test timed-out (le délai imparti au test a été dépassé)

Bluetooth Tests (Tests Bluetooth)

Ces tests vérifient si la radio Bluetooth fonctionne correctement et renvoient des informations associées. Résultats :

  • Name (Nom) : affiche le nom Bluetooth
  • Radio Power Cycle (Mise sous/hors tension de radio) : test successful (le test a réussi) ou test failed (le test a échoué). L'état de la radio avant le test n'est pas modifié.
  • Functional/Non-functional : functional (opérationnelle) ou non-functional (non opérationnelle)
  • Discoverable/Connectable : connectable (connexion possible), discoverable (identifiable) ou none (aucun)

Wifi Tests (Tests WiFi)

Ces tests vérifient le fonctionnement de la radio WiFi et renvoient des informations associées. Résultats :

  • MAC Address (Adresse MAC) : valid (valide) ou invalid (non valide)
  • Network Test (Test du réseau) : affiche connected (connecté) ou not connected (non connecté). Le test échoue si la fonction WiFi n'est connectée à aucun réseau. L'état de la radio avant le test n'est pas modifié. Si le test réussit, les valeurs suivantes s'affichent :
    • strength : indique l'intensité du signal
    • essid : affiche ESSID. Sous Android O et ultérieur, l'emplacement doit être activé dans les paramètres de l'équipement en raison des restrictions Android. Si l'emplacement de l'équipement n'est pas activé, la valeur unknown ssid (ssid inconnu) est renvoyée.
    • ip : affiche l'adresse IP
    • bssid : valid (valide) ou invalid (non valide)
    • speed : affiche la vitesse de connexion
  • Radio Power Cycle (Mise sous/hors tension de radio) : test successful (le test a réussi) ou test failed (le test a échoué)

Battery Tests (Tests de la batterie)

Ces tests vérifient l'état de la batterie et renvoient des informations associées. Résultats :

  • Part Number (Numéro de référence) : affiche le numéro de référence de la batterie
  • Serial Number (Numéro de série) : affiche le numéro de série de la batterie
  • Manufacture Date (Date de fabrication) : affiche la date de fabrication de la batterie
  • Decommission Status (État de mise hors service) : affiche l'état de santé de la batterie comme suit :
    • Good (Satisfaisant) : l'état de santé de la batterie est satisfaisant
    • Decommissioned (Mise hors service) : la batterie a atteint le seuil stipulé (400 cycles de charge à l'heure actuelle), ce qui signifie qu'elle devra bientôt être remplacée
    • Unknown (Inconnu) : il a été impossible d'extraire des informations sur l'état de santé de la batterie
  • Voltage (Tension) : affiche la tension
  • Current (Alimentation) : affiche l'alimentation
  • Temperature (Température) : affiche la température

WWAN Tests (Tests du réseau étendu sans fil)

Ces tests vérifient le fonctionnement de la radio WWAN et renvoient des informations associées. Résultats :

  • Sim State (État de carte SIM) : le mode avion doit être désactivé. L'une des valeurs ci-dessous est affichée :
    • Present (Présente) : la carte SIM est insérée
    • Absent (Absente) : la carte SIM n'est pas insérée
  • Voice State (État de voix) : affiche l'une des valeurs suivantes :
    • Voice in service (Voix en service)
    • Voice out of service (Voix hors service)
    • Voice Emergency only (Voix en cas d'urgence uniquement)
    • Voice power off (Voix hors tension)
    • Unknown voice (État de voix inconnu)
  • Data State (État des données) : affiche l'une des valeurs suivantes :
    • Data connected (Données connectées)
    • Data disconnected (Données déconnectées)
    • Data connecting (Données en cours de connexion)
    • Data suspended (Données interrompues)
    • Unknown data (État des données inconnu)
  • WAN Type (Type de réseau étendu) : affiche le type de réseau, à savoir : LTE, 2G, 3G, 4G ou Not Available (Non disponible)
  • Signal strength (Intensité du signal) : affiche l'une des valeurs suivantes :
    • Not applicable (Sans objet) : la carte SIM n'est pas insérée
    • Unknown (Inconnu) : l'équipement est peut-être en mode avion
    • Signal strength (Intensité du signal) : exprimée en dBm
  • Phone Number (Numéro de téléphone) : valid (valide) ou invalid (non valide), selon que le numéro de téléphone est révélé ou non par le fournisseur de services
  • Device ID (ID d'équipement) : valid (valide) ou invalid (non valide), selon que l'ID de l'équipement est révélé ou non

Audio Test (Test audio)

Ce test vérifie le fonctionnement du microphone et du haut-parleur de l'équipement. Résultats :

  • Audio Test (Test audio) : test successful (le test a réussi) ou test failed (le test a échoué)
