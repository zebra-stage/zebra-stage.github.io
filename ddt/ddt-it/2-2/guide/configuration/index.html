Configurazione

Device Diagnostic Tool 2.2

Panoramica

Questa sezione descrive le impostazioni di configurazione e le opzioni di reporting di Device Diagnostic Tool.

Impostazioni

La schermata Settings (Impostazioni) fornisce il percorso di file necessario per importare/esportare il file di configurazione, inoltre permette di configurare i file di registro relativi a cronologia e stato. Per accedere alle impostazioni toccare il menu Options (Opzioni) nell'angolo superiore destro della schermata principale, quindi selezionare Settings: Impostazioni dell'applicazione

  • File path (Percorso file):
    • Configuration import path (Percorso di importazione del file di configurazione) - specifica il percorso del file di configurazione: /enterprise/usr/deviceseftest/configuration.xml
    • Log output and configuration export path (Percorso di output del registro e di esportazione del file di configurazione) - specifica il percorso della cartella che contiene i file di registro e il file di configurazione esportato: /storage/emulated/0/Android/data/com.symbol.selfdiagnostics/files
  • Status log (Registro di stato) - campo di testo editabile per il nome file del registro di stato. Contiene informazioni sui test condotti, con eventuali parametri associati e modifiche alla configurazione apportate nella schermata Configure Tests (Configurazione dei test). Risiede in /storage/emulated/0/Android/data/com.symbol.selfdiagnostics/files.
  • History log (Registro cronologia) - campo di testo editabile per il registro della cronologia, che contiene informazioni sui risultati dei test. Viene generato quando si esegue un test completo dalla schermata principale. Risiede in /storage/emulated/0/Android/data/com.symbol.selfdiagnostics/files.
  • Log File Max Size (MB) (Dimensione massima del file di registro) - campo di testo editabile che specifica la dimensione massima dei file di registro.
  • Import (Importa) - importa un file di configurazione, che risiede nel percorso di importazione della configurazione /enterprise/usr/deviceseftest/configuration.xml.
  • Export (Esporta) - esporta configuration.xml dalle configurazioni impostate nella schermata Configure Tests (Configurazione dei test), che risiede in /storage/emulated/0/Android/data/com.symbol.selfdiagnostics/files. Una volta esportata, la configurazione può essere applicata ad altri dispositivi copiando il file .xml nel percorso di importazione /enterprise/usr/deviceselftest e quindi toccando il pulsante Import (Importa) nella schermata Settings (Impostazioni).

File di configurazione

Configuration.xml può essere modificato manualmente mediante un qualsiasi editor di testo. Tuttavia, è necessario mantenere il formato tag esistente. Le opzioni possono essere configurate dalle schermate Settings o Configure Tests nell'interfaccia utente, fatta eccezione per le opzioni elencate di seguito:

  • admin_mode – se è impostata su true, configura l'app in modalità Amministratore, consentendo l'accesso alle impostazioni dell'app e alle configurazioni dei test. Valore predefinito = true.
  • continue_testing_on_failure - se è impostata su true, il test procede automaticamente alla sezione successiva, fino a quando non vengono eseguiti tutti i test disponibili, anche se il test della sezione precedente non va a buon fine. Valore predefinito = false.
  • show_extra_data - se è impostata su true, nella pagina dei risultati dei test vengono visualizzati dati informativi. Per visionare i dati informativi interessati, consultare la sezione Parametri nominali. Valore predefinito = true.
  • debug – se è impostata su true, il registro dell'applicazione viene visualizzato in Logcat a fini di debug. Valore predefinito = true.
  • close_on_result_acknowledge - se è impostata su true, l'applicazione si chiude se tutti i test danno esito positivo. Se uno dei test non va a buon fine, l'app non si chiude. Valore predefinito = false. Esempio di impostazioni contenute nel file configuration.xml: true true true true false status.log history.log 25

Configure Tests (Configurazione dei test)

Selezionare i test da condurre e configurare i parametri di prova. Per accedere alla schermata Configure Tests toccare il menu Options (Opzioni) nell'angolo superiore destro della schermata principale, quindi selezionare Configure Tests:

       

Schermata Configure Tests

È possibile specificare dei valori di timeout per i test che richiedono l'intervento dell'utente. Trascorso il tempo predefinito (espresso in secondi), viene eseguito il successivo test selezionato, se così configurato nel file configuration.xml. Parametri di configurazione:

  • Scanner Test (Prova scanner) – verifica il funzionamento dello scanner
    • Test Timeout (Timeout del test) – specifica il tempo di attesa per l'input dell'utente, in secondi
  • Button Test (Prova pulsanti) – verifica la funzionalità dei pulsanti fisici dei dispositivi: push-to-talk, grilletto di scansione (lato sinistro o destro) aumento e riduzione del volume.
    • Test Timeout (Timeout del test) – specifica il tempo di attesa per l'input dell'utente, in secondi
  • (Prova touch screen) – verifica la funzionalità dello schermo di visualizzazione del dispositivo
    • Test Timeout (Timeout del test) – specifica il tempo di attesa per l'input dell'utente, in secondi
  • Bluetooth Tests (Prove Bluetooth) – verifica la funzionalità della radio Bluetooth e restituisce informazioni relative al sistema Bluetooth
    • Name (Nome) – indica il nome della rete Bluetooth
    • Radio Power Cycle (Ciclo di alimentazione della radio) – verifica il funzionamento con la radio Bluetooth accesa/spenta
    • Functional/Non-functional (Funzionante/Non funzionante) – determina se è possibile abilitare/disabilitare la radio Bluetooth entro l'intervallo di tempo specificato. Se la radio Bluetooth funziona, l'esito è "Functional".
    • Discoverable/Connectable (Individuabile/Collegabile) – verifica se la radio è collegabile o individuabile
  • WiFi Tests (Prove Wi-Fi) – verifica la funzionalità della radio Wi-Fi e restituisce informazioni relative al sistema Wi-Fi
    • MAC address (Indirizzo MAC) – verifica se l'indirizzo MAC è valido in base al rispettivo formato
    • Ping Address (Indirizzo Ping) – specifica il nome di dominio per condurre il test del ping
    • Network Test (Test di rete) – esegue il ping dell'indirizzo Web specificato nel campo Ping Address via Wi-Fi e restituisce il tempo impiegato per ricevere una risposta
    • Radio Power Cycle (Ciclo di alimentazione della radio) – verifica il funzionamento con la radio Wi-Fi accesa/spenta
  • Battery Tests (Prove batteria) – verifica lo stato della batteria e restituisce informazioni relative alla batteria
    • Part Number (Codice articolo) – restituisce il codice articolo della batteria
    • Serial Number (Numero di serie) – restituisce il numero di serie della batteria
    • Manufacture Date (Data di fabbricazione) – restituisce la data di fabbricazione della batteria
    • Decommission Status (Stato di decommissioning) – restituisce le condizioni della batteria e indica se debba essere smaltita
  • WWAN Tests (Prove WWAN) – verifica la funzionalità della radio WWAN e restituisce informazioni relative al sistema WWAN
    • Sim State (Stato SIM) – verifica che la scheda SIM sia presente
    • Voice State (Stato voce) – verifica lo stato dei servizi vocali della SIM
    • Data State (Stato dati) – verifica lo stato dei dati della scheda SIM
  • Audio Test (Prova audio) – verifica la funzionalità del microfono e degli altoparlanti del dispositivo
    • Test Timeout (Timeout del test) – specifica il tempo di attesa per l'input dell'utente, in secondi

Se si modificano dei valori dei test, ad esempio Test Timeout o Ping Address, questi rimangono in vigore solo fino a quando l'app è in esecuzione. Chiudendo e riavviando l'app, vengono sostituiti dai valori acquisiti dal file di configurazione.

Per ottenere maggiori informazioni, consultare la sezione Parametri nominali.
