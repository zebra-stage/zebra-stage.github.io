Configurazioni gestite

Device Diagnostic Tool 2.6

Panoramica

Device Diagnostic Tool (DDT) può essere configurato da remoto mediante la funzione Configurazioni gestite, con specifiche sviluppate da Google e dalla comunità di utenti Android per configurare le applicazioni installate e implementare le configurazioni mediante un sistema EMM (Enterprise Mobility Management). Uno schema definisce le funzionalità DDT accessibili al sistema EMM, con le informazioni necessarie per presentare a tale sistema l'interfaccia utente con i dati corrispondenti. Questo meccanismo integra alla perfezione la configurazione remota di DDT nel sistema EMM e permette a Zebra di rilasciare un nuovo schema non appena sono disponibili per l'uso nuove funzionalità. DDT e lo schema Zebra sono disponibili in Google Play Store e devono essere usati insieme per poter utilizzare DDT con le configurazioni gestite al fine di eseguire operazioni diagnostiche.

Requisiti

Requisiti per l'uso della funzionalità Configurazioni gestite con DDT:

  • Dispositivo Zebra con Android 10 o Android 11
  • DDT con schema Zebra scaricato da Google Play Store
  • DDT installato sul dispositivo e avviato almeno una volta
  • Un sistema EMM per l'implementazione del dispositivo
  • Server FTP per caricare i dati di test di DDT
    Suggerimento: per ridurre al minimo i rischi per la sicurezza legati alla divulgazione di credenziali di login, Zebra consiglia di caricare i dati di test su un server dedicato.

Uso di Configurazioni gestite

Questa sezione fornisce istruzioni generali per l'uso di Configurazioni gestite con un sistema EMM. Il layout dell'interfaccia può variare in base all'EMM specifico. Nel sistema EMM, dopo aver aggiunto DDT al catalogo di app dispositivo per l'installazione dell'app, lo schema corrispondente viene recuperato automaticamente e presentato nell'interfaccia utente delle configurazioni gestite EMM per i dati definiti. Mediante questa interfaccia, l'amministratore può selezionare le Configurazioni gestite desiderate Per poter inviare le Configurazioni gestite ai dispositivi e fare in modo che siano accettate, è necessario avviare DDT almeno una volta. Le istruzioni spiegano anche come avviare DDT da remoto con OEMConfig.

Istruzioni generali per l'uso di Configurazioni gestite:

  1. Aggiungere Device Diagnostic Tool al catalogo applicazioni. Nel sistema EMM, individuare o cercare Device Diagnostic Tool 2.6 o successivo da Google Play Store e aggiungerlo al catalogo app. Per effetto di questa operazione, lo schema corrispondente all'app viene recuperato automaticamente e l'app diventa disponibile per l'implementazione sui dispositivi mediante il sistema EMM.

  2. Configurare le restrizioni per l'app mediante Configurazioni gestite come descritto nello schema. Vedere la sezione Restrizioni per l'app di seguito, che descrive le opzioni disponibili.

  3. Inviare una policy che implementa sui dispositivi l'app DDT e le configurazioni gestite create.

  4. Avviare DDT da remoto mediante OEMConfig - DDT deve essere stato installato al punto 3 di questa procedura prima di effettuare questo passaggio:

    • Installare OEMConfig sul dispositivo (se non è già installato).

    • In OEMConfig, aprire Device Administration Configuration.

    • Selezionare Action > Submit XML.

    • Nel campo Submit XML, inserire il testo seguente per avviare DDT:

        
    <wap-provisioningdoc>
    <characteristic type="Intent">
    <parm name="Action" value="StartActivity"/>
    <parm name="ActionName" value="android.intent.action.MAIN"/>
    <parm name="Package" value="com.symbol.selfdiagnostics"/>
    <parm name="Class" value="com.symbol.selfdiagnostics.activities.MainActivity"/>
    <characteristic type="Extra">
    <parm name="ExtraType" value="string"/>
    <parm name="ExtraName" value="LAUNCH_DDT"/>
    <parm name="ExtraValue" value="1"/>
    </characteristic>
    </characteristic>
    </wap-provisioningdoc>

      Nota: rimuovere le interruzioni di riga.

    • Applicare la configurazione OEMConfig ai dispositivi.

Restrizioni per l'app

Per DDT sono disponibili 2 modalità operative: amministratore e utente. Attualmente è supportata solo la modalità amministratore per utilizzare una configurazione gestita per pianificare test ricorrenti e configurare il caricamento dei registri.

Modalità amministratore

Nella modalità amministratore, le restrizioni per l'app DDT sono le seguenti:

  1. Piano di test - Creare o modificare un piano di test che includa gli elementi Pianificazione, Attività e Consegna.
  2. Conservazione registro di test - Selezionare se i piani di test devono essere conservati sul dispositivo oppure eliminati dopo il caricamento automatico sul server.
