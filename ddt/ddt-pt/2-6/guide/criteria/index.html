Critérios de teste

Device Diagnostic Tool 2.6

Visão geral

O Device Diagnostic Tool testa a operação de recursos de hardware do dispositivo com base em critérios específicos, com os seguintes resultados:

  • Aprovado - se o critério for atendido
  • Reprovado - se o critério não for atendido
  • Informações - exibe dados informativos (aprovação/reprovação não é aplicável)
Teste Requer intervenção do usuário Propriedades/ação Critérios de aprovação/reprovação
Scanner Sim Escanear um código de barras Aprovado quando um código de barras é digitalizado com êxito.
Botão Sim Pressionamento do botão:
  - Gatilho do scanner (esquerdo ou direito)
  - Pressionar para falar
  - Aumentar volume
  - Diminuir volume		 Aprovado quando é detectado pressionamento do botão para cada um dos botões.
Tela de toque Sim Grade de toque Aprovado quando o toque é detectado dentro de cada caixa da grade na tela.
Bluetooth Não Informações retornadas:
  - Nome
  - Ciclo de energia de rádio
  - Funcional/
Não funcional
  - Conectável/
Detectável		 Aprovado se o rádio Bluetooth puder ser ligado e as informações forem retornadas com sucesso.
WLAN Não Informações retornadas:
  -Endereço MAC
  - Teste de rede
  - Força do sinal (i)
  - Endereço IP (i)
  - BSSID (i)
  - Velocidade (i)		 Aprovado quando o teste de rede é bem-sucedido.
WWAN Não Informações retornadas:
  - Estado do SIM
  - Estado de voz
  - Estado de dados
  - Tipo de WAN (i)
  - Força do sinal (i)
  - Número de telefone (i)
  - ID de dispositivo (i)		 Aprovado quando o cartão SIM está presente. Aprovado automaticamente em dispositivos não WAN.
Bateria Não Informações retornadas:
  - Número da peça
  - Número de série
  - Data de fabricação
  - Integridade da bateria
  - Tensão (i)
  - Corrente (i)
  - Temperatura (i)
  - Nível (i)
  - Capacidade de corrente (i)		 Aprovado se o estado da bateria estiver bom e as informações da bateria forem recuperadas com sucesso.
Nota: A capacidade atual só se aplica a baterias PowerPrecision+.
Áudio Sim Gravar e reproduzir arquivo Usar intervenção manual se necessário. O usuário grava manualmente o áudio e, em seguida, ouve a gravação. Teste aprovado/reprovado é baseado na discrição do usuário. Os alto-falantes do dispositivo devem ser usados para este teste - o uso de fones de ouvido não é aceitável. Este teste deve ser realizado em um ambiente com ruído de fundo mínimo para que o teste seja bem-sucedido. Ajuste o dispositivo com o volume máximo durante a reprodução para obter melhores resultados.
Nota: Este teste não pode ser realizado se o dispositivo não tiver um microfone, por exemplo, MC18.
Cartão SD Não Informações retornadas:
  - Teste de presença do cartão SD
  - Leitura
  - Gravação
  - Espaço total (i)
  - Espaço livre (i)		 Aprovado se o cartão SD externo estiver presente e se for aprovado nos testes de leitura e gravação. É exibido um aviso se o cartão SD não estiver presente ou se não for aprovado no teste de leitura ou gravação.


Texto na cor vermelhaindica informações confidenciais que não são exibidas. Em vez disso, é exibido “válido” ou “inválido” para indicar se a informação foi detectada no formato adequado. “Válido” indica que o valor apropriado foi obtido. “Inválido” indica que o valor obtido é nulo, vazio ou não corresponde ao padrão de endereço MAC esperado (aplica-se apenas ao endereço MAC).

(i) indica que os dados informativos que são considerados como dados extras, que são exibidos se show_extra_data for definido como “true” no arquivo configuration.xml.

Consulte a seção Configurar testes para obter uma descrição de cada teste.
