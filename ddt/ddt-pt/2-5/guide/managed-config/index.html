Configurações gerenciadas

Device Diagnostic Tool 2.5

Visão geral

O Device Diagnostic Tool (DDT) pode ser configurado remotamente por meio de Configurações Gerenciadas, especificações desenvolvidas pelo Google e pela comunidade Android para configurar aplicativos instalados e implantar essas configurações por meio de EMM (Enterprise Mobility Management). Um esquema define os recursos de DDT disponíveis para consumo pelo EMM, fornecendo as informações necessárias para apresentar à interface de usuário orientada por dados correspondente do EMM. Esse mecanismo integra a configuração remota do DDT dentro do EMM para uma operação sem interrupções e permite que a Zebra divulgue um novo esquema assim que novos recursos estiverem disponíveis para uso. O DDT e o esquema Zebra estão disponíveis na Google Play Store e devem ser usados em conjunto para aproveitar o DDT com suas configurações gerenciadas para executar operações de diagnóstico.

Requisitos

Requisitos para uso de configurações gerenciadas com o DDT:

  • Dispositivo Zebra com Android 10 ou Android 11
  • DDT com o esquema Zebra obtido na Google Play Store
  • O DDT está instalado no dispositivo e deve ser iniciado pelo menos uma vez
  • Um EMM para implantação de dispositivo
  • Servidor FTP para carregar dados de teste do DDT
    Recomendado: Para minimizar os riscos de segurança decorrentes da divulgação de credenciais de login, a Zebra recomenda o upload de dados de teste para um servidor dedicado.

Usar configurações gerenciadas

Esta seção fornece instruções gerais para usar as Configurações Gerenciadas por meio de um EMM. O layout da interface do usuário pode variar em função do EMM. No EMM, depois de adicionar o DDT ao catálogo de aplicativos do dispositivo para instalação do aplicativo, o esquema correspondente é obtido automaticamente e apresentado à interface de usuário de configuração gerenciada por EMM com base nos dados definidos. Através dessa interface do usuário, o administrador pode selecionar as configurações gerenciadas desejadas. Antes de enviar as Configurações Gerenciadas para o(s) dispositivo(s), o DDT deve ser iniciado pelo menos uma vez para aceitar essas configurações. As instruções incluem como o DDT pode ser iniciado remotamente com o OEMConfig.

Instruções gerais para usar as Configurações Gerenciadas:

  1. Adicione o Device Diagnostic Tool ao catálogo de aplicativos. No EMM, navegue ou procure pela Device Diagnostic Tool 2.5 ou superior na Google Play Store e adicione-a ao catálogo de aplicativos. Com isso, o esquema correspondente ao aplicativo é obtido e o aplicativo é disponibilizado para implantação no(s) dispositivo(s) por meio do EMM.

  2. Configure as restrições do aplicativo usando Configurações Gerenciadas conforme descrito no esquema. Consulte a seção Restrições do aplicativo a seguir, que descreve as opções disponíveis.

  3. Envie uma política que implemente o aplicativo DDT e as Configurações Gerenciadas criadas para o(s) dispositivo(s).

  4. Inicie o DDT remotamente por meio do OEMConfig – o DDT deve ser instalado na etapa 3, antes da execução desta etapa:

    • Instale o OEMConfig no dispositivo, caso não tenha sido instalado anteriormente.

    • No OEMConfig, abra a Configuração de Administração de Dispositivos.

    • Selecione Ação > Enviar XML.

    • No campo de entrada Enviar XML, insira o texto abaixo para iniciar o DDT:

      <wap-provisioningdoc><characteristic type="Intent"><parm name="Action" value="StartActivity"/><parm name="ActionName" value="android.intent.action.MAIN"/><parm name="Package" value="com.symbol.selfdiagnostics"/><parm name="Class" value="com.symbol.selfdiagnostics.activities.MainActivity"/><characteristic type="Extra"><parm name="ExtraType" value="string"/><parm name="ExtraName" value="LAUNCH_DDT"/><parm name="ExtraValue" value="1"/></characteristic></characteristic></wap-provisioningdoc>

      Nota: As quebras de linha devem ser removidas.

    • Aplique a configuração do OEMConfig no(s) dispositivo(s).

Restrições do aplicativo

Existem dois modos de operação no DDT: modo de administração e modo de usuário. Atualmente, apenas o modo de administração tem suporte com a configuração gerenciada para agendar testes recorrentes e configurar uploads de logs.

Modo de administração

No modo de administração, as restrições do aplicativo DDT são as seguintes:

  1. Plano de teste – Criar ou modificar um plano de teste que inclui Cronograma, Atividade e Entrega.
  2. Retenção de log de teste – Selecionar se os planos de teste devem ser mantidos no dispositivo ou descartados após o upload automático para o servidor.

Esta seção fornece as opções disponíveis para cada restrição de aplicativo.

Plano de teste

O plano de teste cria ou modifica um plano de teste que inclui o cronograma de teste semanal, especificando o teste para executar e configurando o servidor para receber os resultados do teste.

A retenção de log de teste especifica a ação a ser tomada para todos os arquivos de plano de teste no dispositivo depois de carregá-los para o servidor designado.
