Link-OS Multiplatform SDK
(For Windows Desktop, Android & iOS)

Release Notes

.NET Multi-platform App UI (.NET MAUI)

Build: 3.0.3271

1. Requirements

The SDK has been tested using the following operating systems:

  • 64bit Windows™ 10 & 11 desktop editions
  • Android 12, 13 & 14
  • iOS 15, 16 & 17

SDK functionality is supported on the following Print DNA printer models:

  • ZQ300 Series; ZQ300 Plus Series; ZQ500 Series; ZQ600 Series; ZQ600 Plus Series
  • ZD200 Series; ZD400 Series; ZD500 Series; ZD600 Series
  • ZT100 Series; ZT200 Series; ZT400 Series; ZT500 Series; ZT600 Series
  • ZR300 Series; ZR300 Plus Series; ZR600 Series; ZR600 Plus Series
  • ZE500 Series

Reduced functionality is available for legacy Zebralink printers and those not running Link-OS due to feature differences in the printer operating systems.

Development environment:

  • Microsoft™ .NET MAUI 8
  • Microsoft™ .NET 8
  • Microsoft™ Visual Studio 2022 17.8 or greater
  • XCode 15

2. Tips and Advice

  1. It's necessary to add the latest v3.x version of the Zebra.Printer.SDK.nupkg to the Print Station and Developer Demo projects. This can be found in the \PC-.NET\<version>\lib folder.

3. Known Issues

  1. When connected via a DriverPrinterConnection, the Open() call will not immediately fail if the printer is offline/unreachable. An error will be thrown when attempting to read/write data.
  2. DriverPrinterConnection performance/throughput may be poor. Zebra recommends using a UsbConnection where possible to avoid this issue.
  3. When using Print Station or other Developer Demos on a Windows tablet, the use of a mouse is recommended to ensure user has full access to all features.
  4. Bluetooth connections may become unstable after a failed read/write operation, or following a command which triggers a reboot. If this problem persists, it may require the user to unpair/repair the printer in the device settings.
  5. When editing settings values in the Developer Demo "Settings Demo", the focus will move to a different field each time a character is changed. This is due to an issue in MAUI at the time of the SDK release. To work around the problem, type the new value in the "Port Number" field, select it and copy it. Now delete the value in the setting field you want to change (this is where the field will change focus). Then paste the copied value into the field. Delete the value from the "Port Number" field.
