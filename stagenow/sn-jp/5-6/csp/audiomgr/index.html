SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

Audio Manager (AudioMgr) は、Zebra デバイス上の少数の特殊なオーディオ機能を制御します。たとえば、デバイス上のオーディオの再生を接続されたヘッドセットに限定するか、内蔵のデバイス スピーカに複製するか (この場合両方で再生されます) などです。AudioMgr は、デバイスが充電器に接続されているときに音声を再生するかどうかも制御します。

複製機能は、Zebra M1000 ヘッドセットを接続するためのカスタム ポートを備えた VC80x 車載コンピュータ用に開発されました。AudioMgr では、通常は M1000 スピーカのみに送信されるオーディオを VC80x の内蔵スピーカに強制的に複製することができます。これは、ヘッドセットを装着していない場合に通信できなくなることがある問題への対処です。

AudioMgr は、Zebra VC80x 車載コンピュータでのみサポートされています。

主な機能

  • オン/オフの切り替え:
    • デバイス スピーカでのヘッドセット オーディオの複製 (VC80x のみ)
    • デバイスが充電器に接続されたときのサウンド
    • 着信時のデバイスのバイブレーション
  • 着信バイブレーション設定へのユーザー アクセスを制御
  • メディア出力に最適なパスとして除外するデバイス タイプを指定

充電音のオン/オフ

バッテリ充電器を接続したときに、デバイスが音を鳴らすかどうかを制御します。

パラメータ名: ChargingSounds

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 9.3+

Android API: 26+
1 オンにする バッテリ充電器を接続したときにデバイスから音が鳴るようにします。

MX: 9.3+

Android API: 26+
2 オフにする バッテリ充電器を接続したときにデバイスから音が出ないようにします。

MX: 9.3+

Android API: 26+

オーディオ複製アクション

デバイスのオーディオが、接続された Zebra M1000 ハンドセットを介してのみ再生されるか、内蔵スピーカに複製されるかを制御します。

注: VC80x および WS50 デバイスでのみサポートされます。

パラメータ名: AudioReplicationAction

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 7.2+

Android API: 25+
1 内蔵スピーカでハンドセットのオーディオを複製する ハンドセットとデバイス スピーカでデバイスのオーディオを再生します。

MX: 7.2+

Android API: 25+
2 内蔵スピーカでハンドセットのオーディオを複製しない 接続されたハンドセットのみでデバイスのオーディオを再生します。

MX: 7.2+

Android API: 25+

着信バイブ ユーザー制御

デバイス ユーザーがデバイスでの着信のバイブレーション設定を変更できるかどうかを選択するために使用します。注: このパラメータはバイブ オプションを制御しません。デバイス ユーザーがバイブ オプションを制御できるようにします。

パラメータ名: VibrateForCallsUsage

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 11.3+
1 有効 デバイス ユーザーがデバイスでの着信のバイブ オプションを変更できるようにします。

MX: 11.3+
2 無効 デバイス ユーザーがデバイスでの着信のバイブ オプションを変更できないようにします。

MX: 11.3+

着信バイブ

デバイスでの着信のバイブ オプションを選択するために使用します。

パラメータ名: CallVibration

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 11.3+
1 振動しない 着信のバイブレーションを無効にします。

MX: 11.3+
2 常に振動する 着信のバイブレーションを有効にします。

MX: 11.3+
3 最初に振動し、徐々に鳴る 通話ごとにデバイスが一時的に振動するように設定し、徐々に着信音の音量を上げて鳴らします。

MX: 11.3+

メディア出力の除外

メディア出力に最適なパスとして、対象から除外するデバイスのリストを管理するために実行するアクションを選択するために使用します。個々のメディア出力デバイス タイプは、[最適なパスの除外デバイス] パラメータを使用して選択します。

パラメータ名: BestPathExcludeAction

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 11.3+
1 除外対象のオーディオ周辺機器を追加する 選択したデバイス タイプを除外リストに追加します。

MX: 11.3+
2 除外対象のオーディオ周辺機器を削除する 選択したデバイスを除外リストから削除します。

MX: 11.3+
3 除外対象のオーディオ周辺機器をすべて削除する 除外リストからすべてのデバイスを削除し、空にリセットします。

MX: 11.3+

メディア出力デバイス

メディア出力の最適なパスとして、対象から除外されるデバイスのリストから追加または削除するデバイスを選択するために使用します。すべてのメディア出力に単一のデバイス タイプを使用する場合、残りのデバイス タイプを除外すると、メディア出力が常に目的のデバイスにルーティングされるようになります。

次の場合に表示: [最適なパスの除外アクション] は、[デバイスの追加] または [デバイスの削除] です。

パラメータ名: BestPathExcludeDevice

オプション 名前 説明注記 ステータス 要件
2 WIRED_HEADSET_DEVICE 有線 (3.5mm) ヘッドセットをメディア出力の最適なパスとしての考慮事項から除外します。

MX: 11.3+
3 USB_DEVICE USB で接続されている周辺機器をメディア出力の最適なパスとしての考慮事項から除外します。

MX: 11.3+
4 BT_A2DP_DEVICE 高品質オーディオ配信プロファイル (A2DP) 標準を使用する Bluetooth 周辺機器をメディア出力の最適なパスとしての考慮事項から除外します。

MX: 11.3+
5 HDMI_DEVICE HDMI ケーブルで接続されているデバイスをメディア出力の最適なパスとしての考慮事項から除外します。

MX: 11.3+
