SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

Wireless Manager (WirelessMgr) は、デバイス上のすべての無線の使用を制御します。使用する無線はデバイスごとに異なり、Bluetooth、GPS、NFC、WWAN 無線などがあります。WirelessMgr でデバイス無線を制御することで、管理者は、デバイス ユーザーが、特定の無線機器の機能や、接続先のデバイスやネットワークの機能を利用不可にすることができます。これによって、デバイス ユーザーではうかがい知れないレベルのセキュリティが可能になります。

また WirelessMgr では、GPS ロケーション モードを制御することで、バッテリ寿命を保持したり、一部のデバイスでは Bluetooth 無線の状態 (アクティブ/非アクティブ) を保持したりすることもできます。

「状態」と「使用状況」を区別することが重要となります。「無線機器の状態」は、無線機器がオン (通信可能) になっているか、オフ (非アクティブ) になっているかを示すものです。「使用状況」は、無線機器がデバイスで有効になっているかどうかを判断するものです。無線機器の状態は、使用状況がオン (有効) になっている場合のみ、デバイス ユーザーまたは管理者が制御できます。

Android デバイスには多くの場合、さまざまな無線通信インタフェースが搭載されています。搭載されているインタフェースとしては、Bluetooth、ワイヤレス WAN (WWAN/携帯電話のデータ)、近距離無線通信 (NFC)、単方向 (受信専用) グローバル ポジショニング システム (GPS) などの双方向インタフェースがあります。送信可能な無線機は、不必要にオンにすると、バッテリ寿命に大きく影響する可能性があります。また一部の無線機器は、飛行機での移動中や危険な環境やセキュリティ保護された環境など、特定の状況でオフにする必要があります。

主な機能

  • デバイスの無線状態へのユーザーのアクセスを有効または無効 (オン/オフ) にする:
    • Bluetooth
    • 近距離無線通信 (NFC)
    • グローバル ポジショニング システム (GPS)
    • 無線 WAN (WWAN/携帯電話のデータ)
  • 状態を制御する (オンまたはオフにする):
    • Bluetooth 無線
    • Bluetooth スキャン
    • Wi-Fi のスキャン
    • 位置サービス (Android 8.x Oreo 以降が動作しているデバイス)
  • 内蔵または外部の Bluetooth/Wi-Fi アンテナを選択する
  • バッテリの使用量を制御するための GPS ロケーション モードを選択する:
    • 高精度
    • バッテリ節約
    • デバイスのみ

Bluetooth の有効化/無効化

デバイス ユーザーが大部分のデバイスで Bluetooth 無線の状態 (オン/オフ) を制御できるかどうかを制御します。このパラメータは、Bluetooth 無線をオンまたはオフに切り替えません。デバイス ユーザーがオン/オフを切り替える機能を制御します。

Jelly Bean、KitKat についての注記

  • 有効 (オプション 1): Bluetooth をオンにして KitKat を実行しているデバイスでは、Bluetooth 無線をアクティブな状態にします。Jelly Bean では、Bluetooth 無線をオンにしてデバイスで Bluetooth 無線を使用可能にし、デバイスの状態を変更します。
  • 無効 (オプション 2): Bluetooth がオンになっている KitKat では、Bluetooth 無線を非アクティブ状態にします。Jelly Bean では、Bluetooth 無線をオフにしてデバイスで Bluetooth 無線の使用を抑止し、デバイスの状態を変更できないようにします。

GPS ロケーション モード

バッテリ寿命を長く保つために位置情報の精度が犠牲になるような状況を考慮して、GPS ロケーション モードを選択してデバイスの位置を特定できます。このパラメータでは、Android の 3 つの位置モードを使用します。

  • 高精度モードでは、内蔵 GPS レシーバから受信した情報を Wi-Fi および携帯電話のデータで補完して、最高の精度で位置を計算します。このオプションは、最もエネルギーを消費します。

  • バッテリ節約モードでは、Wi-Fi データと携帯電話のデータのみを使用して位置を特定しますが、Google マップや同様のアプリでは動作しません。このオプションは、消費エネルギーが最も小さくなります。

  • デバイスのみモードでは GPS データのみを使用しますが、設定によっては信号が失われる可能性があります。

注: Android 7.x (Nougat) 以降を実行しているデバイスでのみサポートされます。

パラメータ名: GPSLocationMode

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 6.1+

Android API: 1+
1 高精度 GPS、Wi-Fi、および携帯電話ネットワーク情報を使用して、最高の精度でデバイスの位置を特定します。

MX: 6.1+

Android API: 1+
2 バッテリ節約 Wi-Fi と携帯電話の情報のみを使用してデバイスの位置を特定し、バッテリ持続時間を保持します。

MX: 6.1+

Android API: 1+
3 デバイスのみ GPS 無線のみを使用して位置を決定しますが、設定によっては信号が失われやすくなります。

MX: 6.1+

Android API: 1+

例

Bluetooth の状態の変更を有効にする

注: JellyBean デバイスでは、この XML で Bluetooth 無線をオンにすることによって無線の状態が変更されます。


<wap-provisioningdoc>
    <characteristic type="WirelessMgr" version="4.3" >
        <parm name="Bluetooth" value="1"/>
    </characteristic>
</wap-provisioningdoc>

Bluetooth の状態の変更を無効にする

注: JellyBean デバイスでは、この XML で Bluetooth 無線をオフにすることによって無線の状態が変更されます。


<wap-provisioningdoc>
    <characteristic type="WirelessMgr" version="4.3" >
        <parm name="Bluetooth" value="2"/>
    </characteristic>
</wap-provisioningdoc>

すべての無線をオンにする (JellyBean デバイスの場合)


<wap-provisioningdoc>
    <characteristic type="WirelessMgr" version="4.3" >
        <parm name="Bluetooth" value="1"/>
        <parm name="NFCState" value="1"/>
        <parm name="GPSState" value="1"/>
        <parm name="WWANState" value="1"/>
    </characteristic>
</wap-provisioningdoc>

すべての無線をオフにする (JellyBean デバイスの場合)


<wap-provisioningdoc>

        <parm name="Bluetooth" value="2"/>
        <parm name="NFCState" value="2"/>
        <parm name="GPSState" value="2"/>
        <parm name="WWANState" value="2"/>
    </characteristic>
</wap-provisioningdoc>

すべての無線をオンにする (KitKat デバイスの場合)


<wap-provisioningdoc>
    <characteristic type="WirelessMgr" version="4.3" >
        <parm name="BluetoothState" value="1"/>
        <parm name="NFCState" value="1"/>
        <parm name="GPSState" value="1"/>
        <parm name="WWANState" value="1"/>
    </characteristic>
</wap-provisioningdoc>

すべての無線をオフにする (KitKat デバイスの場合)


<wap-provisioningdoc>
    <characteristic type="WirelessMgr" version="4.3" >
        <parm name="BluetoothState" value="2"/>
        <parm name="NFCState" value="2"/>
        <parm name="GPSState" value="2"/>
        <parm name="WWANState" value="2"/>
    </characteristic>
</wap-provisioningdoc>

<wap-provisioningdoc>
    <characteristic type="WirelessMgr">
        <parm name="AntennaSelection" value="2" />
