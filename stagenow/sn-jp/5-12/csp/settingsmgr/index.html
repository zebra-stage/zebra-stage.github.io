SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

Settings Manager (SettingsMgr) は、Android システム設定パネルで特定のデバイス機能へのアクセスを制御します。これにより、管理者は、デバイス ユーザーが機密性の高い設定を変更、または望ましくない結果を招くことを防止できます。その他の設定パネルの機能制限については、AccessMgr を参照してください。AppMgr を使用して、設定パネルへのアクセスを完全に禁止します。

重要事項

Settings Manager は、Android 設定パネルの設定へのアクセスのみを制御します。設定自体は制御または変更しません。たとえば、Wi-Fi の状態がデバイスで "オフ" の場合 (Wi-Fi Manager CSP を使用して設定)、Settings Manager を使用して Wi-Fi 設定へのアクセスを無効にすると、ユーザーは Wi-Fi の状態を "オン" に変更して、デバイスをネットワークに接続できなくなります。

主な機能

    • 通知メッセージとアプリ通知設定

Wi-Fi アクセス

デバイス ユーザーが Android 設定パネルまたはクイック設定コントロールから Wi-Fi 設定にアクセスできるようにするかどうかを制御します。注: このパラメータは Wi-Fi 設定を変更しません。

パラメータ名: WifiSettingsUI

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (または XML からのこのパラメータがない場合) により、デバイスで変更は加えられず、以前に選択された設定が保持されます。

OSX: 4.3+

MX: 4.3+
1 有効 Android 設定パネルまたはクイック設定コントロールから、デバイス ユーザーが Wi-Fi 構成設定にアクセスして変更できるようにします。

OSX: 4.3+

MX: 4.3+
2 無効 デバイス ユーザーが、システム設定メニューまたはクイック設定コントロールから Wi-Fi 構成にアクセス、または変更を加えられないようにします。

OSX: 4.3+

MX: 4.3+

機内モード アクセス

デバイス ユーザーが Android 設定パネルおよびクイック設定パネルから [機内モード] にアクセスできるかどうか、および Android 5.x Marshmallow 以前を実行しているデバイスの [電源オフ] メニューからアクセスできるようにするかどうかを制御します。デバイスが機内モードに入ると、Bluetooth、携帯電話、Wi-Fi 無線がオフになり、ユーザーが希望する場合は Bluetooth および Wi-Fi 無線をオンに戻すことができます。電源オフ メニューから機内モードを制御するには、Power Key Manager を参照してください。

注: このパラメータを使用する Nougat デバイスでは、[電源オフ] メニューは設定できません。Power Key Manager を参照してください。

ステータス: Android 13 以降を実行しているデバイスでは、この機能には StageNow と互換性のある EMM システム、または Android デバイス ポリシー マネージャ (DPM) を使用してアクセスできます。

パラメータ名: AirplaneMode

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (または XML からのこのパラメータがない場合) では、デバイスに変更は加えられず、以前に選択した設定が保持されます。

OSX: 4.3 ～ 8.0

MX: 4.3+
1 有効 デバイス ユーザーが Android 設定パネルとクイック設定パネル、および電源オフ メニュー (サポートされているデバイス) から [機内モード] 機能にアクセスできるようにします。

OSX: 4.3 ～ 8.0

MX: 4.3+
2 無効 デバイス ユーザーが Android 設定パネルとクイック設定パネル、および電源オフ メニュー (サポートされているデバイス) から [機内モード] 機能にアクセスできないようにします。

OSX: 4.3 ～ 8.0

MX: 4.3+

エンタープライズ リセット アクセス

デバイス ユーザーが Android 設定パネルの [エンタープライズ リセット] 機能にアクセスできるようにするかどうかを制御します。エンタープライズ リセットは、デバイスを、デバイス上の /enterprise パーティションの内容によって通常決定される、エンタープライズ定義のデフォルト状態に戻すことを目的としています。

このパラメータは、エンタープライズ リセットを実行しません。デバイス ユーザーがエンタープライズ リセットを実行する機能を制御します。エンタープライズ リセットは、データ損失の原因となる場合があります。詳細については、Power Manager を参照してください。

ステータス: Android 13 以降を実行しているデバイスでは、この機能には StageNow と互換性のある EMM システム、または Android デバイス ポリシー マネージャ (DPM) を使用してアクセスできます。

パラメータ名: InvokeEnterpriseReset

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (または XML からのこのパラメータがない場合) は、デバイスでは変更は加えられず、以前に選択された設定が保持されます。

OSX: 4.2+

MX: 4.2+
1 有効 デバイス ユーザーが Android 設定パネルから [エンタープライズ リセット] 機能を呼び出すことを許可します。

OSX: 4.2+

MX: 4.2+
2 無効 デバイス ユーザーが Android 設定パネルで [エンタープライズ リセット] 機能を呼び出せないようにします。

OSX: 4.2+

MX: 4.2+
