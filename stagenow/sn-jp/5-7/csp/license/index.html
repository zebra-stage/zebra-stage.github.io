SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

ライセンス マネージャ (LicenseMgr) CSP は、デバイス上のソフトウェア ライセンスを管理し、これにより、管理者またはデバイス上で実行されているアプリが、ライセンスを必要とするソフトウェア機能の操作性を制御できます。ソフトウェア コンポーネントは、ライセンスに基づいて、使用可能、使用不可にすること、または機能制限することができます。コンポーネントにライセンスが必要な場合、有効なライセンスがないと、その機能が制限されるか、機能が使用できなくなります。ライセンスを必要としないソフトウェア コンポーネントは、この CSP の影響を受けません。

主な機能

Zebra ライセンス

  • Zebra または (レガシー) Symbol メソッドを使用して、ライセンス アクションを実行します
  • アクティベーション ID によりソフトウェア ライセンスを有効化/返却/すべて返却/更新します
  • "機能" 名やバージョンでライセンス アプリを有効化/返品/更新します
  • ライセンス ソースを選択します:
    • クラウド
    • クラウド プロキシ
    • ローカル ライセンス サーバー
    • 事前にアクティブ化されたライセンス
  • ライセンス ソースを削除します
  • オンプレミス ライセンス AID サーバーの URL とフレンドリ名を入力します
  • プロキシ経由でクラウド ベース AID サーバーの URL とフレンドリ名を入力します
  • オンプレミス機能ライセンス サーバーの URL とオプションのポート番号を入力します
  • クラウド ベースの機能ライセンス サーバーの URL とオプションのポート番号を入力します
  • Zebra ローカル ライセンス サーバー上の名前付きパーティション (グループ) からライセンス コードを取得します

Symbol ライセンス

  • デバイスにソフトウェア ライセンスを適用します
  • デバイスからソフトウェア ライセンスを取り消します
  • 標準およびカスタム機能に対してライセンス アクションを実行します
  • ライセンスを埋め込むか、参照するかを選択します
  • デバイスでライセンスを確認します

日本語サポート

ライセンス マネージャでは、日本語にローカライズされたデバイスのサポートが制限されています。

次のデバイスではサポートされていません:

  • 日本語版の Zebra ソフトウェアのライセンス (ZSL) UI (英語版 UI のみ)。
  • クライアント ライセンス アプリ機能は、日本語にローカライズされたデバイスでも英語でのみ表示されます。
  • ステージング エラー メッセージは、デバイスと StageNow クライアント アプリが日本語版にローカライズされている場合でも、英語でのみ表示されます。

以下は、StageNow ではサポートされていません。

  • プロファイル作成のための日本語入力。すべての入力は英語のみで行なう必要があります。
  • StageNow が日本語にローカライズされている場合でも、ステージング エラー メッセージは英語でのみ表示されます。

注: StageNow バーコード (およびその他の MX-XML ソース) を使用して構成された設定は、日本語にローカライズされたデバイスでサポートされています。パラメータ プロンプトは日本語で表示されますが、設定は英語で入力する必要があります。

ライセンスの取得

ソフトウェア ライセンスは、Zebra ソフトウェア ライセンス サーバーから取得されます。これらはソフトウェア ライセンス ファイルとして発行され、ライセンス機能名を使用して特定のソフトウェア コンポーネントに関連付けられます。ソフトウェア ライセンス ファイルは、関連するライセンス機能名の使用を有効にするためにのみ使用できます。1 台のコンポーネントは複数の機能を持つことができ、それぞれがライセンス機能名を個別に使用して制御されます。また、1 台のコンポーネントは、単一のライセンス機能名で制御されるすべての機能を持つことができます。

Zebra ソフトウェア ライセンス サーバーから取得したソフトウェア ライセンスは、個々のデバイス ライセンスである可能性があり、各ソフトウェア ライセンス ファイルは特定のデバイスのシリアル番号に関連付けられています。このような場合、適切なソフトウェア ライセンス ファイルを一致する各デバイスに適用する必要があります。Zebra ソフトウェア ライセンス サーバーは、エンタープライズが所有するすべてのデバイスに発行されるエンタープライズ ライセンスも提供します。このような場合、同じソフトウェア ライセンス ファイルを、そのエンタープライズが所有する各デバイスに適用できます。

ライセンスの検証

ソフトウェア ライセンス ファイルは、Zebra ソフトウェア ライセンス サーバーから取得され、ローカルまたはオンラインで保存され、LicenseMgr を通じて参照されます。ただし、サーバーは、そのようなファイルが特定のライセンス機能に一致しているかどうかを検証しません。有効なソフトウェア ライセンス ファイルが、意図した機能以外のライセンス機能に誤って適用されている場合、その機能は有効になり、エラーは発生しません。

これを防止するため、Zebra では、検証メソッド パラメータ (以下) を使用して、ライセンス機能名がソフトウェア ライセンス ファイルに含まれているものと一致していることを確認することをお勧めしています。これにより、不一致が発生した場合に結果の XML にエラーが返されます。

事前にアクティブ化されたライセンスの制限

  • 事前にアクティブ化されたライセンスは、生成された日にのみ有効です。
  • 複数の事前にアクティブ化されたライセンスが生成された場合は、生成されたものと同じ順序で適用する必要があります。
  • 事前にアクティブ化されたライセンスは、他のすべてのライセンス アクションの前に適用する必要があります。
  • 同じ構成プロファイルでアクティベーション ID と事前にアクティブ化されたライセンスのアプリケーションを組み合わせる場合は、事前にアクティブ化されたライセンスを最初に適用する必要があります。

参照値 (オプション 2) は、デバイス ファイル システムに存在する必要があるソフトウェア ライセンス ファイルが、パスとファイル名で参照されていることを示します。注: このオプションでは、リクエスト XML が送信される前にソフトウェア ライセンス ファイルをデバイス ファイル システムに転送する必要があり、デバイスでのソフトウェア ライセンス ファイルの正確なパスとファイル名を指定する必要があります。エンタープライズ リセットの後にライセンスを適用する必要がある場合は、ライセンス ファイルを /enterprise ディレクトリなどの永続的な場所に保存する必要があります。

注: ソフトウェア ライセンス ファイル (埋め込みまたは参照) の提供方法にかかわらず、提供されたソフトウェア ライセンス ファイルには、Zebra ソフトウェア ライセンス サーバーから取得した有効なコンテンツが含まれている必要があります。無効なコンテンツを含むソフトウェア ライセンス ファイルを適用しようとすると、結果の XML にエラーが発生します。また、ソフトウェア ライセンス ファイルが個々のデバイス ライセンスである場合、ソフトウェア ライセンス ファイルに埋め込まれているデバイスのシリアル番号は、そのソフトウェア ライセンス ファイルが適用されるデバイスのシリアル番号と一致する必要があります。一致しないライセンスを適用しようとすると、結果の XML にエラーが生成されます。

  • デバイス ファイル システム内の有効なソフトウェア ライセンス ファイルの有効なパスおよびファイル名である必要があります

次の場合に表示: ライセンス アクションは "適用" で、ライセンスメソッドは "参照"、ライセンス選択は "Symbol" です

パラメータ名: LicensePathAndFileName

要件:

  • MX: 4.3+

削除方式の選択

デバイスから削除するソフトウェア ライセンスを識別する、ライセンス機能名を選択する方法を制御します。

次の場合に表示: ライセンス アクションは "削除" でライセンス選択は "Symbol" です

パラメータ名: SelectMethod

オプション 名前 説明注記 ステータス 要件
1 標準 標準ライセンス機能名によりデバイスから削除されるソフトウェア ライセンスを識別することを示します。

MX: 4.3+
2 カスタム カスタム ライセンス機能名によりデバイスから削除されるソフトウェア ライセンスを識別することを示します。

MX: 4.3+

標準機能の削除

デバイスから削除するソフトウェア ライセンスを識別する目的の標準ライセンス機能名を入力するために使用されます。

次の場合に表示: 選択メソッドは "標準"、ライセンス選択は "Symbol" です

パラメータ名: SelectStandardFeatureName

オプション 名前 説明注記 ステータス 要件
SIMULDC1_0_0 SimulScan OCR/OMR SimulScan ソフトウェア コンポーネントのソフトウェア ライセンスをデバイスから削除します。

MX: +

カスタム機能の削除

デバイスから削除するソフトウェア ライセンスを識別するために、カスタム ライセンス機能名を入力するために使用されます。適切に削除するには、大文字と小文字の違い、空白文字などを含め、デバイスに適用されている元のソフトウェア ライセンス ファイルに含まれている正確なライセンス機能名を入力します。

パラメータ値入力ルール:

  • Zebra ソフトウェア ライセンス サーバーから取得したソフトウェア ライセンス ファイルに含まれる有効なカスタム ライセンス機能名を持つ文字列

次の場合に表示: 選択メソッドは "カスタム"、ライセンス選択は "Symbol" です

パラメータ名: SelectCustomFeatureName

要件:

  • MX: 4.3+

例

適用されたすべてのソフトウェア ライセンスを表示します

リクエスト XML

以下のクエリでは、デバイスに適用されているすべてのソフトウェア ライセンスに関する情報を要求します。

    
    <wap-provisioningdoc>
        <characteristic-query type="LicenseMgr"/>
    </wap-provisioningdoc>

成功 (1 つの SimulScan ライセンス)

    
    <wap-provisioningdoc>
        <characteristic type="LicenseMgr" version="4.3" >
            <characteristic type="ExistingLicense">
                <parm name="SelectCustomFeatureName" value="SIMULDC1_0_0"/>
                <parm name="CompanyName" value="CompanyName1"/>
                <parm name="LicenseType" value="serial"/>
            </characteristic>
        </characteristic>
    </wap-provisioningdoc>

追加のソフトウェア ライセンスが適用された場合は、追加情報が成功の結果 XML に返されます。ソフトウェア ライセンスが適用されていないデバイスに送信された同じリクエスト XML は、次のようなエラー結果の XML を返します。

エラー結果の XML

    
    <wap-provisioningdoc>
        <characteristic type="LicenseMgr" version="4.3" >
            <characteristic-error type="ExistingLicense" desc="Feature is not licensed" />
        </characteristic>
    </wap-provisioningdoc>

特定のソフトウェア ライセンス情報を表示します

以下のクエリは、SimulScan ソフトウェア コンポーネントのソフトウェア ライセンスにのみ適用されます。

リクエスト XML

    
    <wap-provisioningdoc>
        <characteristic type="LicenseMgr" >
            <characteristic-query type="ExistingLicense">
                 <parm name="SelectStandardFeatureName" value="SIMULDC1_0_0"/>
            </characteristic-query>
        </characteristic>
    </wap-provisioningdoc>

成功 (1 つの SimulScan ライセンス)

    
    <wap-provisioningdoc>
        <characteristic type="LicenseMgr" version="4.3" >
            <characteristic type="ExistingLicense">
                <parm name="SelectCustomFeatureName" value="SIMULDC1_0_0"/>
                <parm name="CompanyName" value="CompanyName1"/>
                <parm name="LicenseType" value="serial"/>
            </characteristic>
        </characteristic>
    </wap-provisioningdoc>

失敗

    
    <wap-provisioningdoc>
        <characteristic type="LicenseMgr" version="4.3" >
            <characteristic-error type="ExistingLicense" desc="Feature is not licensed" />
        </characteristic>
    </wap-provisioningdoc>
