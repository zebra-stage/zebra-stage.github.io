デバイス所有者の移行

StageNow 5.10

概要

このガイドでは、ユーザーがインストールしたアプリケーションやデータを失うことなく、Zebra Device Manager (ZDM) からエンタープライズ モビリティ管理 (EMM) システムにデバイスの所有権を移行する方法について説明します。

工場出荷時以外のデータの損失が問題でない場合は、工場出荷時へのリセットが、企業独自の EMM システムへの登録に備えて既存のデバイスの所有権を削除する最も簡単な方法かもしれません。

重要:

  • デバイスが以前に Zebra DNA Cloud に登録されている場合、サードパーティの EMM に登録する前に、デバイスで工場出荷時へのリセットを実行する必要があります。この操作を実行する前にデバイス データをバックアップすることをお勧めします。
  • サードパーティ EMM に登録すると、EMM によってブロックが解除されない限り、Zebra アプリを含む未知のソースのアプリをインストールできなくなります。

要件

  • Windows PC 上の StageNow 5.8 (以降)
  • Android 11 以降を実行している Zebra デバイス
  • デバイス上の MX 11.5 以降
  • 企業の EMM DPC packagename および classname

DPC とは?

企業内で Android を実行しているデバイスでは、デバイス ポリシー コントローラ (DPC) が企業の EMM とデバイス自体の間の仲介役として機能し、システム管理者によって発行されたデバイス ポリシーを強制的に実施する責任を負います。DPC は、特定のアプリのインストールや起動ができないようにし、Android のどの機能やデバイスの周辺機器を使用できるかを制御します。DPC は、デバイス上でのデータとアプリの仕事用と個人用の分離など、他の多くの役割を実行できます。

デバイスには DPC を 1 つのみ設定でき、デバイス所有者 (DO) の場合は、すべてのデバイス アクションに対する最終的な権限を持ちます。

特に明記されていない限り、「デバイス所有者」という用語は個人または企業体を指すのではなく、デバイス上で動作し、そのアクティビティを制御するソフトウェア エージェントを指します。

所有権の移行

I. ZDM を DO として設定する

ZDM から EMM のデバイス ポリシー コントローラ (DPC) に所有権を移行する最初のステップとして、ZDM がデバイス所有者であることを確認します。Zebra では、自身の DPC からのみ所有権を移行できます。

  1. StageNow で、MX 11.5 (以降) をターゲットとする新しいプロファイルを作成します。
  2. インテント CSP をプロファイルに追加します。
  3. インテントアクションパラメータから「EnrollDeviceOwner」を選択します。
  4. ZDM の詳細を入力します。
         a. パッケージ名: com.zebra.devicemanager
         b. クラス名: com.zebra.devicemanager.policy.devadmin.DevAdminReceiver
         c. バンドル データ: 何もしない
  5. バーコードを生成し 、スキャンして ZDM を DO として設定します。

サンプル XML

Zebra Device Manager をデバイス所有者として設定します。


<wap-provisioningdoc>  
<characteristic version="9.2" type="Intent"> 
    <parm name="Action" value="EnrollDeviceOwner" /> 
    <parm name="Package" value="com.symbol.devicemanager" /> 
    <parm name="Class" value=" com.zebra.devicemanager.policy.devadmin.DevAdminReceiver " /> 
 </characteristic> 
</wap-provisioningdoc>

サンプルのみを目的としています。テストと展開には機械生成された XML のみを使用することをお勧めします。

II.DO を EMM に移行する

ZDM から EMM の DCP に所有権を移行するには、セクション I と同じ手順に従い、EMM の DCP の packagename と classname を置き換え、bundledata (該当する場合) を文字列または JSON ファイルとして入力します。

  1. StageNow で、MX 11.5 (以降) をターゲットとする新しいプロファイルを作成します。
  2. インテント CSP をプロファイルに追加します。
  3. インテントアクションパラメータから 「EnrollDeviceOwner」 を選択します。
  4. EMM の DPC の詳細を入力します。
         a. パッケージ名: net.vendor.controlApp.androidwork
         b. クラス名: net.vendor.controlApp.admin.DeviceAdminAgent
         c. DOJsonData: JSON ファイル
        d.DOJsonFileData: [JSON データをフィールドに貼り付け]
  5. バーコードを生成してスキャンし、EMM の DPC を DO として設定します。
