SSM での AppMgr の使用

StageNow 5.10

概要

このガイドは、新しく、近日中に発表される Android のセキュリティ変更に準拠したデバイスで、安全にファイルおよび/またはデータを共有するための Zebra のメカニズムである Secure Storage Manager を通じてアプリケーションをインストールするための App Manager の使用について説明します。Google が要求するセキュリティ制限は、Android 11 を実行しているデバイスから開始して、段階的に導入されます。SSM は MX 11.3 (以降) と互換性があり、App Manager および File Manger で使用するプロセスは、多くの場合、管理者にわかりやすく設計されています。

このプロセスでは、直接ファイル アクセスは許可されなくなり、Zebra StageNow や OEMConfig などの信頼できるアプリのみが SSM を使用してデバイスにファイルを展開し、インストールや他のアプリとデータを共有できます。この方法でファイルを展開すると、SSM が提供する URI を介して目的のターゲットのみがファイルにアクセスできます。SSM は、Android 11 以降を実行しているすべての Zebra デバイスにプリインストールされており、デバイス データを共有および永続化する手段として /enterprise パーティションを置き換えます。

SSM 経由で共有されたデータを受信することを目的としたアプリは、URI 経由でデータを消費 できる必要があります。
このような変更は、特に Zebra デバイスを操作するために必要なわけではなく、データを共有する Android アプリに最終的に要求される新しい Android セキュリティ仕様に準拠するために必要であることに注意してください。

要件

  • Android 11 以降を実行している Zebra デバイス
  • デバイス上の MX 11.3 以降
  • Windows PC 上の StageNow 5.6 (以降)
  • URI データ消費が可能なターゲット アプリ

この操作に関連する File Manager パラメータの詳細については、File Manager のドキュメントを参照してください。

SSM を使用してアプリをインストールする

次の手順では、StageNow の使用方法に精通した管理者向けの簡単な概要を説明します。詳細な手順については、プロファイルの作成に関するドキュメントを参照してください。

SSM を使用してアプリをインストールするには、次の手順を実行します。

  1. MX 11.3 (以降) 用に作成された新しい StageNow プロファイルまたは既存の StageNow プロファイルで、File Manager と App Manager を追加します。
  2. File Manager のファイル アクションの下で、[アプリケーションのファイルを展開] を選択し、[続行] をクリックします。
  3. [ターゲット アプリケーション ファイル定義] に、次の値を入力します。com.zebra.devicemanager/persist_apk/your_apk_name.apk
    これにより、ターゲット アプリ名 (Device Manager)、オプションの target-app フォルダ (/persist_apk) および ファイル名 (your_apk_name.apk) が定義されます。
  4. [ファイルの保持] で、次のように操作します。
         a. [保持] を選択すると、エンタープライズ リセット後も .apk ファイルが保持されます。
         b. [保持しない] を選択すると、エンタープライズ リセット後に .apk ファイルを破棄します。
  5. [ソース アクセス方法] で、ソース .apk ファイルの場所を選択します。
         a. [リモートサーバー上のファイル] オプションを選択して、サーバーからソース ファイルをダウンロードします。
         これにより、サーバーの場所を入力するために [ソース URI2] フィールドが呼び出されます。 Zebra では、最も安全なものとしてこのオプションをお勧めします。
         b. デバイス上のファイルにアクセスするには、[デバイス ファイル システム内のファイル] を選択します。

         このオプションには、デバイスにコピーするファイルのソース パスとファイル名を入力します。例: /sdcard/your_apk_name.apk
         続行する準備ができたら、次の図のような画面が表示されます。
         設定を確認し、[続行] ボタンをクリックします。画像をクリックすると拡大表示され、Esc キーを押すと終了します。
  6. [App Manager アプリケーション アクション] の下で、ドロップダウン リストから [インストール] を選択します。
  7. [ファイル名とパス] に、手順 4 で選択した保持オプションに基づいて、以下のように値を入力します。
         a. [保持] オプションの場合: com.zebra.devicemanager/persist_apk/your_apk_name.apk
         b. [保持しない] オプションの場合: com.zebra.devicemanager/non_persist_apk/your_apk_name.apk
    画像をクリックすると拡大表示され、Esc キーを押すと終了します。
  8. 終了してプロファイルを保存し、バーコードを作成します。
    アプリは、バーコードをスキャンする互換性のあるデバイスに安全にインストールされています。
