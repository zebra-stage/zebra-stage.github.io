SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

ネットワーク接続マネージャ (NetworkConnectionMgr) を使用すると、管理者は携帯電話 (WWAN) および Wi-Fi (WLAN) ネットワークへのデバイス接続の設定を行うことができ、デバイスが常に最適なネットワークを使用できるようにすることができます。。NetworkConnectionMgr を使用すると、管理者はワイヤレス ネットワークのアクセス ポイントのリストを管理および展開し、優先順位でランク付けしたり、接続を改善するために設定された時間間隔で接続を確立するときに最小信号レベルを設定したりできます。

主な機能

  • デバイスの Zebra 拡張ネットワーク接続機能を有効化/無効化する
  • 携帯電話 (WWAN) アクセス ポイントに接続するための最小信号強度を設定する
  • Wi-Fi (WWAN) アクセス ポイントに接続するための最小信号強度を設定する
  • 現在の接続の品質を再評価するまで待機する時間を設定する (Wi-Fi のみ)
  • 携帯電話および/または Wi-Fi ネットワーク名の優先リストをデバイスに展開して接続する

アクセス ポイント アクション

デバイス上の携帯電話 (WWAN) および/または Wi-Fi (WLAN) ネットワーク名のリストを管理するために実行するアクションを選択します。携帯電話ネットワークは、オペレーター名、および関連するモバイル国コード (MCC) とモバイル ネットワーク コード (MNC) で識別されます。Wi-Fi ネットワークは SSID で識別されます。

パラメータ名: AccessPointAction

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 11.9+

Android API: 30+
1 追加/置換 [ネットワーク名リスト ファイル] パラメータで指定されたファイルに含まれる AP の優先リストをデバイスにコピーし、デバイス上の既存のネットワーク名を置き換えます。

MX: 11.9+

Android API: 30+
2 すべて削除 現在デバイスに保存されているすべてのネットワーク名を削除します。

MX: 11.9+

Android API: 30+

ネットワーク名

携帯電話 (WWAN) または Wi-Fi (WLAN) ネットワークの優先順位リストを含む .txt ファイルの名前を入力します。リストの最初の名前に最も高い接続優先度が与えられ、それ以降の名前は優先度が低くなります。

このファイルの内容は、デバイス上の既存の値を置き換えます。一度デバイスに保存されたファイルの内容は編集できません。リスト エントリを追加、削除、変更するには、新しいリスト ファイルを作成し、デバイスにプッシュする必要があります。

パラメータ入力値ルール:

  • アクセス ポイント名のリストを含む少なくとも 1 文字の文字列
  • 各行には、アクセス テクノロジ識別子 (「WLAN」 または「WWAN」) の後に、アクセス ポイント情報 (SSIDまたは [オペレータ名]_MCC-MNC)* をアンダースコア文字で区切って含める必要があります (以下のリストファイルの例を参照)。

リスト ファイルの例:

WLAN_SSID1
WWAN_TestNetwork_001-01
WLAN_SSID2
WWAN_TestNet2_001-02

* MCC = モバイル国コード、MNC = モバイル ネットワーク コード

次の場合に表示: アクセス ポイントのアクションは「追加/置換」

パラメータ名: networkNamesListFile

要件:

  • MX: 11.9+
  • Android API: 30+

接続機能有効/無効

優先ネットワークや接続のための最小信号強度など、ネットワーク接続マネージャの拡張機能をデバイスで使用するかどうかを制御します。

パラメータ名: networkConnectionManagerFeatures

オプション 名前 説明注記 ステータス 要件
0 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 11.9+

Android API: 30+
1 オンにする デバイスのネットワーク接続マネージャ拡張機能を有効にします。

MX: 11.9+

Android API: 30+
2 オフにする デバイスのネットワーク接続マネージャ拡張機能を無効にします。

MX: 11.9+

Android API: 30+

拡張接続パラメータ (チェックボックス)

ネットワークに接続するときに、デバイスに事前設定された接続要件 (AP 信号強度など) を設定するかどうかを選択するために使用されます。

パラメータ名: configs

オプション 名前 説明注記 ステータス 要件
0 (オフ) 事前設定された信号強度要件を選択できないようにし、デバイスの現在の設定を使用します。

MX: 11.9+

Android API: 30+
1 (オン) デバイスの既存の設定に代わって、プリセットの信号強度要件を選択できるようにします。

MX: 11.9+

Android API: 30+

WLAN の最小信号

Wi-Fi (WLAN) ネットワークに接続するための最小信号レベル (dBm) を入力します。値の範囲は、-90dBm (不良) ～ -30dBm (良好) です。値「0」(デフォルト) は無視され、現在の値は変更されません。

パラメータ入力値ルール:

  • 信号強度を示す負の数を表す文字列
  • 「0」を入力すると、現在の値が変更されない

次の場合に表示: 拡張接続パラメータ (configs) が「チェックされている」

パラメータ名: signalLevelToConnectWLAN

要件:

  • MX: 11.9+
  • Android API: 30+

WWAN の最小信号

携帯電話 (WWAN)ネットワークに接続するための最小信号レベル (dBm) を入力します。値の範囲は、-156 dBm (不良) ～ -31 dBm (良好) です。値「0」(デフォルト) は無視され、現在の値は変更されません。

パラメータ入力値ルール:

  • 信号強度を示す負の数を表す文字列
  • 「0」を入力すると、現在の値が変更されない

次の場合に表示: 拡張接続パラメータ (configs) が「チェックされている」

パラメータ名: signalLevelToConnectWWAN

要件:

  • MX: 11.9+
  • Android API: 30+

リフレッシュ間隔

現在の接続の品質を、デバイスに保存されている携帯電話および Wi-Fi アクセス ポイントのリストの品質と比較する前に待機する時間間隔 (秒単位) を入力します。値「0」(デフォルト) は無視され、現在の値は変更されません。

パラメータ入力値ルール:

  • 任意の長さの正の整数 (600 = 10 分)
  • 「0」を入力すると、現在の値が変更されない

次の場合に表示: 拡張接続パラメータ (configs) が「チェックされている」

パラメータ名: refreshInterval

要件:

  • MX: 11.9+
  • Android API: 30+
