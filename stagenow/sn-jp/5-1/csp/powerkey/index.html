SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

PowerKey Manager (PowerKeyMgr) は、デバイスの電源キーを長押しした後に表示される [電源オフ] メニューに表示される機能を制御します。デバイスによっては、電源関連の機能には、電源オフ、再起動、機内モード、タッチパネル、セーフ モード、その他の選択肢があります。一部の Zebra 製品およびデバイスは、バグ レポート機能もサポートしており、この機能は、Bug Report Manager によって制御されます。

Zebra Android デバイスでは、画面ロック タイムアウト (DevAdmin CSP によって制御) がサポートされており、画面ロックが表示され、ディスプレイ画面が再度オンになるまでにデバイス画面がオフ状態に留まる必要のある時間の長さを制御します。(画面がタイムアウトするのではなく) 電源キーを使用してデバイスの電源をオフにすると、タイムアウトに達していなくても、ディスプレイ画面が再度オンになったときに画面ロックを要求することができます。PowerKeyMgr では、この機能を有効または無効にし、デバイス ユーザーが Android の設定パネルを使用してこの機能を変更できるようにするかどうかを制御できます。

注: PowerKeyMgr は、特定のデバイスでサポートされている機能のみを制御します。PowerKeyMgr によって無効にされた機能の動作は、デバイスによって異なります。たとえば、無効になっている [機内モード] コントロールが非表示になるデバイスや [淡色表示] になるデバイスがあり、MC18、MC33、MC40、MC67 デバイスでは、通常どおり表示されますが、影響はありません。

主な機能

  • 次の電源オフ メニュー オプションを有効化/無効化します
    • 機内モード
    • タッチ パネル
    • セーフ モード
  • 自動画面ロック設定パネル オプションを有効化/無効化します
  • [電源キーで画面を瞬時にロック] 設定パネル オプションを有効化/無効化します

機内モードの表示/非表示

機内モード制御に電源オフ メニューからアクセスできるようにするかどうかを制御します。デバイスが機内モードに入ると、Bluetooth、携帯電話、Wi-Fi 無線が無効になり、ユーザーが希望する場合は Bluetooth および Wi-Fi 無線をオンに戻すことができます。

注: Android 8.x Oreo 以降を実行しているデバイスではサポートされていません。無効になっている [機内モード] コントロールの表示は、デバイスによって異なります。デバイスにより、コントロールが非表示になる場合と "淡色表示" になる場合があります。また、MC18、MC33、MC40、および MC67 デバイスでは通常通り表示されますが、効力はありません。

パラメータ名: AirPlaneMode

オプション 名前 説明注記 要件
0 変更しない この値 (または XML からのこのパラメータがない場合) により、電源オフ メニュー設定に変更は加えられません。以前の設定が保持されます。

OSX: 4.3+

OSX 8.0 以降では使用できません

1 メニュー オプションを表示する 電源オフ メニューから機内モードを制御できるようにします。

OSX: 4.3+

OSX 8.0 以降では使用できません

MX: 4.3+
2 メニュー オプションを表示しない 電源オフ メニューから機内モードを制御できないようにします (上記の注を参照)。

自動画面ロックの有効化/無効化

これは Android の自動画面ロック機能のオン/オフ スイッチで、電源キーを使用して画面をシャット オフするたびに画面ロックを強制的に有効にするために使用できます。オフにすると、画面ロック タイムアウト間隔を超えない限り、電源キーを使用してオフにした後に画面をオンにしたときに、ロック画面は表示されません。

パラメータ名: AutoScreenLockState

オプション 名前 説明注記 要件
0 変更しない この値 (または XML からのこのパラメータがない場合) により、画面ロックの設定は変更されません。以前に選択した設定がすべて保持されます。

OSX: 4.3+

MX: 4.3+
1 オンにする 電源キーを使用して画面をオフにすると常に画面を強制的にロックします。

OSX: 4.3+

MX: 4.3+
2 オフにする 画面ロック タイムアウト間隔を超えない限り、電源キーを使用してオフにしたときに画面がロックされないようにします。

OSX: 4.3+

MX: 4.3+

自動画面ロックの表示/非表示

デバイス ユーザーが Android 設定パネルのオプションを使用し、電源キーを使用してデバイス画面をオフにしたときに画面を常にロックするかどうかを決定できるようにするかどうかを制御します。自動画面ロックが有効になっている場合、電源キーを使用してオフにした後、画面をオンにすると、ロック画面が常に表示されます。

パラメータ名: AutoScreenLockOption

オプション 名前 説明注記 要件
0 変更しない この値 (または XML からのこのパラメータがない場合) により、電源オフ メニュー設定に変更は加えられません。以前の設定が保持されます。

OSX: 4.3+

MX: 4.3+

Android API: 23+
1 メニュー オプションを表示する Android 設定パネルから自動画面ロック オプションを制御できるようにします。

OSX: 4.3+

MX: 4.3+

Android API: 23+
2 メニュー オプションを表示しない Android の設定パネルから自動画面ロックオプションを制御できないようにします。

OSX: 4.3+

MX: 4.3+

Android API: 23+

セーフ モードの表示/非表示

このオプションをサポートするデバイスの電源オフ メニューでセーフ モード コントロールを使用できるようにするかどうかを制御します。セーフ モードは、通常、デバイスを正常に起動したときに通常起動する特定のアプリケーションやサービスによって妨害される可能性のあるトラブルシューティング操作を実行するために使用されます。セーフ モードで起動すると、デバイスの動作が安全ではなくなり、デバイスの生産機能に影響を与える可能性があるため、ほとんどの状況では、デバイス ユーザーがセーフ モードで起動できないようにすることが望まれます。

注: TC20/TC25 デバイスでは、Android 8.x Oreo 以降を実行している場合にのみ使用できます。

パラメータ名: SafeMode

オプション 名前 説明注記 要件
0 変更しない この値 (または XML からのこのパラメータがない場合) により、電源オフ メニューに変更は加えられません。以前の設定が保持されます。

OSX: 4.3+

MX: 4.3+
1 メニュー オプションを表示する セーフ モードを電源オフ メニューから制御できるようにします (デバイスがこのオプションをサポートしている場合)。

OSX: 4.3+

MX: 4.3+
2 メニュー オプションを表示しない 電源オフ メニューからセーフ モードを制御できないようにします (デバイスがこのオプションをサポートしている場合)。

OSX: 4.3+

MX: 4.3+
