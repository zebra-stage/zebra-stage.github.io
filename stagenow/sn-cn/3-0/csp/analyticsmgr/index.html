关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

Analytics Manager (AnalyticsMgr) 控制运行 Android 的 Zebra 设备上的数据收集。数据根据功能组以及这些功能组内的各个指标进行整理和收集。AnalyticsMgr 控制收集哪些数据指标、收集的频率、在什么条件下收集，以及收集的数据是存储在设备上还是 Zebra 云中。

使用 Analytics Manager 即表示接受 Zebra 最终用户许可协议。

Zebra 分析的核心是分析引擎，它收集有关设备及其电池、无线电和其他组件的操作和运行状况的信息。此类数据可用于检测需要维护、容易受到安全威胁破坏或者需要其他关注或预防性措施的设备，以保持最佳性能并避免故障。

主要功能

  • 启用/禁用分析引擎的数据收集
  • 启用/禁用设备用户或应用程序更改数据收集设置的能力
  • 设置各个指标的数据收集阈值
  • 控制数据是否存储在设备上还是 Zebra 云中
  • 禁用分析引擎时自动清除设备上的数据
  • 收集有关蓝牙、NFC 和 Wi-Fi 功能的使用数据
  • 收集设备坠落和其他坚固性信息
  • 收集有关自定义（用户定义）功能的数据
  • 将自动更新限制为 Wi-Fi 连接
  • 允许其他设备应用程序使用 AnalyticsMgr 云传输机制
  • 控制以下组中详细数据指标的收集：
    • 电池和 CPU 状态
    • GPS 信息
    • 系统信息
    • WLAN 性能
    • WWAN 性能
  • 按组控制数据收集的频率：
    • 从不
    • 低（不经常）
    • 中（中等频率）
    • 高（经常）
    • 自定义间隔（分钟）

清除数据

控制是否清除（擦除）分析引擎和/或 AnalyticsMgr CSP 在设备上存储的收集数据。此功能仅在分析引擎已禁用时可用。

清除已保存的数据在为特定时间段或某些使用场景下的数据收集建立基准时非常有用。例如，如果设备已脱离联系一段时间，清除“旧”数据可以避免报告包含不再相关或不再有价值的数据。同样，有时刚好在运行某种测试之前清除设备上的数据也非常有用，它可以确保只捕获该测试的数据。

清除数据对于存储在云中的数据没有影响。

在以下条件下显示：“分析操作”设置为“禁用”

选项 名称 说明需要

组操作

控制是否配置分析引擎对所有指标组或在一个或多个单个组进行数据收集。对所有组进行配置时，仅可配置数据收集的频率。选择“配置单个组的级别”以启用各个指标的细粒度数据收集。当选择单个组名称时，该组的可配置指标将变得可用。

指标组分别配置，一个组的设置对其他组没有影响。例如，一个实例中的 Analytics Manager 可配置为启用 WWAN 性能指标的收集，同时使用“不更改”命令指示其他组保留以前配置的设置。在以后的某个时间，可要求 CSP 收集提供了“不更改”命令的所有其他组的 GPS 性能指标。这种情况将允许收集 WWAN 和 GPS 性能指标。除非明确更改，否则将保留所有以前配置的设置。

组名称

选择用于收集详细指标的组名称。分析引擎可以收集下列组名称标识的各个功能组的详细指标。

当选择单个组进行数据收集时，可收集数据的指标列表将变得可用。可以打开或关闭每个指标的数据收集，可以为每个指标分配其自己的条件（或阈值），确定是否按照为该组选择的频率收集该指标的数据。例如，一个组织可能只希望在给定电池充电周期数达到对于最大电池性能最优的次数后开始收集有关数据电池的数据。在达到后，将按照为该功能组选择的间隔开始该电池的数据收集。

为单个组选择的频率应用于组中的所有指标。例如，如果为智能电池选择 30 分钟的间隔，将每隔 30 分钟为智能电池组中每个启用的指标收集一次数据，前提是该指标已达到它的阈值（如有）。

确定是始终为指标收集数据（无条件）还是仅在满足用户定义的阈值（有条件）时收集数据。在任何一种情况下，都会按照为列有该指标的组名称选择的频率收集数据。在设置有阈值时，仅在满足阈值后按照组频率收集数据。

例如，如果一个组织希望仅在电池超过最佳的充电周期次数后开始收集有关设备电池的数据，该数量会在“大于或等于阈值”条件下输入。一旦达到该数量，将开始按照为该功能组选择的频率收集有关电池的数据。

为单个组选择的频率应用于组中的所有指标。例如，如果为智能电池选择 30 分钟的间隔，将每隔 30 分钟为智能电池组中每个启用的指标收集一次数据，前提是该指标已达到它的阈值（如有）。

  • AUTHENTICATING - 正在尝试使用 BSS/SSID 进行身份验证。当 wpa_supplicant 找到适合的 BSS，并且驱动程序配置为相应地进行身份验证时，将进入此状态。
  • COMPLETED - 所有身份验证已完成
  • DISCONNECTED - 客户端未关联，但是可能开始寻找接入点
  • DORMANT - 一种特定于 Android 系统的状态；当客户端发出明确的 DISCONNECT 命令时，将报告此状态
  • FOUR_WAY_HANDSHAKE - WPA 4 路密钥握手正在进行中
  • GROUP_HANDSHAKE - WPA 组密钥握手正在进行中
  • INACTIVE - 不活动状态（wpa_supplicant 已禁用）
  • INTERFACE_DISABLED - 网络接口已禁用
  • INVALID - 通常绝不应会看到的伪状态
  • SCANNING - 正在扫描查找网络
  • UNINITIALIZED - 未连接到 wpa_supplicant
