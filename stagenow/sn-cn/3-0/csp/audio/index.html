概述

Audio Volume UI Manager (AudioVolUIMgr) 可以添加、删除和替换音频配置文件，并且可以选择要在设备上生效的音频配置文件。它还允许管理员修改“出厂预设”音频配置文件（不能删除），以及直接控制设备的“静音”和“振动”状态，它们独立于所有音频流和音频配置文件。

许多 Zebra Android 设备都包含一个 Zebra 音量控制 (ZVC)，该 UI 组件使设备用户可以轻松调节设备中各种音频流的音量。音频流是一个通道，通过它可以生成音频并输出到扬声器或耳机。Android 设备支持的音频流数量一般是固定的，但是可为其生成音频的可能用途数量在理论上是无限的。因此，特定音频流可能用于预期之外的用途。

注意：Zebra 音量控制组件包含在运行 KitKat 及更高版本的大多数 Zebra Android 设备上。除了在运行 Lollipop 的设备上必须手动启用之外，所有设备默认情况下都会启用它。AudioVolUIMgr 仅在包括 ZVC 的设备上受支持。

标准 Android 音频音量控制 UI 标识可按照为每个音频流预定义的固定方式控制的每个音频流。此类标识可能未准确反映使用给定音频流的用途。这可能产生不太理想的用户体验，因为设备用户可能会更改音频流的音量，而没有实现它对其他系统的影响。

ZVC 允许配置向设备用户标识音频流的方式，从而允许自定义用户体验。此外，ZVC 还通过允许配置给定音频流的最小值、最大值和预设值，提供了对用户体验的更多控制。例如，可以将一个音频流的最小音量设置为大于零的某个值，阻止设备用户完全关闭音量并因此可能错过在该音频流上执行的重要音频通知。

ZVC 还提供了对音频配置文件的支持，音频配置文件定义了可能使用一个或多个音频流的不同情形或使用案例。每个音频配置文件可以为某些或所有可用的音频流配置 ZVC 的行为。通过选择一个定义的音频配置文件，可在一个操作中为多个音频流配置 ZVC。例如，可能有一个音频配置文件用于在安静的办公室工作，另一个音频配置文件用于在嘈杂的仓库中工作。

ZVC 可通过按下设备上的音量控制按钮（可用时）或从 Android 设置面板中选择声音 -> 音量屏幕来调用。ZVC 所呈现的 UI 基于使用 AudioVolUIMgr 应用到 ZVC 的最近配置，特别是定义的音频配置文件和当前选择的音频配置文件的集合。

支持以下音频流：

  • STREAM_MUSIC - 通常用于音乐播放
  • STREAM_RING - 通常用于通知呼入呼叫
  • STREAM_NOTIFICATION - 通常用于其他通知。
  • STREAM_SYSTEM - 通过用于系统声音
  • STREAM_ALARM - 通常用于警报
  • STREAM_VOICE_CALL - 通常用于电话呼叫
  • 解码蜂鸣声虚拟音量标度 - 这一特殊的音频流为 Zebra 设备上的条码扫描生成声音通知。

可定义音频配置文件，为以上任何或所有的音频流配置以下设置：

  • 标签 - ZVC 显示的文本标签，用来识别音频流。
  • 图标 - ZVC 显示的图标，用来识别音频流。
  • 可见性 - 控制 ZVC 是否允许设备用户查看和修改音频流的音量设置。例如，这可能用于阻止设备用户更改用于呼入呼叫通知的音频流的音量。

音频配置文件还可以配置四个音频模式中每个音频模式的音量大小：扬声器、接收器、有线耳机和蓝牙耳机。指定的模式根据音频流定向到的物理输出路径来确定。对于每种模式，可以指定以下音量级别。

  • 最小音量级别 - 这会导致 ZVC 阻止设备用户将音频流的音量更改为低于指定值的值。例如，这可用于防止在嘈杂环境中将用于通知的音频流设置得太低。
  • 最大音量级别 - 这会导致 ZVC 阻止设备用户将音频流的音量更改为高于指定值的值。例如，这可用于防止用户在安静环境中将用于音乐播放的音频流音量设置的太高。
  • 预设音量级别 - 这会导致 ZVC 在此音频配置文件选定为当前音频配置文件时将音频流的音量设置为指定值。

注意：Zebra 音量控制组件包含在运行 KitKat 及更高版本的大多数 Zebra Android 设备上。除了在运行 Lollipop 的设备上必须手动启用之外，所有设备默认情况下都会启用它。AudioVolUIMgr 仅在包括 ZVC 的设备上受支持。

主要功能

  • 创建、删除和修改音频配置文件
  • 将音频配置文件设置为当前音频配置文件
  • 将出厂预设音频配置文件设置为当前音频配置文件
  • 更改要作为当前音频配置文件的预设音量级别的音量级别
  • 控制设备的全局静音和振动状态

静音/振动状态

用于使设备静音和取消静音以及使设备振动。

参数名称：MuteVibrateState

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）会导致不更改设备设置；之前选择的任何设置将被保留。

OSX：4.4+

MX：4.4+
1 静音 - 关闭声音 启用设备上的全局静音状态，使所有音频流的输出静音。

OSX：4.4+

MX：4.4+
2 振动 - 关闭声音并打开振动 启用设备上的全局静音和振动状态，使所有音频流的输出静音，并且启用触觉反馈（振动）。

OSX：4.4+

MX：4.4+
3 取消静音 - 打开声音 启用要禁用的设备上的全局静音和振动状态，使所有音频流的输出静音，并且禁用触觉反馈（振动）。

OSX：4.4+

设置音频音量

用于指定受音频配置文件影响的所有音频流的当前音量级别是否应设置为该音频配置文件的预设音量级别。

在以下条件下显示：当前活动配置文件操作是“将音频 UI 配置文件设置为当前配置文件”

参数名称：SetCurrentProfileOption

选项 名称 说明需要
1 不调整音量级别 导致在新的当前音频配置文件中为所有音频流指定的预设音量级别被忽略。如果音频流的当前音量级别小于最小值，它将被更改为最小值，如果音频流的当前音量级别大于最小值，则会更改为最大值。否则，音频流的当前音量级别保持不变。

OSX：4.4+

MX：4.4+
2 调整音量级别 导致在新音频配置文件中为所有音频流指定的预设音量级别成为这些音频流的当前音量级别。此选项确保特定的一组音量在设置新的当前音频配置文件后生效。应该慎用该选项，因为它可能对当前或将来的音频输出有显著的影响。

OSX：4.4+

MX：4.4+

音乐扬声器音量

用于设置在设备处于免提模式时通常用于音乐播放的音频流的最小、最大和预设音量级别。用户无法将音量降低或提高到设置的最小/最大范围之外。如果在设置该范围时音乐音量级别在此范围之外，则根据音量是低于还是高于此音量范围，该音频流的音量将等于最小或最大音量级别。

预设音量级别是必须大于或等于最小音量级别并且小于或等于最大音量级别的第三个音量级别。如果预设音量在最小/最大范围之外，则此值将等于最小或最大音量级别，具体取决于音量级别是低于还是高于此音量范围。这将由 SetCurrentProfileOption 功能用来将通常用于音乐播放的音频流音量调节为此设置的级别。

指定空值（零长度）（或 XML 中缺少此参数）将导致不更改音量级别。

参数值输入规则：

  • 以此顺序包含最小音量、最大音量和预设音量并且使用逗号分隔的字符串。例如，“1,5,3”
  • 音量级别必须为小于 256 的正整数。

在以下条件下显示：音频 UI 配置文件操作是“添加音频 UI 配置文件”

参数名称：STREAM_MUSIC_SPK_LEVEL

需要：

  • OSX：4.4+
  • MX：4.4+
