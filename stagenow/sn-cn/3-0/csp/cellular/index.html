关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

Cellular Manager (CellularMgr) 控制使用设备蜂窝数据连接的方式。此 CSP 仅在配备无线广域网 (WWAN) 适配器且能够访问蜂窝数据网络的 Zebra Android 设备上受支持。尝试在不受支持的设备上使用 CellularMgr 将在结果 XML　文档中返回错误。

CellularMgr 提供控制功能的状态（活动/不活动）以及使用（已启用/已禁用）的能力。一个选项的“状态”可被视为“启用/关闭”开关，程序或设备用户可以使用此功能来按需要设置活动或不活动状态。“使用”一词就像是开关上的盖子，将它锁定在当前位置。当选项的“使用”设置为“已禁用”时，表示盖子被关闭，用户将无法更改开关的位置。启用“使用”就好比抬起盖子，使用户能够将蜂窝功能设置为活动或不活动状态。

主要功能

  • 启用/禁用（控制使用）：
    • 数据漫游
    • 后台数据
    • SIM 卡插槽
  • 打开/关闭（控制状态）：
    • 数据漫游
    • 后台数据
  • 选择 SIM 卡插槽
  • 输入自定义的蜂窝数据限制和警告阈值

SIM 卡槽选择启用/禁用

控制设备用户是否可以使用“选择 SIM 卡槽”参数更改选定的 SIM 卡槽。当禁用时，设备用户无法选择 SIM 卡槽。仅在特定的 Zebra 设备上受支持。

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会更改用户对 SIM 卡槽选择的控制；任何以前选择的设置都将保留。

MX：6.2+
1 启用 使设备用户可以控制使用哪个 SIM 卡槽。

MX：6.2+
2 禁用 阻止用户更改选定的 SIM 卡槽。

MX：6.2+

数据漫游启用/禁用

用于控制设备用户是否具有设备上的“数据漫游”选项的访问权限。当禁用时，无法通过编程方式或通过设备用户打开或关闭数据漫游。

数据漫游选项决定设备是否可以通过活动 SIM 卡所属移动运营商提供的“本地”网络之外的蜂窝数据网络进行通信。必须根据外部运营商征收的漫游费来权衡漫游时通过“外部”网络进行通信的便利性，漫游费用可能比较高。

参数名称：DataRoamingUsage

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会更改数据漫游选项的使用；任何以前选择的设置都将保留。

OSX：4.4+

MX：4.4+
1 启用 允许在设备上使用数据漫游，以及通过编程方式或设备用户控制它的状态（活动/不活动）。

OSX：4.4+

MX：4.4+
2 禁用 禁用设备上的数据漫游，阻止通过编程方式或设备用户控制其状态（活动/不活动）。

OSX：4.4+

MX：4.4+

蜂窝数据限制启用/禁用

在将来实施此功能。

控制是否可以对设备上的蜂窝数据使用施加限制。当启用时，设备用户或在设备上运行的应用程序可以访问能够打开和关闭数据限制以及控制限制值（每月多少 MB）和达到限制时显示的警告消息的其他参数。如果此参数已禁用并且在设备上启用了蜂窝数据，则无法限制蜂窝网络上的数据使用量，应用程序或设备用户无法访问所有相关的参数。

参数名称：CellularDataLimitUsage

选项 名称 说明需要

MX：6.3+
1 启用 允许对蜂窝数据使用量设置限制，以及通过编程方式或由设备用户控制限制的状态（活动/不活动）。

MX：6.3+
2 禁用 阻止在设备上设置蜂窝数据使用量的限制。

MX：6.3+

蜂窝数据限制开/关

这是用于限制设备上的蜂窝数据（每月 MB）的开/关键，以及应用 Android 默认值或自定义限制的选择器。当已启用并且使用 Android 默认限制时，该限制将是 5GB 与“蜂窝数据警告值”参数中指定值 1.2 倍之间的较大值，最大为 1TB。如果选择了自定义限制，该限制将是“蜂窝数据限制值”参数中指定的值。达到限制后，将终止蜂窝数据连接，直到新月份开始或限制值提升。如果没有使用“蜂窝数据限制值”参数指定值，将忽略此参数，以前选择的蜂窝数据限制状态将被保留。

在以下条件下显示：蜂窝数据状态设置为“启用”

选项 名称 说明需要

MX：6.3+
1 启用并设置默认限制 将蜂窝数据使用量限制为 Android 默认值。

MX：6.3+
2 启用并设置自定义限制 将蜂窝数据使用量限制为“蜂窝数据限制值”中设置的数量。

MX：6.3+

MX：6.3+

蜂窝数据限制值

用于指定设备的蜂窝数据使用量限制（每月 MB）。达到限制后，将终止蜂窝数据连接，直到新月份开始或限制值提升。

参数值输入规则：

  • 5-1048576 之间的整数 (5MB-1TB)，与每月数据用量限制相对应。

在以下条件下显示：蜂窝数据状态设置为“启用”，蜂窝数据限制状态设置为“启用并设置自定义限制”

参数名称：CellularDataLimitValue

需要：

  • MX：6.3+

后台数据启用/禁用

用于控制设备用户是否具有设备上的“后台数据”选项的访问权限。当禁用时，无法通过编程方式或通过设备用户打开或关闭后台数据。

后台数据选项确定是否允许不在前台的应用程序维持通过蜂窝网络进行的数据连接（如果它们能够这么做）。例如，某些电子邮件应用程序在其他应用程序处于活动状态或关闭设备屏幕时继续下载邮件。在规划蜂窝带宽利用率时必须考虑在“要求数据时”数据已经就绪的便利性，并且要权衡数据超龄的可能性以及漫游成本。

参数名称：BackgroundDataUsage

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会更改 CellularMgr 设置；任何以前选择的设置都将保留。

OSX：4.4+

MX：4.4+
1 启用 允许在设备上使用后台数据，以及通过编程方式或设备用户控制它的状态（活动/不活动）。

OSX：4.4+

MX：4.4+
2 禁用 禁止在设备上使用后台数据，阻止通过编程方式或设备用户控制其状态（活动/不活动）。

OSX：4.4+

MX：4.4+

后台数据阻止开/关

这是阻止后台数据通信的开/关键。设置为“开”时，仅当应用程序处于前台时允许其使用蜂窝数据连接，在其他所有时间被阻止。设置为“关”时，所有能够在后台运行的应用程序只要在运行，无论是否在前台，都允许它们维持数据连接。

在以下条件下显示：后台数据使用设置为“启用”

选项 名称 说明需要
0 不更改 此值（或 XML 中缺少此参数）导致不会更改 CellularMgr 设置；任何以前选择的设置都将保留。

OSX：1.3+

MX：4.3+
1 打开 可阻止能够在后台运行的应用程序在应用程序不在前台时使用蜂窝数据连接。

OSX：1.3+

MX：4.3+
2 关闭 允许能够在后台运行的应用程序无论是在前台还是在后台都可以使用蜂窝数据连接。

OSX：1.3+

MX：4.3+
