关于 SmartDocs


“SmartDocs”栏可以自定义此页面，仅显示在特定 Zebra 设备上存在的功能。

  1. 使用下拉列表选择设备上的 OSX、MX 和/或 Android 版本，以及或者设备型号，它突出显示了设备中支持的功能。根据所做的选择，不受支持的功能会变灰。
  2. SmartDocs Nav 在查看单个 CSP 的功能时发挥作用。要使用它，请单击一个 CSP（例如“App Manager”），然后在 SmartDocs 栏中进行至少一项选择。不支持的功能将变为灰色。单击前进和后退 SmartNav 按钮可滚动到该 CSP 下一个或上一个支持的功能（SmartNav 在功能矩阵上不受支持）。
  3. 要清除所有选择，请按 SmartDocs 按钮。

在 Android“设置”面板或通过 ADB、EMDK 或 MX CSP 查询设备，可以找到设备的 OSX、MX 和 Android 版本信息。更多信息。

  • OSX 层提供了对基本操作系统的扩展，以实施 Android 未提供的功能。根 OSX 版本号始终与它扩展的 Android 版本的根号码匹配。更多信息。
  • MX 管理系统为设备上的特权和无特权 Android API 提供了一个统一的接口。某些功能需要对 MX 和 OSX 版本进行具体的配对。更多信息。
  • Android 版本有时足以确定功能集。其他情况则需要一个特定的 MX 版本加上 Android 的最低版本。
  • 设备型号可指明在设备上提供 MX 功能的哪个子集。

概述

Status Manager (StatusMgr) 提供了一种设置在 MX 处理 XML 流时显示通知的机制。

主要功能

  • 添加、删除或更换通知
  • 指定通知的页眉、页脚和/或正文
  • 指定要发送有关通知的 Android 操作和/或类
  • 指定与通知相关的 MIME 类型
  • 指定一个标识通知来源的字符串
  • 应用一个过滤器和/或 ID 以减少通知的显示
  • 在以下情况下显示通知：
    • 进程已完成
    • 进程已达到指定的完成百分比

请求操作

用于指定要为状态请求执行的操作。

参数名称：RequestAction

选项 名称 说明需要
1 添加/替换状态请求 添加新的状态请求，或将现有状态请求替换为当前状态请求。

MX：5.0+
2 删除状态请求 删除指定的状态请求。

MX：5.0+
3 通知状态 导致显示通知状态。

MX：5.0+

通知操作

用于指定要为状态请求采取的通知操作。

参数名称：NotifyAction

选项 名称 说明需要
0 无操作 此值（或 XML 中缺少此参数）导致不会更改设置；任何以前选择的设置都将保留。

MX：5.0+
1 通知状态 执行通知操作。

MX：5.0+

请求状态过滤器

用于为请求的状态指定一个可选过滤器。

在以下条件下显示：请求操作不是“无操作”

参数名称：RequestFilter

选项 名称 说明需要
0 请求所有状态

MX：+
1 请求完成状态

MX：+

请求状态过滤器

用于为请求的状态指定一个可选 ID 号码。

参数值输入规则：

  • 至少 1 位的整数

在以下条件下显示：请求操作既不是“无操作”也不是“通知状态”

参数名称：RequestFilter

需要：

  • MX：5.0+
