SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

Enterprise Keyboard Manager (EnterpriseKeyboard) は、エンタープライズ キーボードの設定を制御します。エンタープライズ キーボードは、Android を実行している Zebra デバイスにデータを手動で入力する効率的な手段を提供する Zebra のソフト入力デバイスです。エンタープライズ キーボードは、標準の Android キーボードをベースにしており、プログラム可能なキーを追加して、言語とキー レイアウトをすばやく切り替え、データをアプリケーションに直接スキャンできます。

エンタープライズ キーボードはまた、スペルの提案や修正に Android の個人辞書を使用します。また、お客様やパートナーは、キーボード入力の速度と精度を向上させるために、業界固有の用語をデバイスに入力することができます。個人辞書は、手動またはファイルから入力でき、Zebra の モビリティ拡張機能 (MX) および StageNow ツールを使用して大規模展開できます。個人辞書の詳細をご覧ください。

『Enterprise Keyboard User Guide』も参照してください。

主な機能

有効化/無効化:

  • 自動大文字化
  • 自動修正
  • 不適切な表現のブロック

  • 英数字、数字、スキャン、シンボル、およびマイク入力のタブの表示
  • 音声入力キーの表示 (マイク アイコン)
  • "フリック" を使用して、シフト/長押し文字を入力
  • セカンダリ スクリーンのソフト入力パネルの高さと幅を制御
  • 言語の選択:
    • Android システムに組み込み
    • エンタープライズ キーボードに組み込み
    • カスタム

独自の機能:

  • バーコードのダイレクト スキャン
  • キーボードのレイアウトを切り替えるには、スワイプ、タブまたはキーあるいはその両方を使用します
  • キーボードが呼び出されるたびに表示されるように、優先タブを設定します
  • 計算機、電話、その他のレイアウトを簡単に切り替えることができます
  • シングル タップで言語を切り替えます
  • キーを再マップしてカスタム レイアウトを作成します
  • 室内でも屋外でも簡単に読み取れるカラー スキーム
  • 最適な画面使用のために候補バーを自動的に非表示にします

日本語サポート

Enterprise Keyboard Manager では、日本語にローカライズされたデバイスのサポートが制限されています。

次のデバイスではサポートされていません:

  • 日本語エンタープライズ キーボード (EKB) 設定 UI (英語 UI のみ)。
  • EKB の日本語は、日本語化されたデバイスであっても、キーボード出力でサポートされていません。
  • ステージング エラー メッセージは、デバイスと StageNow クライアント アプリが日本語版にローカライズされている場合でも、英語でのみ表示されます。

以下は、StageNow ではサポートされていません。

  • プロファイル作成のための日本語入力。すべての入力は英語のみで行なう必要があります。
  • StageNow が日本語にローカライズされている場合でも、ステージング エラー メッセージは英語でのみ表示されます。

注: StageNow バーコード (およびその他の MX-XML ソース) を使用して構成された設定は、日本語にローカライズされたデバイスでサポートされています。パラメータ プロンプトは日本語で表示されますが、設定は英語で入力する必要があります。

Tab キー ナビゲーションについて

エンタープライズ キーボードの Tab キーは、テキスト フィールドまたは UI コンポーネントから次のテキスト フィールドまたは UI コンポーネントにカーソルまたは UI フォーカスを移動するように設計されています。Zebra では、Tab キーを使用してこのように操作するようにアプリを作成することをお勧めしています。MX 6.2 および/または Enterprise Keyboard 1.3 より前のバージョンのデバイスでは、Tab キーを押してもカーソルが期待どおりに移動しない場合があります。ネイティブアプリのナビゲーションのヘルプについては、Android 開発者向けドキュメントの「Android Tab キー ナビゲーション」セクションを参照してください。

ナビゲーション モード

キーボード レイアウト間を移動する方法を制御します。[タブ] モードでは、キー レイアウトを任意の方向にスワイプするか、目的のタブを直接タップして、レイアウトを切り替えることができます。タブ バーは、次の単語の候補が表示されているとき (有効な場合) を除き、常に表示されます。[キー] モードでは、デフォルトの Android キーボードと同様に、特定のキーを使用してレイアウトを切り替えることができます。ただし、このモードでは、特定のキーが再マッピングされず、以前の割り当てがオーバーライドされ、それらのキーがそれぞれのキーボード レイアウトにマッピングされます。下の重要なメモを参照してください。

再マッピングに関するメモ:

  • [キー] モードの場合:
    • 数字 P3 および P4、英文字 P1、および記号 P1 キーはキーボード レイアウトにマッピングされています。これらのキーのカスタム再マッピングは無効になっています。
    • 数値 P1 と P2 は、再マップ可能なままです。
  • [英文字] タブを有効にすると、数字 P4 キーの再マッピングが無効になります。
  • [数値] タブを有効にすると、英数字 P1 キーの再マッピングが無効になります。
  • [記号] タブを有効にすると、数字 P3 キーの再マッピングが無効になります。
  • [記号]、[英文字]、または [数字] タブが無効になっている場合は、対応するキーの最後に保存されたキーの再マッピング割り当てが復元されます (ある場合)。

動作に関するメモ:

  • タブ バーまたは候補バーのいずれかが有効になっている場合は、両方が表示されます。
  • [次の単語の候補] バーと [音声入力キー] の両方が無効になっている場合、タブ バーは表示されません。
  • GMS デバイスでは、音声が有効になっていて、次の単語の候補機能が無効になっている場合、単語が候補として表示されるときに、タブ バーに空白の候補バーが表示されます。
  • TYPE_TEXT_FLAG_NO_SUGGESTIONS 入力タイプが有効な場合は、タブ バーも候補バーも表示されません。Android 入力タイプについて。
