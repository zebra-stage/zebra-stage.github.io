SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

ファイル マネージャ (FileMgr) を使用すると、アプリケーションでデバイス上のファイルを管理できます。たとえば、FileMgr では、アプリがサーバーからデバイスにファイルをダウンロードしたり、デバイスのある領域から別の領域にファイルまたはフォルダをコピーしたり、デバイス上のファイルを削除したりできます。ファイル マネージャは、XML ドキュメントに埋め込まれたファイルを抽出し、基本認証ありまたはなしでインターネットからダウンロードすることもできます。

主な機能

  • ローカルまたはリモート コンピュータからデバイスにファイルをコピーします
  • XML オブジェクトに埋め込まれたデータからデバイスにファイルをコピーします
  • ファイルをデバイス上のある場所から別の場所にコピーします
  • デバイスからファイルを削除します
  • 複数のファイルを含むフォルダまたはアーカイブ ファイルをデバイスにコピーします
  • HTTP、HTTPS、FTP、または FTPS (FTP over TLS/SSL) を使用してサーバーからファイルをダウンロードします
  • FTP または FTPS を使用してデバイスからサーバーにファイルをアップロードします
  • URI に組み込まれている場合は、HTTP サーバーで基本認証を実行します
  • StageNow で生成された ".archive" ファイルをダウンロードして、HTTP または HTTPS を使用して処理します
  • Wi-Fi またはモバイル ネットワークの接続が切断された後、操作を再試行または再開します
  • 接続が使用可能になるのを待ってダウンロードを開始します
  • コピー時または移動時に重複ファイルが発生した場合の動作を制御します
  • 複数のファイルをコピーまたは移動するときの並べ替え順序を選択します
  • アップロードされたファイルの命名パターンを指定します
  • アップロード後にファイルを削除します
  • ファイル アクセスを許可するターゲット アプリケーションを指定します
  • エンタープライズ リセット後にデバイス上のファイルを保持するかどうかを選択します

重要:

  • 範囲指定されたストレージは、次のようなデバイス フォルダへのアクセスをブロックする場合があります。
    • /enterprise
    • /sdcard/Android/data
    • /storage/emulated/0/Download
  • ファイル マネージャは匿名 FTP をサポートしていません。

ファイル アクション

デバイス上のファイルまたはフォルダ (空の場合) に対して実行するアクションを選択するために使用されます。

SSM の使用

Android 11 から段階的に開始される Google が義務付けたセキュリティ制限に備えて、Zebra は、Android のセキュリティ要件に準拠して、元のアプリからターゲット アプリにファイルやデータを安全に共有するメカニズムである、Secure Storage Manager (SSM) を実装しています。このプロセスでは、直接ファイル アクセスは許可されなくなり、SSM を介して共有されるデータを受信するように意図されているアプリに対して、軽微な変更が必要になります。

SSM は、MX 11.3 以降で提供される File Manger CSP と互換性があり、そのプロセスは、多くのファイルおよび/またはデータ共有シナリオにおいて管理者に透過的に設計されています。ただし、SSM を介して共有されるファイルおよび/またはデータの受信者として指定されたアプリは、URI を介してファイルにアクセスできる必要があります。このような変更は、Zebra デバイスでの作業には特に必要ありませんが、データを共有するすべての Android デバイスで最終的に必要となる Android のセキュリティ要件に準拠するために必要になります。

SSM は、Android 11 以降を実行しているすべての Zebra デバイスにプリインストールされています。

必須パラメータ

ファイル アクション オプション 10 (以下の表) には、次の追加パラメータが必要です (または、示されているようにオプションで連携します)。

  • TargetApplicationAndFileName (必須) は、ターゲット アプリ名、オプションの target-app フォルダ、およびファイル名を次の形式で組み合わせた「ターゲット アプリケーション ファイルの定義」を提供します。
    • com.targetApp.package.name/optionalAppFolderName/file.name (下の注を参照)
  • TargetApplicationSignature (必須) は、「ターゲット」アプリ (ファイルが共有されている) が純正であることを確認します
  • SourceAccessMethod2 (必須) は、デバイスにコピー/転送される共有ファイルの場所を提供します。
    セキュリティを強化するために、Zebra では、[リモート サーバー上のファイル] オプションを選択することを推奨しています (下の「SourceURI2」を参照)
  • SourceURI2 は、共有するファイルのサーバーの場所を提供します ([リモート サーバー上のファイル] オプションを使用する場合に必要です)
  • PersistFile (オプション) を使用すると、エンタープライズ リセット後にコピーされたデータやファイルを安全に保持できます

ターゲット アプリケーション ファイル定義について

「ターゲット アプリケーション ファイル定義」パラメータ (「TargetApplicationAndFileName」) は、アプリケーションのパッケージ名 (上の例の com.targetApp.package.name) と共有するファイルの名前 (file.name) の組み合わせを受け入れます。このファイルには、デバイスにインストールするためのアプリ インストーラ パッケージ (.apk)、アプリ設定を構成するための構成ファイル、またはその他のタイプのファイルを指定できます。

定義 (/optionalAppFolderName/) の中央セクションは、ターゲット アプリのファイルを整理するために使用できる、ターゲット アプリの下の [オプション] フォルダ (またはパス) です (または、まだ存在しない場合はそのフォルダに作成されます)。このストレージ領域にアクセスできるのは、指定されたターゲット アプリのみで、このアプリは、デバイス上の別のアプリまたはターゲット アプリ自体にすることができます。

SSM を介して共有されるデータを受信するアプリは、URI を介してデータを消費できる必要があります。

Android コンテンツ プロバイダのドキュメントも参照してください

ターゲット ファイル名のパターン

リモート システムにアップロードされたファイルのファイル名を生成するためのパターンを入力するために (オプションとして) 使用されます。

パラメータ値入力ルール:

  • ファイル命名パターンを含む 0 ～ 255 文字の文字列
  • モデル、シリアル番号、日時の定数を受け入れます
  • デフォルトのパターンは "SerialNo_FileName" です。
  • パターンが入力されていないか無効な場合は、デフォルトのパターンが使用されます

例: 指定されたパターンの "model-serial-date-time" により、ファイル名は次のようになります。
「TC57-12345678901234-DD-MM-YYYY-HH:MM:SS-FileName」

注: Android 8.x Oreo および Android 10 以降を実行しているデバイスでのみサポートされます。

ソース URI

サーバーでのファイルの場所の URI を入力するために使用されます。HTTP、HTTPS、FTP、FTPS (FTP over TLS または SSL) プロトコルと、URL に埋め込まれた認証情報の基本認証をサポートします。構文については、「例」セクションを参照してください

重要: ファイル マネージャは匿名 FTP をサポートしていません。

パラメータ値入力ルール:

  • ソース ファイルの URI を含む文字列
  • 認証情報は URI 内に組み込まれている必要があります
  • デフォルト (TLS) をオーバーライドするには、"protocol=SSL" を使用します。
  • デフォルト (false) を上書きするには、"implicit=true" を使用します。

注: HTTP/S には MX 7.0+ が必要です。FTPS には MX 7.1+ が必要です。ファイル マネージャは匿名 FTP をサポートしていません。

ユーザー名に特殊文字をエンコード

ユーザー名の特殊 (予約済み) 文字 (@ 記号) はエンコードする必要があります。たとえば、「username@company.com」は、SourceURI に「username%40company.com」と表示されます (以下を参照)。


<wap-provisioningdoc>
    <characteristic version="5.0" type="FileMgr">
        <parm name="FileAction" value="1" />
    <characteristic type="file-details">
        <parm name="TargetAccessMethod" value="2" />
        <parm name="TargetPathAndFileName" value="/sdcard/TestFile.apk" />
        <parm name="SourceAccessMethod" value="1" />
        <parm name="SourceURI" value="ftp-p://username%40company.com:password@192.168.0.100:21/Test/TestFile.apk />
    </characteristic>
    </characteristic>
</wap-provisioningdoc>
