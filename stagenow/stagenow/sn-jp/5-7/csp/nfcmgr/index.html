SmartDocs について


[SmartDocs] バーでは、特定の Zebra デバイスに存在する機能のみを表示するように、このページをカスタマイズできます。

  1. ドロップダウンを使用してデバイスの OSX、MX、Android のバージョンを選択するか、デバイスのモデルを選択すると、デバイスでサポートされている機能が強調表示されます。サポートされていない機能は、選択内容に基づいてグレー表示になります。
  2. SmartDocs Nav は、個々の CSP の機能を表示するときに機能します。使用するには、CSP (つまり「App Manager」) をクリックし、SmartDocs バーで少なくとも 1 つを選択します。サポートされていない機能はグレー表示になります。その CSP でサポートされている機能の間を前後にスクロールさせるには、SmartNav の [進む] ボタンと [前へ] ボタンをクリックします (機能マトリックスでは SmartNav はサポートされていません)。
  3. すべての選択をクリアするには、[SmartDocs] ボタンを押します。

デバイスの OSX、MX および Android のバージョン情報は、Android の設定パネルに表示されるか、ADB、EMDK、または MX CSP を使用してデバイスを照会することで確認できます。詳細情報。

  • OSX レイヤーは、Android で提供されていない機能を実装するために、基本オペレーティング システムの拡張機能を提供します。ルート OSX のバージョン番号は、拡張対象とする Android バージョンのルート番号と常に一致します。詳細情報。
  • MX 管理システムは、デバイス上の特権および非特権 Android API に対して、統一されたインタフェースを提供します。一部の機能では、MX バージョンと OSX バージョンの特定のペアリングが必要です。詳細情報。
  • 機能セットを決定するには、Android バージョンで十分な場合があります。また、特定の MX バージョンと Android の最小バージョンが必要になる場合もあります。
  • デバイス モデルは、MX 機能のどのサブセットがデバイスに存在するかを示します。

概要

NFC Manager (NfcMgr) は、NFC 無線に供給される電源および機能の状態や動作など、デバイスの近距離無線通信オプションを制御して、パフォーマンスを向上させ、または相互運用性の問題を回避します。NFC は短距離ワイヤレス仕様で、磁場誘導を使用して、接触している 2 つのデバイス、または近接している 2 つのデバイスが通信できるようにします。

サポートされているデバイス

NFC Manager は、次の Zebra デバイスでのみサポートされています。

  • MC93
  • TC52
  • TC57
  • TC72
  • TC77
  • TC8380

主な機能

  • 有効化/無効化
    • NFC 無線とその通信機能
    • NFC データ交換形式 (廃止)
    • ピアツーピア モード (廃止)
    • ポーリング モード
    • CPU ブースト モード
    • カード エミュレーション モード
    • Android の "ロック画面" が表示されている場合の NFC の使用
  • 使用するタイプ A、B、F、または V タグを選択します
  • タイプ A および B カード (TC55) の通信速度を選択します
  • ISO 14443-4 カード (TC75) の通信速度を選択します
  • デバイスをデフォルトの NFC 設定にリセットします
  • 次のカード技術をサポートしています。
    • ISO 14443 タイプ A および B (ISO/IEC 14443A、14443B)
    • ISO 15693
    • FeliCA (JIS X6319-4)
    • タイプ V (ISO/IEC 15693 に準拠するものすべて)

NFC の有効化/無効化

これは、近距離無線通信を使用して他のデバイスと情報を交換する機能を制御する、デバイス上の NFC 無線のオン/オフ スイッチです。

注: MC93、TC52、TC57、TC72、TC77、TC8380 デバイスでのみサポートされます。

パラメータ名: EnabledNFC

オプション 名前 説明注記 ステータス 要件
86 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 8.3+
1 有効 NFC 無線をオンにします。

MX: 8.3+
0 無効 NFC 無線をオフにします。

MX: 8.3+

デフォルトにリセット (チェックボックス)

すべての NFC 設定をシステム定義のデフォルト値にリセットするかどうかを制御します。この機能は、不明な NFC 設定があるデバイスを既知の状態に戻すために使用できます。

パラメータ名: ResetToDefaults

オプション 名前 説明注記 ステータス 要件
1 (オン) すべての NFC 設定をシステム定義のデフォルト値にリセットします。

MX: 8.3+
0 (オフ) NFC 設定は変更されません。

MX: 8.3+

カード エミュレーション モードの有効化/無効化

これは、カード エミュレーション (CE) モードのオン/オフ スイッチです。有効にすると、デバイスは非接触型スマート カードのように動作し、クレジット/デビット カード、トランジット カード、アクセス カードなどの代わりに使用できます。CE が有効になっている場合、デバイスは独自の RF フィールドを生成せず、NFC リーダーによる生成に依存します。

注: MC93、TC52、TC57、TC72、TC77、TC8380 デバイスでのみサポートされます。

パラメータ名: enableCE

オプション 名前 説明注記 ステータス 要件
86 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 8.3+
1 有効 デバイスでカード エミュレーション モードを有効にします。

MX: 8.3+
0 無効 デバイスでカード エミュレーションを無効にします。

MX: 8.3+

NFC データ交換形式の使用 (廃止)

2 つの NFC デバイス間で情報を交換するときに NFC データ交換形式 (NDEF) を使用するかどうかを制御します。NDEF は、デバイス間でアプリケーション定義のペイロードを交換、または NFC タグにペイロードを保存できるバイナリ形式です。

注: 廃止予定。NFC Manager によるこの機能の制御は今後のリリースで廃止されます。MC93、TC52、TC57、TC72、TC77、TC8380 デバイスでのみサポートされます。

パラメータ名: skipNdef

オプション 名前 説明注記 ステータス 要件
86 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 8.3+
1 使用する デバイスが NDEF を使用してデータを交換できるようにします。

MX: 8.3+
0 使用しない デバイスが NDEF を使用してデータを交換できないようにします。

MX: 8.3+

ピアツーピアの使用 (廃止)

2 台の NFC デバイス間でピアツーピア通信を許可するかどうかを制御します。一般的な P2P アプリケーションには、個人間での送金、写真、名刺またはその他のタイプのデータ、および Android Beam を使用するあらゆるものの交換が含まれます。

注: 廃止予定。この機能は今後のリリースで廃止されます。MC93、TC52、TC57、TC72、TC77、TC8380 デバイスでのみサポートされます。

パラメータ名: p2p

オプション 名前 説明注記 ステータス 要件
86 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 8.3+
1 使用する 2 台の NFC デバイス間でのピアツーピア データ交換を許可します。

MX: 8.3+
0 使用しない ピアツーピア データ交換ができないようにします。

MX: 8.3+

ポーリング モード

デバイスが近くの NFC タグまたはピアツーピア モードで通信しようとしているデバイスを検出しようとしているときに呼び出される、ポーリング モードを選択するために使用されます。検出速度と消費電力の最適なバランスを見つけるには、このパラメータを使用します。

注: MC93、TC52、TC57、TC72、TC77、TC8380 デバイスでのみサポートされます。

パラメータ名: pollingMode

オプション 名前 説明注記 ステータス 要件
86 変更しない この値 (またはこのパラメータが XML に存在しないこと) によってデバイスの設定は変更されません。以前に選択した設定はすべて保持されます。

MX: 8.3+
2 ハイブリッド 検出速度と消費電力のバランス調整を試みるハイブリッド ポーリングを有効にします。

MX: 8.3+
1 標準 最速で、最も消費電力が高くなる、標準ポーリングを有効にします。

MX: 8.3+
0 低消費電力 最も低速で、デバイスの消費電力が最も少ない、低消費電力ポーリングを有効にします。

MX: 8.3+
