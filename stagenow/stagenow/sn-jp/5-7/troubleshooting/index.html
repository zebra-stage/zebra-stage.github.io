StageNow のトラブルシューティング

StageNow 5.7

このセクションでは、StageNow の使用中に発生する可能性のある一般的なエラーについて説明し、その対処方法についてのアドバイスを行います。

XML 構成の不具合

問題

ステージングが完了すると、デバイスに「FMC のダウンロード中に FTP でのファイルの読み取りがタイムアウトになりました」というメッセージ (または類似のメッセージ) が表示されます。

原因

このエラーは通常、ステージング中に FTP を使用してパブリック サーバーから 1.5GB を超える大きなファイルをダウンロードしたときに発生します。

対処方法

ファイル転送に FTP ではなく FTPS を使用してみてください。

Wi-Fi ホットスポット

注: Windows 10 を実行しているシステムでは、Wi-Fi ホットスポット ステージングはサポートされていません。

問題

StageNow の Wi-Fi ホットスポット機能をオンにしようとすると、「Wi-Fi ホットスポットをオンにできません」というメッセージ (または類似のメッセージ) が表示されます。

原因

これは、ホスト PC の Wi-Fi アダプタが初期化されていない場合や、StageNow の起動時に Wi-Fi アダプタが非アクティブになっている場合に発生することがあります。

対処方法

問題を解決するには、次の手順を試してください。

Wi-Fi アダプタを再起動する

  1. Windows ホストの右下隅にある [ネットワーク] アイコンを右クリックし、[ネットワークと共有センターを開く] を選択して次の画面を開きます。

  2. 左側のペインで、[アダプタ設定の変更] を選択します。

  3. コマンド プロンプトを開き、「netsh wlan show drivers」と入力して Wi-Fi ドライバ アダプタ名を検索します。

  4. [ネットワーク接続] ウィンドウでネットワーク アダプタを右クリックし、[無効] を選択します。

  5. もう一度右クリックして、[有効] を選択します。数分後に、StageNow ツールを使用してみます。

サポート外のホットスポットを追加する

注: 次のものと異なる結果が出た場合は、Wi-Fi ホットスポットに関する Windows ドライバやソフトウェアの問題と考えられます。

  1. 管理者としてコマンド プロンプトを開きます。

  2. 「netsh wlan show drivers」と入力し、下図の緑色で示された結果メッセージを探します。

    システムが Wi-Fi ホットスポットをサポートしていない場合は、次のメッセージが表示されます。

  3. コマンド プロンプトで次のコマンドを入力します。「netsh wlan set hostednetwork mode=allow ssid=key=」

    注: HotspotName は Wi-Fi ホットスポットの名前です。

    HotspotPassword は Wi-Fi ホットスポットのパスワードで、5 文字以上である必要があります。

    Wi-Fi ホットスポットが正常に作成されると、次の結果が表示されます。

  4. 「netsh wlan start hostednetwork」と入力して、ホットスポットを開始します。

  5. 「netsh wlan stop hostednetwork」と入力して、ホットスポットを停止します。

ダウンロードしたファイルが破損している

問題

管理ホスト PC からデバイスにファイルを転送する [ファイルの転送/コピー] アクションを実行しようとすると、「ダウンロードされたファイルが破損しています。これは、ネットワーク接続が不安定であることが原因の可能性があります」のようなメッセージが表示されます。

原因

これは、ソース ファイルと宛先ファイルのチェックサムが一致しないためです。

対処方法

問題を解決するには、次の手順を試してください。

  • エラーが発生した PC 以外の PC でプロファイルが作成され、StageNow にインポートされた場合:
    • a. インポートしたプロファイルの FileMgr セクションに移動します
    • b. コピーするファイルへのパスが現在使用中の PC のパスと一致するように、[ソース パスとファイル] パラメータを編集します
    • c. 変更を保存し、新しいバーコードを作成します
  • 現在使用中の管理ホスト PC でプロファイルが作成された場合は、次の手順を実行します。
    • a. 管理ホスト PC で %appdata%\StageNow\AST\FTP\PROFILES フォルダに移動します
    • b. そこで見つかったチェックサム ファイルを削除します
    • c. 変更を保存し、新しいバーコードを作成します

プラグイン CSP の問題

プラグイン CSP で作成されたプロファイルは、内蔵の CSP で作成されたプロファイルとは異なる動作をします。最も重要な相違点は、プロファイルのインポートと削除です。

画像をクリックすると拡大表示され、Esc キーを押すとキャンセルします。

問題

プラグイン CSP として CSP ライブラリにインポートされた .apk を削除しようとすると、エラーが発生します。

対処方法

  1. すべてのプロファイルを、管理者のホスト PC 上の場所にエクスポートします。
  2. PC の %appdata% ディレクトリから StageNow DB を削除します。
  3. エクスポートしたプロファイルをインポートし直します。
