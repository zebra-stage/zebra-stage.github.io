Parametri nominali

Device Diagnostic Tool 2.2

Panoramica

Device Diagnostic Tool testa il funzionamento delle funzioni hardware dei dispositivi in base a specifici criteri, con i seguenti risultati:

  • Pass (Esito positivo) - se i criteri sono soddisfatti
  • Fail (Esito negativo) - se i criteri non sono soddisfatti
  • Information (Informazioni) - visualizza dati informativi (se i criteri Pass/Fail non sono applicabili)
Test Richiede l'intervento dell'utente Proprietà/azione Criteri Pass/Fail
Scanner Sì Scansione di un codice a barre Pass quando la scansione di un codice a barre viene eseguita correttamente.
Pulsante Sì Pressione del pulsante:
  - Grilletto di scansione (lato sinistro o destro)
  - Push-to-talk
  - Aumento del volume
  - Riduzione del volume		 Pass se si rileva la pressione per ciascuno dei pulsanti.
Touch screen Sì Griglia di contatto Pass se si rileva contatto in ogni riquadro della griglia dello schermo.
Bluetooth No Informazioni restituite:
  - Nome
  - Ciclo di alimentazione della radio
  - Funzionante/
non funzionante
  - Collegabile/
individuabile		 Pass se è possibile riavviare la radio Bluetooth e se le informazioni vengono restituite correttamente.

  -Indirizzo MAC
  - Test di rete

  - Potenza del segnale (i)

  - Codice articolo

  - Temperatura (i)		 Pass se la batteria è in buone condizioni e se le informazioni relative alla batteria vengono acquisite correttamente.
Audio Sì Registrazione e riproduzione di un file È richiesto l'intervento manuale dell'utente. L'utente registra manualmente un brano audio e ascolta la registrazione. Pass/Fail a discrezione dell'utente. Per questo test è necessario utilizzare gli altoparlanti del dispositivo - l'uso delle cuffie non è consentito. Per garantirne la riuscita, il test dovrebbe essere condotto in un ambiente con rumore di fondo ridotto al minimo. Durante la riproduzione, impostare il volume massimo sul dispositivo per ottenere risultati migliori.
Nota: questo test non può essere eseguito se il dispositivo è privo di microfono, come il modello MC18.


Il testo in rossoindica informazioni sensibili che non vengono visualizzate. In alternativa, vengono visualizzate le diciture “valid” (valida) o “invalid” (non valida), per indicare se l'informazione è fornita nel formato corretto. “Valid” indica che è stato restituito il valore corretto. “Invalid” indica che il valore restituito è nullo, vuoto oppure non corrisponde alla sequenza di indirizzi MAC prevista (si applica solo agli indirizzi MAC).

(i) indica dati informativi considerati come dati aggiuntivi, visualizzati se show_extra_data è impostato su “true” (vero) nel file configuration.xml.

La descrizione del singoli test è contenuta nella sezione Configurazione dei test.
