Uso di Device Diagnostic Tool

Device Diagnostic Tool 2.2

Panoramica

Device Diagnostic Tool supporta due modalità operative:

  1. Admin mode (Modalità Amministratore) - L'amministratore può accedere alle impostazioni dell'app, modificare le configurazioni dei test ed eseguire i singoli test. Questa è la modalità predefinita.
  2. User mode (Modalità Utente) - L'utente è autorizzato esclusivamente all'esecuzione dei test; non è previsto l'accesso alle impostazioni dell'app né alle configurazioni dei test. Per impostare la modalità Utente, disabilitare admin_mode come descritto nella sezione Configurazione.

Avvio dell'app

Per eseguire Device Diagnostic Tool si può adottare uno dei seguenti metodi:

  1. Manualmente, mediante l'interfaccia utente - L'utente avvia Device Diagnostic Tool ed esegue manualmente i test. Consultare la seguente sezione Interfaccia utente.
  2. Da remoto, con un sistema EMM (Enterprise Mobility Management) - Avviare Device Diagnostic Tool sullo sfondo e generare un log con il comando:
    adb shell am broadcast -n "com.symbol.selfdiagnostics/com.symbol.selfdiagnostics.SESReceiver"

Per avviare Device Diagnostic Tool è necessario adottare solo uno dei metodi descritti. L'uso di due metodi in contemporanea può dare luogo a comportamenti inattesi.

Interfaccia utente

Al primo avvio dell'app, la schermata principale visualizza i test eseguibili: Schermata principale
Se sono già stati eseguiti dei test, il nome del test è seguito dalla data dell'ultima esecuzione.

Toccando Run Tests (Esegui prove) sulla schermata principale vengono eseguiti tutti i test. La procedura di esecuzione si basa sui test selezionati nella schermata Configure Tests (Configurazione dei test). Quando si avvia un test, l'utente viene invitato a eseguire ulteriori azioni, quali: prova scanner, prova pulsanti, prova touch screen e prova audio. Si consiglia di eseguire tutti i test durante l'uso normale del dispositivo, non quando è inserito in un alloggiamento o collegato a un computer tramite cavo USB. Una volta concluso il test, vengono visualizzati i risultati:
   Pass (Esito positivo) - Il test è stato superato. Il risultato rientra nei parametri nominali.
   Fail (Esito negativo) - Il test non ha soddisfatto i parametri nominali.

   Information (Informazioni) -Dati acquisiti e visualizzati.

In alternativa, in modalità Amministratore è possibile eseguire singoli test toccando la categoria desiderata e quindi toccando Run Tests.
I test non superati e scaduti sono contrassegnati con la stessa icona di pericolo.

Scanner Test (Prova scanner)

L'utente viene invitato a eseguire la scansione di un codice a barre. Risultati

  • Scanner Test (Prova scanner) – visualizza i dati del codice a barre
  • Label Type (Tipo di etichetta) – visualizza il tipo di codice a barre o il decodificatore oggetto della scansione

Nota: quando si esegue uno Scanner Test sul modello TC55, l'utente deve premere a lungo il pulsante dello scanner, affinché il dispositivo lo rilevi.

Button Test (Prova pulsanti)

L'utente viene invitato a premere i pulsanti fisici presenti sul dispositivo: grilletto di scansione (lato sinistro o destro), push-to-talk, aumento e riduzione del volume. Risultati

  • Button Test – test successful (riuscito), test failed (non riuscito) o test timed-out (scaduto)

Nota: se si esegue una nuova mappatura dei pulsanti fisici il comportamento del Button Test potrebbe risentirne.

Touch Screen Test (Prova touch screen)

L'utente è invitato a toccare ogni riquadro della griglia dello schermo

  • Touch Screen Test – test successful (riuscito), test failed (non riuscito) o test timed-out (scaduto)

Bluetooth Tests (Prove Bluetooth)

Verifica la funzionalità della radio Bluetooth e restituisce informazioni relative al sistema Bluetooth. Risultati

  • Name (Nome) – visualizza il nome della rete Bluetooth
  • Radio Power Cycle (Ciclo di alimentazione della radio) – test successful (riuscito) o test failed (non riuscito). Lo stato della radio viene mantenuto alle condizioni precedenti a questa prova.
  • Functional/Non-functional (Funzionante/Non funzionante) – functional (funzionante) o non-functional (non funzionante)
  • Discoverable/Connectable (Individuabile/Collegabile) - connectable (collegabile), discoverable (individuabile) o none (nessuno)

Wifi Tests (Prove Wi-Fi)

Verifica la funzionalità della radio Wi-Fi e restituisce informazioni relative al sistema Wi-Fi. Risultati

  • MAC Address (Indirizzo MAC) – valid (valido) o invalid (non valido)
  • Network Test (Test di rete) –– visualizza connected (connessa) o not connected (non connessa) Il test non riesce se il sistema Wi-Fi non è connesso ad alcuna rete. Lo stato della radio viene mantenuto alle condizioni precedenti a questa prova. Se il test va a buon fine, vengono visualizzati i seguenti valori:
    • strength (potenza) – indica la potenza del segnale
    • essid – visualizza ESSID; sui modelli Android O e superiori, è necessario che sia abilitata la funzione di geolocalizzazione nelle impostazioni del dispositivo, a causa delle limitazioni imposte da Android. Se questa impostazione non è abilitata, il valore restituito è unknown ssid (SSID ignoto).
    • ip – visualizza l'indirizzo IP
    • bssid – valid (valido) o invalid (non valido)
    • speed (velocità) – visualizza la velocità di connessione
  • Radio Power Cycle (Ciclo di alimentazione della radio) – test successful (riuscito) o test failed (non riuscito)

Battery Tests (Prove batteria)

Verifica lo stato della batteria e restituisce informazioni relative alla batteria stessa. Risultati

  • Part number (Codice articolo) – visualizza il codice articolo
  • Serial number (Numero di serie) – visualizza il numero di serie
  • Manufacture Date (Data di fabbricazione) – visualizza la data di fabbricazione della batteria
  • Decommission status (Stato di decommissioning) – visualizza lo stato di salute della batteria, ossia:
    • Good (Buono) – la batteria è in ottime condizioni
    • Decommissioned (Da smantellare) – la batteria ha raggiunto la soglia prevista, attualmente 400 cicli di ricarica, pertanto deve essere sostituita al più presto.
    • Unknown (Ignoto) – indica un problema nel recupero delle informazioni in merito alle condizioni della batteria
  • Voltage (Tensione) – visualizza la tensione
  • Current (Corrente) – visualizza la corrente
  • Temperature (Temperatura) – visualizza la temperatura

WWAN Tests (Prove WWAN)

Verifica la funzionalità della radio WWAN e restituisce informazioni relative al sistema WWAN. Risultati

  • Sim State (Stato SIM) – È necessario disattivare la modalità aereo. Visualizza uno dei seguenti valori:
    • Present (Presente) – La scheda SIM è installata
    • Absent (Assente) – La scheda SIM non è installata
  • Voice State (Stato voce) - Visualizza uno dei seguenti valori:
    • Voice in service (Servizi vocali attivi)
    • Voice out of service (Servizi vocali non attivi)
    • Voice Emergency only (Solo servizi vocali di emergenza)
    • Voice power off (Spegimento mediante voce)
    • Unknown voice (Servizi vocali sconosciuti)
  • Data State (Stato dati) – Visualizza uno dei seguenti valori:
    • Data connected (Connessione dati attiva)
    • Data disconnected (Connessione dati non attiva)
    • Data connecting (Connessione dati in corso)
    • Data suspended (Dati sospesi)
    • Unknown data (Dati sconosciuti)
  • WWAN Type (Tipo di WWAN) – visualizza il tipo di rete, ad esempio: LTE, 2G, 3G, 4G o Not Available (Non disponibile)
  • Signal Strength (Potenza del segnale) – Visualizza uno dei seguenti valori:
    • Not applicable (Non applicabile) - Scheda SIM assente
    • Unknown (Ignota) – il dispositivo potrebbe essere in modalità aereo
    • Signal strength (Potenza segnale) - in dBm
  • Phone Number (Numero di telefono) – valid (valido) orinvalid (non valido), a seconda che il numero di telefono sia stato o non sia stato reso noto dal gestore del servizio.
  • Device ID (ID dispositivo) – valid (valido) o invalid (non valido), a seconda che l'ID del dispositivo sia stato o non sia stato reso noto

Audio Test (Prova audio)

Verifica la funzionalità del microfono e degli altoparlanti del dispositivo. Risultati

  • Audio Test (Prova audio) – test successful (riuscito) o test failed (non riuscito)
