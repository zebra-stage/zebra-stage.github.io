About Zebra DNA Cloud

Zebra DNA Cloud 5.4

What's new in Zebra DNA Cloud 5.4

Overview

Zebra DNA Cloud (ZDNA) is an administrative solution for managing and configuring Zebra devices and apps. Like a cloud-based version of Zebra StageNow, ZDNA also performs licensing and other administrative tasks, and will eventually do much more. Also like StageNow, ZDNA leverages the power of Zebra's Mobility Extensions (MX), is controlled through a simple, intuitive UI, and can stand alone or work with a company's existing Enterprise Mobility Management (EMM) system.

The ZDNA dashboard provides a high-level system overview, including device and battery status, license allocation
and available Android updates, refreshing every 24 hours. Click image to enlarge; ESC to exit.

KNOWN ISSUES

  • NEW: App Setup Profiles created with DNA Cloud 5.2 (or older) must be modified for devices running Android 14:
    • Use the "Edit" function in DNA Cloud 5.3 (or later) to open the existing Profile(s).
    • Redeploy the Profile(s) to Android 14 devices.
      Profiles newly created with ZDNA 5.3 are unaffected.
  • NEW: Status of a deployment shows as "In-Progress" despite one or more devices in "Issue" state.
  • The “Lock Screen Visibility” feature of OEMConfig 13.3.0.5 (and later) is NOT supported. If enabled on the device, the lock screen appears blank on the ZDNA console during Remote Support sessions.
  • Devices enrolled in ZDNA 4.2 using a barcode created with a prior ZDNA version do not enroll correctly, and appear with a icon.
    To resolve this issue:
    • Create a new Profile with ZDNA 4.2 (or later) and re-enroll with the generated barcode.
      ~OR~
    • In ZDNA 4.2 (or later), open the original Profile using the "Edit" feature and save the barcode. Re-enroll device(s) using the newly generated barcode.
  • The “Enterprise” Wi-Fi Security Mode is not supported on devices running Android 13.
  • When first booting (or rebooting from either a cold boot or hard reset), all Zebra devices running Android 13 (or later) must be unlocked (using PIN, password or pattern) before the DNA Cloud client app can launch. Also applies to 6490- and 6375-platform devices running Android 11.
  • When creating a managed configuration for Enterprise Keyboard, remapping a programmable key requires two backslash ("\") characters (rather than one, as documented).
  • On SDM660-platform devices running Android 11, DNA Cloud does NOT support these device-OS update packages:
    • HE_FULL_UPDATE_11-13-13.00-RG-U26-STD-HEL-04.zip
    • HE_FULL_UPDATE_11-13-13.00-RG-U00-STD-HEL-04.zip
    • HE_FULL_UPDATE_11-13-13.00-RG-U12-STD-HEL-04.zip
  • Regarding devices with the BSPs listed below:
    • 6490-platform: AT_FULL_UPDATE_11-17-16.00-RG-U00-STD-ATH-04.zip
    • 6490-platform: AT_FULL_UPDATE_11-19-15.00-RG-U00-STD-ATH-04.zip
    • SDM660-platform: HE_FULL_UPDATE_11-31-27.00-RG-U00-STD-HEL-04.zip
      For full ZDNA functionality on such devices:
    • If already enrolled, generate a new barcode using ZDNA 3.11 or later, re-enroll and update to the latest BSP.
    • If not already enrolled, generate a new barcode using ZDNA 3.11 or later and update to the latest BSP.
