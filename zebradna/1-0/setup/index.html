Zebra DNA Cloud Setup

Zebra DNA Cloud

Overview

This section covers all procedures for an organization to begin using Zebra DNA Cloud.

ALL of the following steps must be completed in this order:

  1. Configure your organization's firewall
  2. Register with Zebra.com (one-time per org/admin)
  3. Create at least one "Enrollment Profile"
  4. Enroll one or more devices using the Enrollment Profile
  5. If applicable, Apply license(s) to device(s)

Browser Requirements

The Zebra DNA Cloud console is supported on the following browsers:

  • Apple Safari (v14.0 or later)
  • Google Chrome
  • Microsoft Edge

I. Configure Network

From the device, the zDNA client app communicates with zDNA Cloud servers using the following:

Using Wildcards

  • Open HTTPS port (443 by default) between the zDNA client app zDNA Cloud servers
  • Allow domains through the firewall or proxy:
    • connectivitycheck.gstatic.com
    • *.googleapis.com
    • *.firebaseio.com
    • *.cloudfunctions.net
    • [ProjectID].firebaseapp.com ([ProjectID] is supplied by Zebra during onboarding)
  • Domains allowed through the firewall or proxy (accessed by admin PC):
    • [Web portal] - supplied by Zebra during onboarding.

NOT Using Wildcards

If wildcards are not supported or desired, add the following domains to the allow list:

  • connectivitycheck.gstatic.com
  • www.googleapis.com
  • firestore.googleapis.com
  • cloudfunctions.googleapis.com
  • us-central1-[ProjectID].cloudfunctions.net
  • [ProjectID].firebaseio.com (The ProjectID is supplied by Zebra during onboarding)
  • [ProjectID]-default-rtdb.firebaseio.com (The ProjectID is supplied by Zebra during onboarding)
  • s-usc1c-nss-*.firebaseio.com (* represents multiple characters. Since * can change over time, the firewall must allow any domain with this pattern for the app to function)
  • [ProjectID].firebaseapp.com (The ProjectID is supplied by Zebra during onboarding)

III. Enroll Device(s)

The "Enrollment Profile" section is primarily for creating the minimum settings required to allow a device to communicate with the internet for further configuration and management (or for hand-off to an EMM). This process also downloads the zDNA client app to the device and enrolls it in the zDNA Cloud system. To stage a device with these settings, create a new Enrollment Profile and use the device to scan the barcode that appears at the end of the process.

Repeat this process as many times as needed for different device types, Wi-Fi configurations, office locations or other purposes.

Create an Enrollment Profile:

1. In the Initial Setup section, click the "New Initial Setup" button: Click image to enlarge; ESC to exit.

2. Enter the Wi-Fi settings for the device in its intended location.
Click "Next" button when finished. Click image to enlarge; ESC to exit.

3. Uncheck "LifeGuard Over the Air" box if an updated system OTHER THAN Zebra's system is to be used to send updates and security patches to the device.
Click "Next" button when finished. Click image to enlarge; ESC to exit.

4. Enter the following information as indicated:
    a. Enrollment Profile Name - use auto-generated name or enter custom
    b. Administrator message - appears on barcode page and Profile listings
    c. Device Group Association - helps organize by model, function, dept. etc.
Click "Next" button when finished. Click image to enlarge; ESC to exit.

5. Review settings. If necessary, click "Edit" to modify any setting(s).
Click "Next" button when finished. Click image to enlarge; ESC to exit.

6. With the StageNow app running on the device, scan the generated barcode to configure and enroll the device.
Then click "Done" button. Click image to enlarge; ESC to exit.

Initial device staging and enrollment are complete.

Device enrollment can be confirmed on the "My Devices" page: Click image to enlarge; ESC to exit.
