Migrating to Android 13

LifeGuard

PAGE LAST UPDATED: June 19, 2023

Overview

Android 13 presents some potential challenges for organizations planning to migrate their Zebra SDM660 platform devices from earlier Android versions. The move requires conversion from full-disk to file-based encryption, which typically requires a device reset and loss of data and non-factory apps. However, Zebra is developing a unique process that can preserve (backup/restore) approximately 3GB of data locally on devices prior to migration.

The first section of this guide covers migration scenarios in which data preservation is required. If data preservation is NOT required, skip to the A-13 Quick Upgrade section for Zebra's alternative upgrade process.

This process is NOT recommended for devices that comply with Common Criteria Certification.
     Such devices can be upgraded to Android 13 using the normal process with no risk of data loss.

Upgrade to Android 13 (no data loss)

The process documented below enables preservation of apps, data and settings on the device as part of the upgrade process. To simplify the process and help preserve device data when migrating to Android 13, Zebra has developed Zebra File Guardian, a utility that administrators can use to select and prioritize data to be preserved. The tool also permits file prioritization, allowing for a single "Golden Configuration" to be created for each device persona, or "Golden Device," so that a single instruction set for preserving the MOST important data can be deployed to devices that might contain data in excess of the maximum.

Process Overview

  1. Download Zebra Conversion Package (ZCP) to admin host PC
  2. IF NECESSARY: Remove password policies from devices to ensure that migration operations can be completed.†
  3. ON GOLDEN DEVICE ONLY:
        a. Remove file-access restrictions to enable USB-based file extraction (supports adb only) during process setup.† Learn how to set up adb.
        b. Download the Android 13 image from the Zebra Support Portal and push to device.††
  4. Push ZCP to a "Golden" device (see below) use Zebra MX Power Manager (or the equivalent EMM function) to perform an "OS Upgrade" and reboot device.
  5. File Guardian utility appears. Select files to be preserved and save backup as the "Golden Configuration" for that device class.
  6. Pull the Golden Config from the device to the host PC.
  7. Push Golden Config to a specific location on target device(s) being upgraded to Android 13.
  8. Download the desired Android 13 image from Zebra, change file name and push to specific location on target device(s).
  9. Push the Zebra Conversion Package to the target device(s) and perform "OS Upgrade" with Zebra Power Manager Action or equivalent EMM function.
  10. When Golden device loads Android 13, restore file-access restrictions.

† Such policies must be restored manually after successful migration.

†† This step allows the device to return to normal use after its utility in the migration process.

The process continues automatically, rebooting several times before running Android 13 when completed successfully.
USERS SHOULD BE INSTRUCTED NOT TO INTERFERE WITH THE AUTOMATED MIGRATION PROCESS.
Zebra STRONGLY recommends testing the migration process on a small group of devices from each persona before wider enterprise deployment.


 ATTENTION!!
Before proceeding, please read the Notes, Limitations and Preconditions sections below.
Failure to do so could lead to data loss and/or unusable devices after attempting migration.

Notes

  • Applies to SDM660-platform devices only (see SDM660 device list)
  • Converts device to file-based encryption (from disk-based)
  • Preserves approximately 3GB of apps and data
  • The File Guardian utility allows data prioritization for devices containing more than 3GB to preserve
  • Any data in /enterprise partition is preserved
  • Does NOT support Zebra's "Smart Profile" feature
  • GMS app data and settings are preserved. GMS apps installed as part of A-13 might require an OTA update.
  • Expected availability: Late June, 2023

Limitations

application data and non-factory applications will not be configured to be backed up unless the Admin explicitly says to do so and that system configuration is AUTOMATICALLY backed up without any action (or choice) by the Admin.

  • Android System settings are backed up by default; no action or file-selection is required.
  • Apps, app data and settings MUST be explicitly referenced to be backed up in the "Golden Configuration".
  • Maximum backup size is approximately 3GB.
  • Devices with SD Cards formatted as Adoptable Storage are NOT supported.
  • The process works only on the device's Primary User. Work profiles, COPE and other multi-user scenarios are NOT preserved and cannot be recreated in Android 13.
  • GMS data and settings are preserved; GMS apps are deleted.
  • App(s) originally installed by an EMM system from Google Play and NOT backed up using the Zebra utility and must be reinstalled manually after migration.
  • If an automated upgrade fails and the device remains booted in the Zebra Conversion Package (interim OS), the upgrade to Android 13 must be completed manually; the interim OS is NOT suitable for production environments.

Failure to follow all steps and meet all preconditions (below) prior to beginning the upgrade process could halt the automated process (and require manual intervention to continue) and/or result in migration failure (possibly in data loss and/or temporarily leaving device in an unusable state).

Preconditions

The following conditions must be met on devices before attempting to upgrade using this process.

  1. Battery must have 30% charge (or more) or be connected to AC power
  2. "Secure startup" PIN/password feature must be DISABLED to allow process to continue unattended
    To check: Settings > Biometrics and security > Secure startup
  3. Screen lock must be DISABLED to allow process to continue unattended
    To check: Settings > Security > Screen Lock > PIN/Password/Pattern/Swipe
  4. Password Policy must be DISABLED
    For EMM users, it also might be necessary to remove any password policy to ensure automated migration operations to complete. Such policies must be restored manually after migration is completed.
  5. SD Card must NOT be configured as Adoptable Storage
    LOSS OF CUSTOMER-APP DATA COULD RESULT
  6. Device encryption type must be "FDE"
    To check: Settings > About Phone > SW Components > Encryption Type > block

Zebra strongly recommends backing up devices before attempting migration.

Migration Process Details

This process is NOT recommended for devices that comply with Common Criteria Certification.
     Such devices can be upgraded to Android 13 using the normal process with no risk of data loss.

To upgrade Zebra SDM660-platform devices to Android 13:

1. Identify "Golden" device(s), defined as device(s) typical of usage persona(s) within the organization that might include devices with:

  • Specific Zebra-provided apps and data
  • Specific company-installed apps and data
  • System settings such as pre-configured Wi-Fi access points, passwords, etc.
  • EMM enrollment data and agent apps
  • Files and/or folders created by the company or device users

2. Download Zebra Conversion Package (ZCP), use Zebra MX Power Manager (or the equivalent EMM function) to perform an "OS Upgrade" on the ZCP file and reboot device.

3. At boot, the device automatically runs File Guardian, a Zebra utility used to select apps and data to be backed up and restored to devices after migration. The resulting backup is called a "Golden Configuration."

4. Use adb to extract Golden Config from device at /enterprise/Usr/persist/backup_restore_config.xml and store it on the administrative host PC.

5. Use StageNow, OEMConfig, an EMM system to push the Golden Config to target device(s) in the same location:

  • /enterprise/Usr/persist/backup_restore_config.xml

6. Download the desired Android 13 OS image appropriate for the device:

  • TC21 devices with GMS
  • TC21 devices without GMS

Reminder: This process is for Zebra SDM660-platform devices with FDE only. Devices on most other platforms can use the Quick Upgrade process with no fear of data loss.

7. Push the Android 13 OS image to all device(s) (except Golden device) using the following path and file name:

  • /data/tmp/public/FBE_FOTA.zip
    The Android 13 image MUST be in this location with this file name or the process will FAIL.

8. Push the Zebra Conversion Package to all target device(s) (except Golden device) to be upgraded to Android 13.

9. Perform an "OS Upgrade" on the ZCP using Zebra MX Power Manager (or the equivalent EMM function).

10. Repeat the process for devices from other personas.

The remainder of this process continues automatically, rebooting several times before running Android 13 when completed successfully.
USERS SHOULD BE INSTRUCTED NOT TO INTERFERE WITH THE AUTOMATED MIGRATION PROCESS.

Once migration is completed, restore policies removed to facilitate the process, if any.
Zebra STRONGLY recommends testing the migration process on a small group of devices from each persona before wider enterprise deployment.

Failure Recovery

If an upgrade fails to complete, it could leave the device running the Zebra Conversion Package (ZCP). The steps below might be useful for restarting the upgrade attempt. Based on information displayed on the ZCP screen, perform one or more of the following steps:

  1. If device is stuck at installation of Android 13 OS image, try rebooting. ZCP will attempt to install Android 13 OS image again.

  2. If Step 1 fails, put device in recovery mode. Instructions for entering recovery mode vary slighly by device. Some can be found in the Device Update guide elsewhere in these LifeGuard TechDocs. Use Android Debug Bridge (adb) commands to push and install the Android 13 OS image. This should install and boot the device into Android 13.

  3. If Android 13 boots but is unstable (apps repeatedly crash, preserved data missing, etc.), contact Zebra Support for further assistance.

A-13 Quick Upgrade (data loss)


 STOP!!
All device data is erased when upgrading SDM660 devices to Android 13 except data in the /enterprise partition.

Upgrading to Android 13 is supported only on Zebra TC21 devices at this time. Other models will follow as Android 13 becomes available. During the upgrade process, an Enterprise Reset is performed, removing all device data except that stored in the /enterprise partition. The process is otherwise identical to that of a traditional upgrade (shown below).

The following instructions are for Zebra StageNow, and should be modified accordingly for use with Zebra OEMConfig or a company's own an EMM system.

To upgrade to Android 13 without preserving device data:

1. Download the Android 13 OS image appropriate for the device:

  • TC21 devices with GMS
  • TC21 devices without GMS

2. Push the OS image to the device(s) by any available means (adb, StageNow, EMM, etc.).

3. Create a Profile with Power Manager to perform an OSUpdate, generate barcode(s) and scan with device scanner.

Success is indicated when device boots with Android 13.
Proceed with EMM enrollment and additional provisioning as desired.

Notes

  • LifeGuard Over the Air is not supported for Android 13 OS image deployment (coming later in 2023)
  • Zebra has successfully tested persistence only when using Zebra Persist Manager
  • Persistence by means other than Zebra PersistMgr cannot be guaranteed
  • Upgrades to Android 13 are supported only from devices running Android 8, 9, 10 or 11
