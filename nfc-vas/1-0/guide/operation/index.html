Operation

Zebra Apple VAS SDK 1.0

Overview

There is one main component of the SDK, ZebraNfcVAS, the VAS configuration object. The ZebraNfcVas object handles all the communication with the passes, the configuration of the NFC system and pass decryption. The Pass contains the payload of the pass along with additional meta data about the pass.

Configuration Objects

A configuration object must contain at least one Pass Type ID, but may contain multiple Pass Type IDs. This identifies which pass types should be returned. A configuration object may contain a private key. The private key should be associated with the Pass Type IDs in the configuration object. It is the key that is used to decrypt data from the pass that matches one of the associated Pass Type IDs. It is also acceptable to have multiple configuration objects. Each configuration object is used to query the Apple device for a pass.

Example:

  • Configuration Object #1
    • PassTypeId:
      • pass.com.zebra.customer_loyalty
      • pass.com.zebra.cafe_loyalty
    • Private Key: private key that decrypts either of the two above pass types.
  • Configuration Object #2
    • PassTypeId:
      • pass.com.zebra.special_holiday_event_pass
    • PrivateKey: private key that decrypts the data from the pass

The application developer could specify either one or both of the above configuration objects when attempting to read passes. The SDK iterates through each Pass Type Id in each configuration object. In the above example, assuming both configuration objects were provided, the read sequence is:

  1. pass.com.zebra.customer_loyalty
  2. pass.com.zebra.cafe_loyalty
  3. pass.com.zebra.special_holiday_event_pass

ZebraNfcVas Object

The ZebraNfcVas Object connects with the Zebra NFC VAS service, sets up the configuration for the NFC polling and handles reading of the passes from a mobile wallet. Once the pass has been read, ZebraNfcVas decrypts the pass data (if all information is provided and correct) and returns the Pass object to the caller.

Pass Object

The pass object encapsulates the information about the pass read. This includes:

  • pass payload (decrypted if possible)
  • pass type (Apple VAS)
  • payload date (date and time pass was read)
  • Pass Type Id / MerchantID associated with the pass

This allows a developer to validate that a pass is the correct type and that it is not a replay (the date and time is within 10 seconds of the actual date and time) as per Apple requirements.
