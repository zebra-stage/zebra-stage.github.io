Zebra Apple VAS SDK 1.0

Overview

The Zebra Apple VAS SDK is an Android library built on top of the existing Android NFC APIs and Apple VAS protocol. It provides simplified configuration and control of the processes required for accessing and operating on the VAS pass (pass data) stored in a mobile device. This guide discusses the APIs available for NFC VAS.

To use the SDK, import the Android .aar file into the application project (libs folder).

setPrivateKey()

    public void setPrivateKey(String key)

Sets the private key in PEM format. This is the private key that is used to decrypt the VAS pass object. This overwrites any existing value.

Parameter: String key This is the PEM formatted Private key to be used to decrypt the selected pass.

Example:

    -----BEGIN EC PARAMETERS-----\n
    alkidnklde==\n
    -----END EC PARAMETERS-----\n 
    -----BEGIN EC PRIVATE KEY——\n 
    Aslkdflhksdfjfkk728klxld...\n
    ak7hjvgn9b8o3bv9hae98fn2...\ 
    n 9asjh90fuh34n2uh33==\n
    -----END EC PRIVATE KEY-----\n

Return value: None

Throws: InvalidParameterException if the key is not correctly formatted, is null, or empty.

connectToReader()

    pubilc void connectToReader ()

Obtains a connection to the NFC adapter.

Parameter: None

Return value: Returns error code indicating success or failure:

  • ERROR_NFC_IS_DISABLED - there is an NFC adapter, but it is not active
  • ERROR_NO_NFC_SUPPORT - this hardware does not have NFC support
  • ERROR_SUCCESS - connection is successful

onPassDetected()

    public void onPassDetected(Pass pass)

The interface method onPassDetected() is called when an AppleVAS NFC token is read. The Zebra Apple VAS SDK passes the VAS token and an identified to indicate whether the token is encrypted or decrypted (APPLEVAS-DECRYPT or APPLEVAS) Needs rewording

Parameter: Pass object

Return value: None

Pass is a Getter class where all Pass data can be accessed via getter methods:

  • public String getStatusWord() – Returns the StatusWord of the NFC tag detected
  • public Result getResultCode() - Returns the ResultCode of the NFC tag detected
  • public byte[] getMerchantId() - Returns the merchantID of the NFC tag detected
  • public String getPassTypeId() - Returns the PassTypeID of the NFC tag detected
  • public Type getPassType() - Returns the PassType of the NFC tag detected
  • public byte[] getPayloadMessage() - Returns the PayloadMessage of the NFC tag detected
  • public Date getPayloadDate() - Returns the PayloadDate of the NFC tag detected
  • public boolean isDecrypted() – Returns the status of the payload message - decrypted or encrypted

Security

Clarify this WHOLE section

Zebra Apple VAS SDK has implemented advanced security features helping to protect the private key operations and connections of the authorized VAS service with the business application. The enhanced security mechanism is designed to ensure the application connects with a genuine Zebra VAS service.

The security features implemented with Zebra Apple VAS SDK include but are not limited to the following:

  • The authenticity of service through SDK.
    • Since OOB VAS Service is not part of the device software (OS), it needs to be obtained from a trusted source
    • The service code and binary are secured by secure-dex process.
    • The service is signed by Zebra (unprivileged) key.
    • The SDK validates the service by checking the signature of the service before proceeding further (internal SDK mechanism).

Since the SDK is an Android .aar file imported into the application project, it is possible the .aar file code can be decompiled and be regenerated into an .aar with modifications linking to a similar rogue service. This can compromise the security, encryption private key and some other critical data, causing other possible security risks. Zebra recommends application developers to implement the following integrity check to ensure the authenticity of Zebra SDK:

  • The application developer should ensure that the .aar file checksum matches the checksum of the SDK provided by Zebra. The SDK is posted on the Zebra support portal along with its file checksum details. To run the checksum of the SDK, Zebra is recommends using any free “sha512sum” verification tool. The check sum for Zebra Apple VAS SDK (.aar) version 1.0.0 is:

    /* This method will be called when a pass is detected and read during polling */
    public void onPassDetected(Pass pass) 
    {
        if(pass.getPassType() == Pass.Type.AppleVas) {
            if(pass.getResultCode() == Pass.Result.Success) {
                // Decrypted Pass
                DateFormat format = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
                format.setTimeZone(TimeZone.getTimeZone("Etc/UTC"));
                String formatted = format.format(pass.getPayloadDate()) + "Z";
                Log.v(TAG, "--- PASS Received and Decrypted ---");
                Log.v(TAG, "Pass Date: " + formatted);
                Log.v(TAG, "Payload as HEX: " + ByteArrayToHexString(pass.getPayloadMessage()));
                Log.v(TAG, "Payload as ASCII: " + new String(pass.getPayloadMessage(), StandardCharsets.UTF_8));
                Log.v(TAG, "Payload is from pass with Pass Type Id: " + pass.getPassTypeId());
                Log.v(TAG, "This Pass Type Id corresponds to merchantId: " +
        ByteArrayToHexString(pass.getMerchantId()));
            }

            else if(pass.getResultCode() == Pass.Result.IncorrectPrivateKey) {
                // Encrypted Pass
                Log.v(TAG, "--- PASS Received but Wrong Private Key or No Private Key provided for Decryption ---");
                Log.v(TAG, "Encrypted pass payload as HEX: " + ByteArrayToHexString(pass.getPayloadMessage()));
                Log.v(TAG, "Payload is from pass with Pass Type Id: " + pass.getPassTypeId());
                Log.v(TAG, "This Pass Type Id corresponds to merchantId: " +
        ByteArrayToHexString(pass.getMerchantId()));
            }

            else if(pass.getResultCode() == Pass.Result.DecryptionFailure) {
                // Encrypted Pass
                Log.v(TAG, "--- PASS Received but Decryption Failure ---");
                Log.v(TAG, "Encrypted pass payload as HEX: " + ByteArrayToHexString(pass.getPayloadMessage()));
                Log.v(TAG, "Payload is from pass with Pass Type Id: " + pass.getPassTypeId());
                Log.v(TAG, "This Pass Type Id corresponds to merchantId: " +
        ByteArrayToHexString(pass.getMerchantId()));
            }

            else if(pass.getResultCode() == Pass.Result.Failure) {
                // No Pass
                Log.v(TAG, "--- Pass Not Read for some reason - see the Status Word ---");
                Log.v(TAG, "Returned Status Word is: " + ByteArrayToHexString(pass.getPayloadMessage()));
            } 
        }
    }
