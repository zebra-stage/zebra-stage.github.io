关于企业主页屏幕

企业主页屏幕 2.8

概述

企业主页屏幕是 Zebra Technologies 的一款免费 Android 应用程序，它为管理员提供一种控制 Zebra 设备上应用程序和设置访问的简单方法，无需编写自定义代码。通过使用 Simple Touch 界面，EHS 可轻松限制对一个或多个指定应用程序的使用，防止更改设备设置，以及锁定文件系统。可在数秒内对其进行配置，且可通过 MDM 部署设置。如有必要，EHS 设置可存储在一个易于读取和手动编辑的简单 XML 文件中。

EHS 工作原理如下：将其插入 Stock Android 应用程序启动器和主页屏幕。首次运行时，它会显示一个如下所示的屏幕，提供要打开哪个主页应用程序以及是否将选择设为永久。还可以将 EHS 安装为默认启动器，从而绕过选择器对话框。

注意：EHS 的许多功能都可在设备上手动实现，以编程方式通过 EMDK 实现，或使用 StageNow 或第三方移动设备管理 (MDM) 系统（如果受该 MDM 系统支持）远程实现。EHS 只是将功能置于单个工具中。

企业主页屏幕不支持本地化。

EHS 2.8 新增功能

首次安装时，EHS 会为管理员显示此简单选择：

选择“始终”会将 EHS 锁定为默认主页应用程序，且默认针对用户锁定以下设备功能：

  • 飞行模式
  • USB 调试
  • 访问文件系统
  • 锁屏（用于解锁屏幕）
  • 锁屏屏幕摄像头和搜索功能
  • 状态栏设置图标
  • “系统设置”面板的完全访问权限

管理员可授权或限制访问单个功能或完全隐藏设置面板。

注意：选择“始终”后，某些设备在重新启动之前需要暂停 10 秒钟，以确保更改持续生效。

EHS 2.8 新增功能

此版本不支持 Android KitKat。

EHS 2.8 仅支持运行 Android Lollipop 及更高版本的 Zebra 设备。Zebra 继续支持在运行 KitKat 的设备上使用更低版本的 EHS。要在运行 KitKat 的设备上使用 EHS，请下载 EHS 2.6 或更低版本。

设备支持

EHS 2.8 增加对运行 Android Nougat 的以下设备的支持：

  • MC33
  • TC20*
  • TC25*
  • TC51
  • TC56
  • TC70x
  • TC75x

( * ) 需要设备特定的 .apk 文件。请参阅下载页面了解详细信息。

新功能

  • Android Nougat 设备上的启用/禁用最近应用程序按钮
  • 启用/禁用 EHS 日志记录和设置最大日志文件大小
  • 现在，在配备 Zebra Power Precision Plus 电池的设备的电池信息和快速信息屏幕上显示电池健康度
  • 启用/禁用墙纸拉伸以展开图像，从而填充设备屏幕

Android Nougat 备注

适用于运行 Android Nougat 7.x 及更高版本的设备：

  • 为在 EHS 中启用安全模式，Android N 中不再支持手动文件推送来安装设备根证书。要将证书文件部署到设备，请通过 StageNow 或 EMDK 使用 Certificate Manager。了解有关安全模式的更多信息。

Android Marshmallow 备注

仅适用于运行 Android 6.x Marshmallow 的设备：

  • 如果在用户模式下进行选择，则在某些设备上重新启动后，选择 EHS 作为默认启动器将不永久。要避免此问题，请切换至管理员模式，然后按下“主页”按钮；此时将显示“选择主页应用程序”弹出窗口。选择 EHS 并轻触“始终”以进行永久选择。

  • 如果在运行 Android 6.0 Marshmallow 的设备上撤销位置权限，将在 EHS 的“无线信息”屏幕的“扫描结果”下显示空白的可用接入点列表。默认情况下已启用，可通过“设置”->“应用程序” ->“EHS”->“权限”来访问位置权限。

  • 如果在 Marshmallow 设备上禁用 Wi-Fi，则 MAC 地址将在 EHS 的“无线信息”屏幕中显示为“未知”。

Android Lollipop、Marshmallow 和 Nougat 备注

适用于运行 Android Lollipop 5.x 及更高版本（包括 Marshmallow 和 Nougat）的设备：

  • 在运行 Android L、M 或 N 的设备上，无法通过 EHS 控制状态栏下拉菜单（在这些设备上，该功能在“管理员模式首选项”面板中显示为“灰色”）。要控制状态栏下拉菜单（也称为“通知下拉菜单”），请通过 Zebra EMDK 或 StageNow 工具使用 UI Manager。

  • 状态栏设置图标行为已更改。虽然在所有设备上都不支持通过 EHS 禁用此类“设置”面板访问，UI Manager 中的一项新功能允许通过 EMDK、StageNow 或第三方 MDM 系统控制状态栏设置图标（又称为“通知快速设置图标”），从而覆盖任何 EHS 设置。仅适用于具有 MX 6.0 及更高版本的设备，适用于 Android Lollipop 及更高版本；之前的任何设备限制仍适用。

  • 在运行 Android M 或 N 的设备上或一些运行 Android L 的设备上，无法通过 EHS 控制飞行选项已禁用功能。对于“飞行选项已禁用”功能在管理员模式首选项面板中显示为“灰色”的设备，仍可通过 Zebra EMDK 或 StageNow 工具使用 Power Key Manager 访问该功能。

  • 运行 Android L 及更高版本的设备在设备重新启动后保留“最近应用程序”列表，进而会出现潜在的安全风险。EHS 2.8 仅在 Nougat 设备上禁用“最近应用程序”按钮，以帮助应对此风险。对于所有设备，可通过 Zebra EMDK、StageNow 或第三方 MDM 系统使用 App Manager 清除该列表。有关更多信息，请参阅“高级功能”部分中的安全备注。

  • Kiosk 模式不应与屏幕固定一起使用，这是一项 Android L 及更高版本中以类似方式工作的功能。

  • 绕过锁屏功能无法在重新启动运行 Android L 及更高版本的设备后解锁屏幕。此功能在运行 KitKat 和 Jelly Bean 的设备上正常工作。

  • 将 EHS 配置为接受系统方向设置（EHS 默认值）时，可通过仅运行 Android L 及更高版本的设备上的“快速设置”面板更改屏幕方向。如果 EHS 管理员将方向设置为横向或纵向模式，则设备用户将无法再更改方向设置。

ET50/ET55 设备备注

仅适用于具有 Google 移动服务 (GMS) 的 Zebra ET50 和 ET55 设备。

  • 在用户模式下按下“主页”按钮时，启动器选择弹出窗口将不会出现，从而防止用户或管理员在设备上轻松选择默认启动器应用程序。要避免此问题，请切换至管理员模式，然后按下“主页”按钮；此时将显示“选择主页应用程序”弹出窗口。选择 EHS 并轻触“始终”以进行永久选择。

  • 如果在用户模式下远程（即通过 MDM）卸载 EHS，则会出现系统 UI 永久崩溃。在远程卸载 EHS 之前，Zebra 建议向设备推送 EHS 配置文件 (enterprisehomescreen.xml)（在启用搜索应用程序的情况下）。

请参阅高级设置部分以了解完整的 enterprisehomescreen.xml 配置文件参考。

EHS 2.5 新增功能

设备支持：

  • 运行 Android 5.1.1 Lollipop 的 MC32（请参阅 Android 6.x Marshmallow 备注）
  • 运行 Android Marshmallow 6.0.1 的 TC51/TC56 和 TC75x 触摸式数据终端（请参阅 Android 6.x Marshmallow 备注）
  • 运行 Android Lollipop 5.1.1 的 ET50/ET55 平板电脑（请参阅 Android 5+ 备注和 ET50/ET55 设备备注）
  • 运行 Android Lollipop 5.1.1 的 TC8000 触摸式移动数据终端和扫描器（请参阅 Android 5+ 备注）
  • 运行 Android Lollipop 5.1.1 的 TC70/TC75 触摸式数据终端（请参阅 Android 5+ 备注）
  • 运行 Android 4.4 KitKat 的 MC67 移动数据终端系列（请参阅 Android 4.4 备注）

2017 年 6 月更新：

  • 增加了针对运行 Lollipop 的 MC18 和 MC40 的支持
  • 增加了针对运行 Marshmallow 的 TC70x 的支持

EHS 2.5 不再支持：

  • 采用 Android 4.1.2 Jelly Bean 的设备
  • 采用 Android 4.4.2 KitKat 的 TC70（BSP 版本 112414）
    对于采用 Android 4.4.2 KitKat 的 TC70 和运行 Jelly Bean 的 Zebra 设备，Zebra 继续支持 EHS。下载 EHS 2.4 以定向这些设备。

EHS 2.5 中的新功能：

  • 更易于下载和安装。现在，EHS 2.5 下载包包括单个 APK，用于在所有支持的 Zebra 设备上安装；不再具有多 APK 安装程序。

  • 立即链接到 Techdocs，这是获取 Zebra 面向用户文档的重要网站，现在是每次 EHS 安装时的标准组件。

EHS 2.4 新增功能

  • 支持用于运行 Android 5.0 Lollipop 的 Zebra WT6000 - 佩戴式终端。（请参阅 Android 备注）

  • 包数据选项 - 允许在启动时将应用程序参数、用户数据或其他键-值对注入到应用程序中。

  • 应用程序启动标志 - 允许在启动应用程序时指定一个或多个 Android Intent 标志，从而根据需要指导启动行为。

  • 服务自动启动 - 只要启动 EHS 就允许启动一个或多个 Android 服务。

  • 通配符 - 减少 EHS 配置文件中所需的代码行，以指定应用程序以用户模式显示。

  • 自定义应用程序图标 - 允许针对应用程序和链接自定义图标，从而覆盖默认系统图标。

  • 自定义标题栏图标 - 允许自定义图标显示在 EHS 标题栏中，从而覆盖默认 EHS 图标。

  • 应用程序图标大小 - 为管理员和用户模式应用程序图标提供五种大小。

  • 电池和 Wi-Fi 快速视图 - 在单一屏幕上显示设备电池和 Wi-Fi 信号的实时状态。

  • 应用程序图标标签文本颜色 - 现在采用透明背景、白字，以提高可读性。

EHS 2.3 新增功能

  • 启用/禁用设备重新启动 - 适用于 EHS 在安装后首次启动时。
  • TC8000 KitKat 版本 4.4 支持（2016 年 1 月新增）
