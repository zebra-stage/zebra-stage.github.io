企业主页屏幕设置

企业主页屏幕 2.8

概述

这些说明旨在提供有关企业主页屏幕的安装、配置、使用和基本工作原理的基础知识，建议任何不熟悉 EHS 首次设置的人员都阅读这些说明。请注意，此处详述的一切内容都可通过移动设备管理 (MDM) 系统实现自动化，并可利用 EHS config 文件直接操作，此文件在高级设置部分中记录。

EHS 的许多功能都可在设备上手动实现，以编程方式通过 EMDK 实现，或使用 StageNow 或第三方移动设备管理 (MDM) 系统（如果受该 MDM 系统支持）远程实现。EHS 只是将这些功能整合到单个方便的工具中。

注意：要进行安装和设置，该 EHS.apk 文件需要存在于可与目标设备通信并可写入所有存储区域的 PC 或 Mac 上。如有必要，请下载 EHS 并在计算机和设备之间建立连接。然后，从此处继续。

手动安装

这些说明适用于使用 EHS 2.8 安装程序 (.apk) 文件直接安装（从计算机安装到单个设备）。

安装程序 (.apk) 文件信息

EHS 2.8 下载包括两个单独的 .apk 文件：

  • EHS_0207xx_A.apk - 适用于除 TC20/TC25 之外的所有设备
  • EHS_0207xx_B.apk - 适用于 TC20/TC25 设备

请参阅 EHS v2.8 支持的设备的完整列表。

请阅读本指南后面有关如何卸载 EHS 的内容。

7.轻触或按下 HOME 键。将出现类似如下图像的屏幕：


8.现在选择企业主页屏幕并轻触“仅此一次”。这允许在设置期间轻松在 EHS 与 Android 启动器之间切换。每次轻触 HOME 键时都会显示此选项，直至选择“始终”永久性选择该选项。

注意：选择“始终”后，某些设备在重新启动之前需要暂停 10 秒钟，以确保更改持续生效。

用户设置访问权限

EHS 提供对设备设置（例如网络）的广泛控制，且当设备处于“用户模式”下时，用户可使用资源（例如摄像头和文件系统）。管理员可以允许完全控制所有设备设置，限制只可访问几个设置，或完全隐藏设备设置。

在“用户模式”下，默认情况下将禁用以下功能：

  • 飞行模式
  • USB 调试
  • 访问文件系统
  • 锁屏（用于解锁屏幕）
  • 锁屏屏幕摄像头和搜索功能
  • 状态栏设置图标
  • “系统设置”面板的完全访问权限

要选择设备在“用户模式”下的可用设置和/或修改其他用户模式首选项：

14.在“管理员模式”下，从菜单中选择“首选项”：


15.滚动“首选项”面板，然后选择所需的设置：

自动安装

这些说明适用于从组织自己的 MDM 服务器将 EHS 和/或 enterprisehomescreen.xml 配置文件远程安装到多个托管设备上。或者，可通过 Zebra EMDK 或 StageNow 工具使用其 App Manager 设置类型服务完成远程部署和管理。

重要说明：要在启用托管配置文件的设备中执行 EHS，必须在设备中配置托管配置文件之前安装 EHS。

要使用 MDM 安装 EHS，请指示 MDM 按以下顺序执行以下操作：

1.如果需要，创建 enterprisehomescreen.xml 文件，其中包含所需的用户应用程序和设置。否则，将使用默认设置和应用程序安装 EHS。有关创建和编辑配置文件的详细说明，请参阅高级设置部分。

2.部署 EHS 和配置文件（如有）到设备。这包括 .apk 文件和在上一步中创建的 XML 文件（如有）。

3.使用以下软件包和活动命令启动应用程序：

已部署文件

包名称

主活动/类名称

EHS_020xxx.apk

com.symbol.enterprisehomescreen

com.symbol.enterprisehomescreen.HomeScreenActivity


4.将企业主页屏幕设置为设备上的默认主页应用程序。

5.重新启动设备以完成安装。这是必需步骤。

MDM 系统的功能有所不同。有关如何配置这些命令的具体信息，请参阅 MDM 文档。

注意：应用程序安装、卸载、设置默认启动器和其他操作都可通过 Zebra EMDK 或 StageNow 工具使用 App Manager 完成。

卸载

重要说明

在运行 Android 5.x Lollipop 和 GMS 的 ET50/ET55 设备上，如果未在“用户模式”下安装 EHS，则可能会发生永久性的系统 UI 崩溃。在卸载 EHS 之前，Zebra 建议向设备推送 EHS 配置文件 (enterprisehomescreen.xml)（在启用搜索应用程序的情况下）。有关配置文件的使用，请参阅高级设置部分。

手动卸载

重要说明：EHS 配置的设备设置（例如 USB 调试、系统设置受限制和一些其他设置）可能不会恢复到安装 EHS 之前的状态。Zebra 建议在删除 EHS 之前根据需要重新配置设备。

要删除 EHS，只需使用 Android App Manager 卸载 EHS 应用程序。

1.从“管理员模式”屏幕中选择设置：

2.轻触“应用程序”以弹出 Android App Manager：

3.找到并轻触“企业主页屏幕”：

4.轻触“卸载”按钮：

5.单击“确定”进行确认：

可选

以上步骤不会从 /enterprise/usr 文件夹中删除 enterprisehomescreen.xml 配置文件，这可能会影响以后安装的 EHS 版本的行为。

要删除旧版本的 enterprisehomescreen.xml 文件：

6.将设备连接至安装有 Android Debug Bridge (ADB) 的计算机。

7.在命令提示符处，执行以下命令：

adb shell rm /enterprise/usr/enterprisehomescreen.xml

EHS 手动卸载现已完成。

自动卸载

这些说明适用于使用组织自己的 MDM 服务器进行远程卸载，以从多个托管设备上删除 EHS。此任务也可以通过 Zebra EMDK 或 StageNow 工具使用 App Manager 服务完成。

重要说明

  • 在带 GMS 的 ET50 和 ET55 设备上，如果在“用户模式”下远程卸载 EHS（即通过 MDM），可能会导致永久性的系统 UI 崩溃。在卸载 EHS 之前，Zebra 建议向设备推送 EHS 配置文件 (enterprisehomescreen.xml)（在启用搜索应用程序的情况下）。有关配置文件的使用，请参阅高级设置部分。

  • EHS 配置的设备设置（例如 USB 调试、系统设置受限制和一些其他设置）可能不会恢复到安装 EHS 之前的状态。Zebra 建议在删除 EHS 之前根据需要重新配置设备。

要使用 MDM 卸载 EHS，请指示 MDM 执行以下操作：

1.卸载（删除）EHS 应用程序，其中参考软件包和活动名称，如下表所示。

已部署文件

包名称

主活动/类名称

EHS_020xxx.apk

com.symbol.enterprisehomescreen

com.symbol.enterprisehomescreen.HomeScreenActivity


2.可选：如果新版本的 EHS 将部署新 enterprisehomescreen.xml 配置文件，请跳到步骤 3。否则，指示 MDM 从设备中删除文件 /enterprise/usr/enterprisehomescreen.xml。这将从之前安装的 EHS 应用程序中清除配置设置。

更改默认启动器

可在“设置”面板中使用简单的更改或通过卸载 EHS 将 EHS 作为设备上的默认启动器删除。其中任一场景都可在设备上手动实现，以编程方式通过 EMDK 实现，或使用 StageNow 或 MDM（如果支持）远程实现。

如果从设备中删除 EHS，且 Android 启动器是设备上唯一剩余的启动器，它将成为默认启动器。如果删除 EHS 后，设备上仍有多个启动器，应选择一个新的默认启动器，以确保所需操作。

以下流程仅适用于 Android 4.4.3 及更高版本。要将 EHS 作为 4.4.3 之前的 Android 版本上的默认启动器删除，请按照（上述）说明使用 Android App Manager 手动卸载 EHS。

要手动更改设备上的默认启动器而不删除 EHS：（Android 4.4.3 及更高版本）

1.在“管理员模式”下，打开“设置”面板并轻触“主页”：

2.轻触“Android 启动器”，选择其作为默认启动器和主页屏幕。

注意：此处看到的回收站图标提供了一种卸载 EHS 和恢复 Android 启动器的备用方法。
