管理対象の構成の使用

Zebra EHS 6.0

概要

管理対象の構成は、Google と Android コミュニティによって開発された仕様の一部で、これをサポートするエンタープライズ モビリティ管理 (EMM) システムを通じて、インストールされているアプリケーションとデバイスをリモートで構成するためのものです。管理対象の構成 (MC) は、当初の範囲が限られていたため、当初は「アプリ制限」として知られていましたが、現在では、デバイスのハードウェア、ソフトウェア、セキュリティなどのさまざまな設定をリモートで構成するために Zebra で使用されています。

MC を使用して管理できる特定のアプリの機能は、そのスキーマで定義されます。EHS スキーマは、EMM コンソールを通じて Google Play にアクセスし、管理用に EHS を選択するとダウンロードされます。スキーマは、EMM で使用できる機能を定義し、EMM コンソール内でアプリの管理 UI を表示するために必要な情報を提供します。†このデータ駆動型 UI メソッドを使用すると、新しい .exe をダウンロードすることなく、新しい機能とそれに対応する UI 属性を利用可能になるとすぐに配信できます。管理対象の構成を使用した Zebra デバイスの構成の詳細については、Zebra OEMConfig を参照してください。

†EHS 管理 UI は、使用している EMM システムによって若干異なります。

MC を使用した EHS の構成

現時点では、すべての EHS 設定は、 EnterpriseHomeScreen.xml 構成ファイルを介してのみ管理できます。今後 Zebra では、MC を通じて管理できる機能を増やし、すべての機能が含まれるようにする予定です。その時点で、EnterpriseHomeScreen.xml 構成ファイルは廃止予定です。

現在の MC の機能:

  • EHS 設定ファイルを企業全体に安全に大量展開する
  • デバイス ユーザーのアクセスを承認されたアプリのみに制限する
  • デバイスのアプリや設定への不正な変更を防止する

要件

  • Android 11 以降を実行する 1 台以上の Zebra デバイス
  • EHS を備えたサポートされている EMM と、Google Play からダウンロードされたそのスキーマ
  • EHS 5.0 (以降) がデバイスにインストールされている (必要に応じて EMM によって展開可能)
