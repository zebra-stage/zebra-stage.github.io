Tread Depth Reader API

Tread Depth Reader 1.0

Overview

Tread Depth Reader API is designed based on the Android Intent architecture, reducing library dependencies and integration time for app developers. To ensure the interface is secure, it uses explicit and pending intents.

This document offers a comprehensive guide on accessing Tread Depth Reader APIs. It describes the API components necessary for interaction and fulfills functional and non-functional requirements for applications that retrieving tread data from the reader accessory.

All the return data, including tread data, state changes, errors and exceptions are reported through the Android onActivtyResult() method.

APIs available:

  • INITIATE – Initiate Reader framework session.
  • START – Start measurement and send data back to the application via pending intent.
  • RELEASE – Release Reader framework session.
  • GET_PARAMS – Get the Tread Reader parameter values.
  • SET_PARAMS – Set the Tread Reader parameter values.
  • VERSION – Get the Tread Reader framework version(set) data.
  • CURRENT_STATE - Querying the Tread Reader Framework, Connection State and Framework State
  • ON_STATE_CHANGE - Notify when any given time if the framework Connection State or Framework State change.

Requirements

Requirements must be met to ensure the successful invocation of intent APIs and obtain accurate readings from the Tire Tread Depth reader.

1. Add Queries Tag

For Android 13 devices and higher, add the <queries> tag in the AndroidManifest.xml of the Android application project. This defines the package name for the Tread Depth Reader and is required to allow the Tread Depth Reader API to communicate with the application.

<queries>
    <package android:name="com.zebra.intentapis" />
</queries>

Create a Java class named IntentAPIConstantUtils within the Android application project and add the code below. This helps manage and maintain clean, consistent values throughout the project.

APIs

Initiate

INITIATE starts the Tread Reader service and must be called only once within an application's lifecycle before any other APIs. This ensures the Tread Reader APIs are enabled and can communicate with the application. To properly start the Tread Reader service, INITIATE must be called using the startForegroundService() method.

Request

Use the following action to call INITIATE:

    com.zebra.intentapis.action.TTDR_INITIATE

Package: com.zebra.intentapis

Request Keys:

Key Type Description/Value
TTDR_CALLBACK_RESPONSE PendingIntent Receives the response


Sample code:

    Intent intent = new Intent();
    intent.setPackage(IntentAPIConstantUtils.TTDR_SDK_PACKAGE);
    intent.setAction(IntentAPIConstantUtils.TTDR_INTENT_ACTION_INITIATE_READER);
    PendingIntent pendingIntent = createPendingResult(IntentAPIConstantUtils.TTDR_REQUEST_CODE_START_FRAMEWORK, new Intent(), PendingIntent.FLAG_UPDATE_CURRENT);
    intent.putExtra(IntentAPIConstantUtils.TTDR_CALLBACK_RESPONSE, pendingIntent);
    startForegroundService(intent);

Response

The following key-value pairs are received in the onActivityResult() method after invoking INITIATE API.

Key Type Description/Value
TTDR_ACTION String com.zebra.intentapis.action.TTDR_INITIATE
TTDR_ACTION_STATUS Integer -1 = Success
0 = Failed


Exception: In the event that the framework throws an exception while invoking INITIATE api below mentioned key value pairs should expected as response to the onActivityResult() method.

Key Type Description/Value
TTDR_ACTION String

com.zebra.intentapis.action.TTDR_EXCEPTION
TTDR_INTENT_EXCEPTION_OCCURRED_ACTION String com.zebra.intentapis.action.TTDR_INITIATE
TTDR_INTENT_EXCEPTION_NUMBER Integer Integer value that assigns for segregates exception
TTDR_INTENT_EXCEPTION_OCCURRED_ACTION String Description of the exception occurred

Sample Code

    protected void onActivityResult(int requestCode, int resultCode, @Nullable Intent data) {
        super.onActivityResult(requestCode, resultCode, data);

        if (requestCode == IntentAPIConstantUtils.TTDR_REQUEST_CODE_START_FRAMEWORK) {    
            String mACTION = data.getStringExtra(IntentAPIConstantUtils.TTDR_ACTION);

            if (Objects.equals(mACTION, IntentAPIConstantUtils.TDR_INTENT_ACTION_INITIATE_READER)){
                int mACTION_STATUS = data.getIntExtra(IntentAPIConstantUtils.TTDR_ACTION_STATUS,0);
                if (mACTION_STATUS == IntentAPIConstantUtils.TTDR_ACTION_SUCCESS) {
                    Log.e(mACTION,"ACTION SUCCESS");
                }
                else if (mACTION_STATUS == IntentAPIConstantUtils.TTDR_ACTION_FAILED){
                    Log.e(mACTION,"ACTION FAILED");
                }  
            } 

            else if (Objects.equals(mACTION, IntentAPIConstantUtils.TTDR_INTENT_ACTION_EXCEPTION)) {
                String mExceptionOccurredAction = data.getStringExtra(IntentAPIConstantUtils.TTDR_INTENT_EXCEPTION_OCCURRED_ACTION);
                int mExceptionNumber = data.getIntExtra(IntentAPIConstantUtils.TTDR_INTENT_EXCEPTION_NUMBER,0);
                String mExceptionDescription = data.getStringExtra(IntentAPIConstantUtils.TTDR_INTENT_EXCEPTION_DESCRIPTION);
            }
        }
    }

Tread Data

Tread data is received to the application as a list which contains tread depth readings on inner, center and outer treads in Json format, each object containing four attributes: confidenceLevel, metricTreadDepth, position, status.

Attribute Description/Value
metricTreadDepth Tread Depth measurement provided in units of millimeters (mm). The host application can convert mm to inches by multiplying this variable by 0.03937.
Example Conversation from mm to inches:
3.97 mm * 0.03937 = 0.1562989”
0.1562989” is equivalent to 5/32”
Note: 1/32” = 0.03125”
position Enum reporting the position of the Tread Depth
confidenceLevel This value represents the confidence in the measurement taken. The value range is 0 – 100, with 0 representing the lowest confidence and 100 representing the highest confidence. This value will vary based on factors like swipe speed, environmental conditions and tire complexity (e.g. tread geometry and patterns). As a guideline, the following thresholds should be considered for judging confidence level:
     81-100 : High confidence measurement, generally user swipe was controlled and tread measurement is reliable.
     61-80 : Measurement is acceptable. A mid-range score may indicate that the measurement quality is degraded due to tread geometry and/or user swipe behavior. As an example, a narrow tread and fast swipe speed could create conditions where the confidence factor is lower than 80.
     0-60 : Lowest confidence measurement. Levels 60 and lower usually indicate that the swipe behavior was not stable or larger than typical measurement variation.
status Enum represents a status of the Tread Depth

Tread Reader State

CURRENT_STATE queries the Tread Reader framework, connection state and framework state after the Tread Reader has initiated.

ON_STATE_CHANGE notifies the client app when the framework connection state or framework state changes.

The framework must be initiated prior to making these API calls.

Request

Actions:

    com.zebra.intentapis.action.TTDR_CURRENT_STATE
    com.zebra.intentapis.action.TTDR_ON_STATE_CHANGE

Request Keys:

Key Type Required Description/Value
TTDR_CALLBACK_RESPONSE PendingIntent Receives the response

Sample Code

    private void getCurrentState(){
        Intent intent = new Intent();
        intent.setAction(IntentAPIConstantUtils.TTDR_INTENT_ACTION_CURRENT_STATE);
        PendingIntent currentStatePendingIntent = createPendingResult(IntentAPIConstantUtils.TTDR_REQUEST_CODE_START_FRAMEWORK, new Intent (), PendingIntent.FLAG_UPDATE_CURRENT);
        intent.putExtra(IntentAPIConstantUtils.TTDR_CALLBACK_RESPONSE, currentStatePendingIntent);
        sendBroadcast(intent);
    }

    private void checkForOnChangeState(){
        Intent intent = new Intent();
        intent.setAction(IntentAPIConstantUtils.TTDR_INTENT_ACTION_ON_STATE_CHANGE);
        PendingIntent currentStatePendingIntent = createPendingResult(IntentAPIConstantUtils.TTDR
        _REQUEST_CODE_START_FRAMEWORK, new Intent(), PendingIntent.FLAG_UPDATE_CURRENT);
        intent.putExtra(IntentAPIConstantUtils.TTDR_CALLBACK_RESPONSE, currentStatePendingIntent);
        sendBroadcast(intent);
    }


Response

The following key-value pairs are received to the onActivityResult() method after invoking CURRENT_STATE and ON_STATE_CHANGE APIs.

Key Type Description/Value

Set Parameters

Use SET_PARAMS to configure the Tread Reader parameters based on the functionality desired. The following are required:

  • The Tread Reader framework is initiated.
  • The Tread reader device is connected to the mobile computer.
Parameter Return Type Permissions Description
hardware_version String Read Only Returns the Tread Depth Reader hardware version.
sdk_version String Read Only Returns the version of the SDK. This is updated with any change to the API or interface exposed by the SDK.
service_version String Read Only Returns service version information, which is updated with changes to the tire tread reading algorithm or software changes associated with the reader framework.
firmware_version String Read Only Returns the Tread Depth Reader firmware version information, which is typically updated with service updates.
reswipe String Read / Write Used to flag the next tread measurement session. (i.e. START) as a secondary measurement.
Values:
False = Default value; normal tread measurement session. Set after Enable / Disable transitions.
True = A reswipe measurement session occurred. The tread reading framework will take data from the previous read attempt and combine the best fit measurement candidates. The aggregated result is returned to the application. Note: This parameter is designed to be temporary and applies to the next call to START. After START this parameter is set to the default state “False” which indicates a normal swipe. If a measurement session returns an Error during a reswipe this value will return to it’s default state “False”.
scan_count String Read Only Returns the total number of scans recorded from the Tread Depth Reader Accessory for supported hardware. Returns "Incompatible Hardware" for older accessories.
reader_timeout String Read / Write Set the scan duration of a scan:
• 0010 = Ultra-High (2000 frames/scan)
• 0008 = High (1200 frames/scan)
• 0004 = Mid (1000 frames/scan)
• 0002 = Low (800 frames/scan) - Default

If an exception is thrown, the following key-value pairs would be part of the response to the onActivityResult() method:

Key Type Description/Value

TTDR_INTENT_EXCEPTION_NUMBER Integer Integer value that assigns for segregates exception
TTDR_INTENT_EXCEPTION_DESCRIPTION String Description about the exception that occurred

Exception Codes:

  • INVALID_CONTEXT = Attempted to disable the reader through a closed TreadReader object
  • INVALID_STATE = The Reader Object is not in a proper state to set the parameters
  • INVALID_PARAMETER_VALUE = An invalid value was passed
  • UNKNOWN_PARAMETER = An unknown parameter was passed
  • HARDWARE_ERROR = Unable to communicate with the Tread Depth Reader. Hardware (Critical).
  • REMOTE_COMMUNICATION_ERROR = Unable to communicate with the Framework
  • FRAMEWORK_ERROR = The framework has terminated, is still recovering.
