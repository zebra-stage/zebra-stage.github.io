Set Up Power Input Configurator

Power Input Configurator 1.4

Overview

Zebra Power Input Configurator (zPIC) is an Android app available from the Zebra Support Portal (link below), and is installed as any other Android app. It can be deployed using Zebra StageNow or a company's own EMM system with Zebra OEMConfig on devices with the minimum LifeGuard updates shown below.

Requirements

  • Power Input Configurator (zPIC) app from Zebra Support Portal. Download now.
  • At least one supported Zebra device on which to create power configuration file(s). See supported devices and OS versions.
  • One or more target devices running Android 13 or Android 14 with the corresponding LifeGuard update:
    • KC50 with Android 13: 13-40-29.00-TG-U00 (or later)
    • KC50 with Android 14: 14-28-03.00-UG-U42 (or later)
    • ZEC500 with Android 14: 14-34-20.00-UG-U00 (or later)

Deployment

Once device power settings are configured and exported, the resulting .json file(s) can be deployed across an enterprise.

TESTING DISCLAIMER

Zebra strongly recommends fully testing all configuration files on a small group of devices before wider deployment.

  • Using an Android file app or device browser, navigate to the following device location and locate the saved config files:
    /sdcard/Android/data/com.zebra.powerinputconfigurator/files

  • Copy the desired file(s) onto the host computer.

  • Target devices must be powered off and restarted after receiving zPIC configuration file for its settings to take effect.

• Use StageNow or an EMM system with Zebra OEMConfig to create a deployment Profile using FileManager and Persist Manager (StageNow) the Download File Persistence function (OEMConfig):

  • File Action > “Deploy file for an application”
  • In the Target application file definition field, enter: “com.zebra.powermanagerservice/zpm_config_[user filename].json”
  • (replace “[user filename]” with the actual file name)
  • Apply persistence as needed based on the deployment tool in use
  • After deployment, restart the device to apply the new settings.
  • NOTE: Configurations applied directly to the device running zPIC (and not through the process above) do not persist on the device following an Enterprise Reset.

Configuration File Notes

  • The zPIC app is NOT required on target devices being remotely configured.

  • zPIC automatically names exported files according to the selected power class. FILES NAMES MUST NOT BE CHANGED MANUALLY.

  • A device can contain multiple zPIC-generated configuration files; at device startup, the appropriate config file is selected based on the detected power source type (and power class, if PoE).

  • Location and names of exported config files:
    /sdcard/Android/data/com.zebra.powerinputconfigurator/files/ zpm_config_KC50S_STD_DC.json zpm_config_KC50S_STD_PoE_class4.json zpm_config_KC50S_STD_PoE_class6.json zpm_config_KC50S_STD_PoE_class8.json

  • The targeted device’s model number is embedded within exported file names.

  • Use StageNow or an EMM system with Zebra OEMConfig to create a deployment Profile using FileManager and Persist Manager (StageNow) the Download File Persistence function (OEMConfig):

    • File Action > “Deploy file for an application”
    • In the Target application file definition field, enter: “com.zebra.powermanagerservice/zpm_config_[user filename].json”
    • (replace “[user filename]” with the actual file name)
    • Apply persistence as needed based on the deployment tool in use

  • After deployment, restart the device to apply the new settings.

  • NOTE: Configurations applied directly to the device running zPIC (and not through the process above) do not persist on the device following an Enterprise Reset.
