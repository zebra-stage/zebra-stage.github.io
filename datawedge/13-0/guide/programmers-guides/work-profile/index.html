Work Profile

DataWedge 13.0

Overview

Formerly known as Corporate-Owned, Personally Enabled (COPE) mode, Android work profile enables organizations to enforce corporate policies and restrictions while keeping personal data private. It supports the utilization of a single device for both work and personal use. Work profile ensures the protection and privacy of corporate data while keeping personal applications separate and private, thereby maintaining end-user privacy.

Devices provisioned with work profile offer the convenience of switching between work or personal profile use. To use DataWedge in work profile, both DataWedge and the scanning framework must be enabled as system applications within the work profile.

Zebra recommends device administrators to control DataWedge access and configuration in the work profile by utilizing DataWedge Manager CSP. This includes preventing a user from accessing the DataWedge UI and preventing automatic import of configuration files.

Requirements

  • Zebra Android 13 devices or higher
  • DataWedge v13.0.210 or higher

Enable DataWedge

For DataWedge to operate in work profile mode, DataWedge and the scanning framework must be enabled as system apps within the work profile. Follow the steps below to perform this. These steps may vary depending on the enterprise mobility management (EMM) system in use.

  1. Open Device Policy Controller from your EMM.
  2. Under Policy management, tap Enable system apps.
  3. Tap DataWedge.
  4. Tap Enable system apps again.
  5. Tap Scanning Framework.
  6. The work profile now has DataWedge and the scanning framework enabled. Both DataWedge and DWDemo are visible within the work profile.

Note: After enabling DataWedge in the work profile, immediate attempts to open it may not be successful due to the loading time it requires. However, subsequent attempts should open it successfully.

DataWedge Supported Features

DataWedge only supports the following features in work profile:

  • Barcode Input – Scanning barcodes via an internal imager
  • Keystroke Output – Dispatch the scanned data via Keystrokes
  • Intent Output - Dispatch the scanned data via Intents
  • IP Output - Dispatch the scanned data to a specified IP address using TCP or UDP
  • Data Capture Plus – On-screen button to trigger scanning
  • ADF – Advanced data formatting rules
  • BDF – Basic data formatting rules
  • Import/Export DataWedge configurations
  • Mass deployment

Limitations

Limitations of DataWedge in work profile:

  1. Since SD card support is no longer available starting with Android 13, the SD card file path cannot be used to copy the exported DataWedge configuration file to a host PC.
  2. The following DataWedge features are not supported in work profile:
    • Voice Input
    • Serial Input
    • Scanning using the Launcher profile

Personal Profile Behavior

After enabling DataWedge and the scanning framework in the work profile, DataWedge remains visible in the personal profile but will not be functional. This is due to the fact that DataWedge can operate in only one profile at a time. For DataWedge to function in the personal profile, a factory reset is required.

Note: If DataWedge was in use in the personal profile just before it is enabled in the work profile, it will automatically shut down in the personal profile. This occurs because DataWedge becomes inoperative in the personal profile after being enabled in the work profile.
