概述

DataWedge API 示例应用程序演示直到版本 6.8（创建时的最新版本）的所有主要 DataWedge Intent API 的功能。要利用此示例应用程序，需要对 DataWedge 有基本的了解。有关 DataWedge API 的更多信息以及有关此示例应用程序的深入了解，请参阅 Zebra 工程师 Darryn Campbell 编著的 DataWedge API - 优势和使用场景一文（此 DataWedge API 示例源自此文）。

注意：此应用程序仅以演示为目的。它按原样提供（无保证或担保），可根据个人需求进行修改。

使用的 API

API (6.x)

  • SoftScanTrigger - 用于启动、停止或切换软件扫描触发器
  • ScannerInputPlugin - 启用/禁用活动配置文件使用的扫描器插件
  • enumerateScanners - 返回设备上可用的扫描器的列表
  • setDefaultProfile - 用于将指定的配置文件设置为默认配置文件
  • resetDefaultProfile - 用于将默认配置文件重置为 Profile0
  • switchToProfile - 切换到指定的配置文件

API (6.2)：

  • 删除配置文件
  • 克隆配置文件
  • 重命名配置文件
  • 获取活动配置文件
  • 列出配置文件
  • 启用/禁用 Datawedge

API (6.3)：

  • 获取版本信息
  • 注册/撤消注册扫描器通知
  • 创建配置文件
  • 设置配置
  • 恢复配置
  • 首先为 6.x 定义的 API 的版本 6.3 变体，例如 SoftScanTrigger、ScannerInputPlugin 等。

API (6.4)：

  • 设置配置（使用“自动”扫描器）
  • 获取 Datawedge 状态

API (6.5)：

  • 为支持的 API 提供结果代码（大部分是 Set...API）
  • 获取配置
  • 获取已禁用的应用程序列表
  • 设置已禁用应用程序列表
  • 切换扫描器
  • 切换扫描器参数

API (6.6)：

  • 设置报告选项
  • 按友好名称切换扫描器

API (6.7)

  • 设置导入配置

API (6.8)

  • 忽略已禁用配置文件
  • 获取“忽略已禁用配置文件”状态
  • 切换 Simulscan 参数

要求

  • 创建此示例是为了支持直到版本 6.8（在创建时存在的版本）的 DataWedge API。查看 DataWedge API 文档以了解可能影响功能的任何未来版本更改。
  • DataWedge（内置于所有 Zebra 设备中）正在设备上运行。

设置

设置与此应用程序关联的 Datawedge 配置文件：

  1. 在“DataWedge 6.3 API”部分下的“创建配置文件”中，轻触“发送”命令。

  2. 检查是否使用以下设置在 DataWedge 中创建了名为“DW API 练习程序配置文件”的新配置文件：

    • 配置文件：已启用
    • 条码输入：已启用
    • Intent 输出：已启用
    • Intent 操作：com.zebra.dwapiexerciser.ACTION
    • Intent 类别：[留空]

使用

安装并启动示例应用程序。API 功能按 DataWedge API 版本进行分类。要快速启动并运行，请在“DataWedge 6.3 API”部分中的“创建配置文件”选项下，轻触发送。这会在 DataWedge 中创建“DW API 练习程序配置文件”，并将此应用程序关联到该配置文件。它可以针对 Intent 广播进行注册，以从 DataWedge 接收扫描的数据。

图 3：创建配置文件

轻触发送按钮以对任何命令执行操作。发送命令之后，一个弹出消息会出现，指示发送的命令的状态。扫描条码之后，会在顶部填充数据以及扫描源和解码器信息。
