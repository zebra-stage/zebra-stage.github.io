Document Capture Programmer's Guide

DataWedge 14.1

Overview

Document Capture, a feature of NextGen (NG) SimulScan, is a data capture solution designed to extract information from documents, forms, and labels. It operates by scanning barcodes or capturing images of entire or partial documents. Using NG SimulScan templates, it specifies the data to be captured from barcodes or images and delivers this information to the application for further processing.

Requirements

  • Refer to the Feature Matrtix for devices that support Document Capture
  • A camera or supported 2D imager
  • DataWedge v8.0.37 or higher, configured to access NG SimulScan
  • A Mobility DNA license is required to use the Document Capture feature of NG SimulScan

Document Capture

Document Capture Implementation:

1. Generate the Template

Templates are important for identifying the data to be captured and processed. Each template corresponds to a specific document type, such as a shipping label or manufacturing form. The template defines both the Form region (entire document) and the Field regions (specific elements like barcodes, text, or images) for data capture.

Consult Template Builder for instructions to create the template.

2. Deploy Template to Device

After creating the template, transfer it to the device. Use StageNow, leveraging DataWedge Mgr CSP, to securely import the template for DataWedge use. Either generate a StageNow barcode for scanning to deploy the template to the device, or use an EMM (Enterprise Mobile Management) system to deploy the .XML file created through StageNow.

For step-by-step instructions, refer to Import NextGen SimulScan Templates.

3. Create the DataWedge Profile

Follow steps to create the DataWedge profile and associate it to the application that is receiving the captured data.

4. Configure the Profile

Configure the DataWedge profile for Document Capture:

  1. In the NG SimulScan Configuration, set the Scanning mode to Document Capture.
  2. Select the imported template in Document Selection. The selection is populated after template deployment.

5. Use Content Provider

Use DataWedge's content provider to acquire, process, and output the data from document capture. Enable the Use Content Providers option under Intent Output in the DataWedge profile. Refer to the Content Provider Programmer's Guide for instructions on retrieving and processing the data for application output.
