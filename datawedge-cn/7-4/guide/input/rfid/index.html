RFID 输入

DataWedge 7.4

概述

无线射频识别 (RFID) 输入使用集成的 RFID Zebra 移动计算终端从 RFID 标签获取数据。目前仅 MC3300R 支持此功能。RFID 是 DataWedge 的另一个输入源 - 数据传送与条码和 MSR（磁条读取器）等其他输入源类似。采集的数据将根据配置文件按键输出部分中的基本数据格式 (BDF) 和高级数据格式 (ADF) 选项进行处理和格式化。

重要说明：有关初始设置的说明，请参阅 MC3300R 集成商指南中的“设置 MC3300R”一节。

版本历史记录

  • DataWedge 7.3.22 - 引入功能

RFID 输入

DataWedge 配置文件中的 RFID 输入提供切换 RFID 输入、指定触发器模式和配置 RFID 读取器设置的选项。


已启用 - 启用/禁用 RFID 标签读取
读取器选择 - 显示内部 RFID 读取器的名称
硬件触发器 - 启用/禁用硬件触发器以激活 RFID 读取。如果启用，则设置触发器模式。
配置读取器设置：

  • 过滤重复标签 - 启用/禁用只显示一次已读取数据的功能，而不是从多个读数复制相同的数据。
  • 标签读取持续时间 - 事件传播到应用程序的时间量（以毫秒为单位）。如果在超时之前释放触发器，则数据会在触发器释放后返回到应用程序。返回到应用程序的数据仅限于 Android 捆绑包有效负载的容量 - 超过该容量的已读取数据都将被丢弃。
  • 天线发射功率 - RFID 读取器要传输的信号功率大小。值越高，就越能够读取 RFID 标签。范围是 5 至 30 dBm。
  • 会话 - 配置读取会话。有关详细信息，请参阅 EPC UHF Gen2 空中接口协议标准。支持的值：
    • 会话 0 - 基于 EPC UHF Gen2 空中接口协议标准规范
    • 会话 1（默认）- 基于 EPC UHF Gen2 空中接口协议标准规范
    • 会话 2 - 基于 EPC UHF Gen2 空中接口协议标准规范
    • 会话 3 - 基于 EPC UHF Gen2 空中接口协议标准规范
  • 存储体 - 配置要读取的存储体。某些标签上的集成电路 (IC) 在存储体之间有不同的位分配，以便有更多的用户存储器或更长的 EPC（电子产品代码）编号。请参阅标记规格以确定所需的存储体类型。支持的值：
    • 无（默认值）- 没有要读取的存储体。
    • 保留 - 存储标签的访问和终止密码，每个 32 位。终止密码会永久禁用该标记。访问密码用于锁定和解锁标签的写入功能。因此，如果指定了密码，则可以进行写操作。通常在应用程序包含敏感数据时使用。
    • EPC - 用最少 96 位的存储器存储 EPC 代码。
    • 用户 -（可选）扩展存储器，用于存储更多信息。存储器大小可以有多种，通常为 512 位，最高 8K。可用于不适合 EPC 存储器的附加数据，如温度或到期日期。
    • TID（标签标识）- 存储制造商提供的唯一标签标识符，通常无法更改。
  • 触发模式 – 选择激活 RFID 读取器的模式。支持的值：
    • 立即（默认）- 根据按下触发器的时间读取标签。
    • 连续 - 在按下触发器一次后连续读取标签。再次按下触发器可终止标签读取。
  • 反馈 – 配置读取 RFID 标签的音频或视觉反馈。选项：
    • 蜂鸣器 - 音频反馈
    • LED - 视觉反馈

按键映射

MC3300R 包含多个触发器，默认情况下都用于条码和 SimulScan 功能。要将硬件触发器用于 RFID 操作，需要将其映射到 RFID。要遵循的步骤：

手册说明：

  1. 打开 Android 设置。
  2. 轻触 Key Programmer。
  3. 轻触 SYMBOL_RFID.

CSP（配置服务提供商）说明：对于 CSP，请使用 StageNow 在设备部署期间更新按键映射。有关更多信息，请参阅 KeyMapping Manager。

数据

有关 RFID 数据读取的重要说明：

  • 默认情况下，在读取每个标签后都会添加换行符。
  • 适用情况下，如果同时读取多个标签，则可以返回来自多个数据读取的数据集。
  • 如果通过按键输出接收数据，则需要将字符间延迟设置为 50 毫秒。


RWDemo

RWDemo 用于展示 DataWedge 中的 RFID 功能。此应用程序预装在受支持的 Zebra RFID 设备（如 MC33R）上，在启动时会创建与 RWDemo 关联的 DataWedge 配置文件。
