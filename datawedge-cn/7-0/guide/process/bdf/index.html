基本数据格式

DataWedge 7.0

概述

通过“输出插件”将其发送到关联应用程序或服务器之前，处理插件以指定方式操作已获取的数据。处理插件控件显示为“基本数据格式”和“高级数据格式”，并与其输出插件一起在应用程序的设置面板中分组。

处理插件指定：

  • 基本数据格式（即使用按键附加前缀、后缀等）
  • 高级数据格式（基于规则的数据操作、操作触发器等）

基本格式进程插件允许 DataWedge 在将其传递到输出插件之前将前缀和/或后缀添加到捕获的数据。它还允许插入 TAB 和 ENTER 按键，可用于将光标从应用程序的一个字段移动到另一字段，以促进一系列数据获取任务。

如果需要，BDF 还可将获取的数据转换为十六进制符号。例如，如果获取的条码数据是 012345，则此选项可转换和发送 303132333435 的十六进制等效数据。

基本数据格式

基本数据格式提供一种简单的方法：在将其传递到“输出插件”之前，可以使用自定义值或按键附加或前置已获取的值。它还允许将数据转换为十六进制格式。如果未启用“基本数据格式”，则将捕获的数据传递到选定“输出插件”，而不修改。

基本数据格式输出选项

数据前缀 - 在发送之前将指定字符添加（前置）到已获取数据的开头。

数据后缀 - 在发送之前将指定字符添加（附加）到已获取数据的末尾。

发送数据 - 当来到前台时允许将捕获的数据传输到关联的应用程序。默认情况下处于启用状态。
注意：禁用此选项仅防止传输已捕获的数据；任何前缀和/或后缀字符串将始终传递到关联的应用程序，即使此选项处于禁用状态也是如此。

以十六进制格式发送 - 以十六进制格式发送数据。例如，如果已获取的条码数据是 012345，则此选项会发送 303132333435 的十六进制等效项。

发送 TAB 键 - 将 TAB 字符附加到已处理的数据。

发送 ENTER 键 - 将 Enter 字符附加到已处理的数据。

DataWedge 输出选项：

  • 按键 - 输出已获取的数据，如同按下键盘
  • 互联网协议 - 使用 TCP 或 UDP 通过网络输出数据
  • Intent - 将数据作为 Intent 额外项交付到应用程序
