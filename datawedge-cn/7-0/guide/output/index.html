输出插件

DataWedge 7.0

概述

输出插件将已处理的数据发送到关联的应用程序或服务器。对 DataWedge 中包含的内容说明如下。

按键输出插件收集已处理的数据，并将其作为一系列按键发送到关联的应用程序，从而仿真用户按键操作。

Intent 输出插件将已处理的数据作为 Android Intent 内的有效负载发送到前台应用程序。

IP 输出插件允许将已捕获的数据发送到使用 TCP 或 UDP 传输协议的指定 IP 地址和端口，以及发送到运行 Zebra IPWedge 软件的 Windows 服务器。有关更多信息，请参阅 IP 输出指南。

DataWedge 输出选项：

  • Intent（编程数据越区切换）
  • 互联网协议（通过 TCP 或 UDP 到 PC 或服务器）
  • 按键（键盘仿真）

“自动”选项将根据下面的规则自动从可用设备列表确定最佳扫描设备。

自动扫描器选择规则：

  • 如果已安装 Zebra 扫描模块或扫描/MSR 模块，将选定 2D 成像器。
  • 如果没有安装扫描模块，将选择摄像头。
  • 选择摄像头时，将使用后置摄像头执行扫描。
  • 当选定 2D 成像器时，将使用已安装的扫描或扫描/MSR 模块执行扫描。

蓝牙扫描器

DataWedge 支持下列 Zebra 蓝牙扫描器：

  • RS507 无绳指环式扫描器
  • RS6000 指环式扫描器
  • DS3678 超耐用型扫描器

根据以下规则，支持蓝牙扫描器：

  • 必须配对和连接扫描器，才能在配置文件中，对 RS507 进行初始配置。
  • 完成初始配置后，便可以在配置文件中启用和禁用蓝牙扫描器，即使其与设备已断开连接。但是，要配置读取器参数、解码器和其他扫描器设置，则必须连接蓝牙扫描器。
  • DataWedge 正在使用另一个自动选择的扫描器时，如果连接蓝牙扫描器，则 DataWedge 不会自动重新连接到后面连接的蓝牙扫描器。要重新启用蓝牙扫描器，需连接该扫描器并在配置文件中选择它或重新选择“自动选择”选项。
  • 自动选择和电池切换 - 如果将扫描器选择设置为“自动”，并且在电池切换前启用 RS507，那么在切换电池后重新连接时，DataWedge 将继续使用该 RS507 扫描器。如果切换电池后 RS507 未重新连接，则 DataWedge 将恢复到当前默认的扫描器。
  • 挂起时保持启用 - 蓝牙和可插拔的扫描器支持此模式，并可能会使得电池消耗速度比挂起模式原本预期的更快。注意：按下任何受支持扫描器的扫描触发器后，Zebra 计算设备将从挂起模式唤醒。

SimulScan 输入

SimulScan 输入插件允许同时捕获多部分表单上的条码、图像、文本、签名、电话号码和其他数据。SimulScan 输入插件为 DataWedge 增加了此功能。根据指定的 SimulScan 模板获取表单数据时，可使用处理插件根据需要处理数据或设置其格式。

SimulScan 输入插件选项。

SimulScan 捕获备注：

  • 通过 SimulScan 捕获的文本级联成单个字符串，并针对该字符串执行处理。
  • 如果在配置文件中启用了条码输入插件，则在该配置文件中启用 SimulScan 将导致禁用条码输入插件。

设备选择 - 支持在设备摄像头或系统设置的默认扫描设备之间进行选择。

模板选择 - 为正在配置的配置文件设置 SimulScan 模板。

DataWedge 包含的模板：

  • BankCheck.xml - 从支票的机器可读取区域 (MRZ) 中获取账号和路由编号。

  • Barcode1.xml - 解码任何码制的单个条码。

  • Barcode2.xml - 解码相同或不同码制的两个条码。

  • Barcode4.xml - 解码相同或不同码制的四个条码。

  • Barcode5.xml - 解码相同或不同码制的五个条码。

  • Barcode10.xml - 解码相同或不同码制的 10 个条码。

  • BookNumber.xml - 对 10 或 13 位数 ISBN 代码进行解码。

  • DocCap+Optional-Barcode.xml - 将表单作为图像捕获并根据需要对条码（如果存在）进行解码。如果选择“无”，则这是默认表单。

  • DocCap+Required-Barcode.xml - 获取表单，并解码任何可用的条码。

  • TravelDoc.xml - 从旅行证件（如护照）的机器可读取区域 (MRZ) 获取信息。

  • Unstructured Multi-Line.xml - 使用 OCR 获取多行字母/数字文本。

  • Unstructured Single Line.xml - 使用 OCR 获取单行字母/数字文本。

SimulScan 包含的所有模板名称前面均带有“默认”字样以及连字符。

可以使用此选项选择已复制到以下设备目录中的自定义模板 XML 文件：

/enterprise/device/settings/datawedge-cn/templates

注意：默认情况下，/enterprise 目录下的文件和文件夹对 Android 文件浏览器不可见；手动输入路径，即可看到它们。

合作伙伴和其他授权用户可以使用 Zebra 的 SimulScan Template Builder 在线创建自定义模板。

区域分隔符 - 用于为 SimulScan 文本区域数据配置分隔符。当存在多个文本区域时，将在获取表单上每个区域的数据字符串之间插入区域分隔符。区域分隔符可以与按键插件操作键字符设置一起使用（请参阅下文），以将 SimulScan 区域数据发送至单独的文本字段。

可能的值：

  • 无（默认值）
  • 制表符
  • 换行符
  • 回车符

注意：

  • 条码、OCR 和 OMR 区域视为文本区域。使用按键输出和 IP 输出时，仅将文本区域数据发送至前台应用程序或远程服务器。
  • 图片区域数据只能通过 Intent 输出插件来检索。

按键输出

按键输出插件收集已处理的数据，并将其作为一系列按键发送到关联的应用程序，从而仿真用户按键操作。

按键输出插件选项。

操作键字符 - 启用嵌入条码或 MSR 数据的特殊字符注入。

可能的值：

  • 无 - 注入无操作键
  • 制表符 - 在 ASCII Tab (0x09) 字符位置注入操作键
  • 换行符 - 在 ASCII LF (0x0A) 字符位置注入操作键
  • 回车符 - 在 ASCII CR (0x0D) 字符位置注入操作键

多字节字符延迟 - 用于设置发送多字节字符的字符间延迟（以毫秒为单位）。此参数可帮助避免在将 Unicode 和多字节字符发送到 Android 浏览器时出现问题。默认情况下，将值设置为零。如果在发送按键时遇到错误，则以 100 毫秒为增量增加延迟值。

键事件延迟 - 用于设置将控制字符作为按键调度到前台应用程序的延迟（以毫秒为单位）。

基本数据格式

基本格式处理插件提供了一种简便的方法向采集的数据附加或前置自定义值或按键，然后再将其传递至输出插件。它还允许将数据转换为十六进制格式。如果未启用基本格式插件，捕获的数据将原封不动地传递至所选的输出插件。

基本数据格式输出插件选项。

数据前缀 - 在发送之前将指定字符添加（前置）到已获取数据的开头。

数据后缀 - 在发送之前将指定字符添加（附加）到已获取数据的末尾。

发送数据 - 默认情况下处于启用状态，当相关联的应用程序来到前台时，此选项支持将已捕获的数据传输至相关联的应用程序。注意：禁用此选项仅阻止传输已捕获的数据；任何前缀和/或后缀字符串将移交给相关联的应用程序，即使此选项处于禁用状态。

以十六进制格式发送 - 以十六进制格式发送数据。例如，如果已获取的条码数据是 012345，则此选项会发送 303132333435 的十六进制等效项。

发送 TAB 键 - 将 TAB 字符附加到已处理的数据。

发送 ENTER 键 - 将 Enter 字符附加到已处理的数据。

高级数据格式

高级数据格式处理插件允许自定义已获取的数据，以满足任何基于包含单个或多个条件和操作的复杂规则集的要求。有关更多信息，请参阅高级数据格式指南。

Intent 输出

Intent 输出插件允许以编程方式将已获取的数据传递到使用 Android Intent 机制的应用程序。Intent 激活应用程序（其活动、服务和广播接收器）的核心组件。Intent 对象是描述所需操作的信息捆绑包。它包括要处理的数据、应执行操作的组件类别和一些其他的相关指令。启动 Intent 时，Android 会查找适当的组件以响应 Intent，启动组件的新实例（如果需要），以及将 Intent 对象传递给它。

组件宣传其可通过“Intent 过滤器”处理的 Intent 种类，这些过滤器在 AndroidManifest.xml 文件中作为 <intent-filter> 元素进行指定。一个组件可能有任意数量的“Intent 过滤器”，每个过滤器描述不同的功能。

Intent 输出插件选项。

DataWedge 如在其 AndroidManifest.xml 中所述那样，通过 Intent 类别中的 Intent 操作调用 Intent。例如，如果 DataWedge 清单包含行...

<intent-filter>
    ...
    <action android:name="com.myapp.action" />
    <category android:name="android.intent.category.DEFAULT" />
    ...
</intent-filter>

...则 Intent 输出插件中的 Intent 操作将是 com.myapp.action，并且 Intent 类别将是 android.intent.category.DEFAULT。

“Intent 交付”选项允许指定交付 Intent 的方法。通过三种交付机制的其中一种交付基于 Intent 的数据：

  • 通 startActivity 发送

  • 通过 startService 发送

  • 广播 Intent

通过“广播 Intent”发送 Intent 交付时，DataWedge 在广播 Intent 中设置接收器前台标记 Intent.FLAG_RECEIVER_FOREGROUND，使广播收件人有权使用更短超时时间间隔在前台优先运行。仅当将 Intent 交付设置为“广播 Intent”时设置此标记。注意：仅当在设备启动后立即看到交付 Intent 延迟时才使用此标记。

解码相关数据

可以使用以下调用来检索添加到 Intent 捆绑包的解码相关数据：

  • Intent.getStringtExtra()

上述调用可以与以下 String 标签一起使用：

  • String LABEL_TYPE_TAG = "com.symbol.datawedge.label_type"；字符串包含条码标签类型

  • String DATA_STRING_TAG = "com.symbol.datawedge.data_string"；字符串包含输出数据作为字符串。如果是级联的条码，则解码数据将级联在一起并作为单个字符串发出。

  • String DECODE_DATA_TAG = "com.symbol.datawedge.decode_data"；解码数据返回为字节数组列表。在大多数情况下，每个解码都会有一个字节数组。对于支持级联的条码码制（即 Codabar、Code128、MicroPDF 等），已解码数据存储在多字节数组中（每个条码一个字节数组）。客户端可以通过传递索引获取每个字节数组中的数据。

MSR 相关数据

可以使用以下调用来检索添加到 Intent 捆绑包的 MSR 相关数据：

  • Intent.getStringtExtra()

上述调用可以与以下字符串标签一起使用：

  • String MSR_DATA_TAG = "com.symbol.datawedge.msr_data"；MSR 跟踪中的数据将级联在一起，并作为一个字节数组发出。根据配置，将包含“开始/结束”标记和跟踪分隔符。

  • String MSR_TRACK1_TAG = "com.symbol.datawedge.msr_track1"；MSR 跟踪 1 数据将返回为一个字节数组。

  • String MSR_TRACK2_TAG = "com.symbol.datawedge.msr_track2"；MSR 跟踪 2 数据将返回为一个字节数组。

  • String MSR_TRACK3_TAG = "com.symbol.datawedge.msr_track3"；MSR 跟踪 3 数据将返回为一个字节数组。

  • String MSR_TRACK1_STATUS_TAG = "com.symbol.datawedge.msr_track1_status"；MSR 跟踪 1 解码状态为整数，其中 0 表示已成功解码。

  • String MSR_TRACK2_STATUS_TAG = "com.symbol.datawedge.msr_track2_status"；MSR 跟踪 2 解码状态为整数，其中 0 表示已成功解码。

  • String MSR_TRACK3_STATUS_TAG = "com.symbol.datawedge.msr_track3_status"；MSR 跟踪 3 解码状态为整数，其中 0 表示已成功解码。

SimulScan 相关数据

可以使用以下调用检索添加到 Intent 捆绑包中的与 SimulScan 相关的数据：

  • Intent.getStringtExtra()
  • Intent. getParcelableArrayListExtra()
  • Bundle.getInt()
  • Bundle.getString()
  • Bundle.getByteArray()

上面的调用可使用以下 String 标签：

  • String SIMULSCAN_TEMPLATE_NAME_TAG = "com.symbol.datawedge.simulscan_template_name"；SimulScan 用于捕获表单的模板的名称。

  • String SIMULSCAN_REGIONS_BUNDLE_TAG= "com.symbol.datawedge.simulscan_region_data"；返回一个捆绑包数组，其中每个捆绑包都包含有关区域和表单的数据和信息。

  • String SIMULSCAN_REGION_NAME_TAG = "com.symbol.datawedge.simulscan_region_name"；返回到达区域的捆绑包对象的区域名称。要获取区域名称，应调用 Bundle.getString()。

  • String SIMULSCAN_REGION_ID_TAG = "com.symbol.datawedge.simulscan_region_id"；返回到达区域捆绑包对象的区域 ID。区域 ID 为整数，并且可以通过调用 Bundle.getInt () 来检索。

  • String SIMULSCAN_REGION_STRING_DATA= "com.symbol.datawedge.simulscan_region_string_data"；返回区域的字符串数据。字符串数据配有条码、OCR 和 OMR 数据。

  • String SIMULSCAN_REGION_BINARY_DATA= "com.symbol.datawedge.simulscan_region_string_data"；以字节数组的形式返回区域的数据。二进制数据仅用于图片区域和表单图像。图片和表单数据均可加载到位图中，并显示在应用程序中。

  • String SIMULSCAN_REGION_TYPE_TAG = "com.symbol.datawedge.simulscan_region_type"；返回到达区域的捆绑包对象的区域类型。区域类型是字符串，并且可以通过调用 Bundle.getString () 来检索。

区域类型的可能返回值：

  • 条码 - 区域是条码。
  • OCR - 区域是一个光学字符识别 (OCR) 区域（即名称或地址）。
  • OMR - 区域是一个光学标记识别 (OMR) 区域（即复选框、单选按钮）。
  • 图片 - 区域为图片；数据将采用 JPEG 格式。
  • 表单 - 表单类型用于指定该捆绑包所包含已获取表单的图片。表单图像将采用 JPEG 格式。

重要说明：对于某些扫描应用程序，最好将解码后的数据直接发送到当前活动，且不一定显示。对于此类实例，必须在其 AndroidManifest.xml 文件中指定该活动为“singleTop”。不以这种方式指定活动将导致在每次解码时启动一个活动实例，并且将数据发送到每个新生成的副本。

有关 Android Intent 的更多信息，请参考 Android 开发人员网站。

IP 输出

IP 输出插件支持通过网络将已捕获的数据传输到运行 IPWedge（由 Zebra 制作的一个小型 Windows 应用程序）的计算机。PC 接收数据作为按键或将其存放在其剪贴板中，从本质上讲，这是将设备变成 PC 的无线扫描器。

要配置设备以使用 IP 输出插件，则必须了解 PC 的 IP 地址，以及 PC 将要侦听的端口号。要先设置 PC，请参阅 IPWedge 指南，获取 IPWedge 下载和设置说明。然后，从此处继续。

设置 IP 输出插件：

从要从中激活 IP 输出插件的配置文件中：

1.找到配置文件的“IP 输出”部分。

2.选中“已启用”和“远程 Wedge”复选框，启用 IP 输出及与 IPWedge 服务器组件的通信。

IP 输出插件选项。

3.选择数据传输所需的协议（TCP 或 UDP）或接受默认值 (TCP)。

4.输入运行 IPWedge 软件的服务器的 IP 地址。

5.如果端口号不是默认值 58627，则输入端口号。

使用不带 IPWedge 的 IP 输出插件

可以使用 IP 输出插件将已捕获的数据发送到无 IPWedge 的远程设备。在数据接收端，PC 或移动设备应该有一个客户端应用程序，其侦听来自 IP 输出插件中的已配置端口和 IP 地址的 TCP 或 UDP 数据。要配置 IP 输出插件以将已捕获的数据发送到远程计算机或设备，请按照下列步骤操作。

1.找到配置文件的“IP 输出”部分。

2.选中“已启用”复选框，并取消选中“远程 Wedge”复选框。

3.选择数据传输所需的协议（TCP 或 UDP）或接受默认值 (TCP)。

4.输入运行 IPWedge 软件的服务器的 IP 地址。

5.如果端口号不是默认值 58627，则输入端口号。

警告：Zebra 不支持此使用场景。

Data Capture Plus (DCP)

Data Capture Plus（以前称为“Data Capture Panel”）支持将设备屏幕的某些区域指定为扫描触发器。轻触指定的屏幕区域，DataWedge 会做出响应，就像它对扫描器按钮或其他硬件触发器做出的响应一样。

DCP 默认处于禁用状态。“DataWedge 配置文件”配置屏幕允许应用程序用户对加载某个特定配置文件后是否在屏幕上显示 DCP 进行配置。如果用户选中此选项以启用 DCP，下面显示的五个参数会出现在首选项屏幕上，并且您可根据需要进行配置。

注意：如果在当前配置文件中禁用了扫描器，则不会显示 DCP。

用于设置扫描触发器的 Data Capture Plus 选项。

Data Capture Plus 提供了下列可配置参数：

按钮停靠位置 - 设置浮动 DCP 按钮的初始停靠位置。用户在运行时进行的更改将保存到活动配置文件中。停靠选项：

  • 仅右侧
  • 仅左侧
  • 任一侧

启动模式 - 设置 DCP 的启动模式。如果配置为作为按钮启动，则可以在运行时通过拖动来更改 DCP 模式，但将无法在配置文件中更改启动状态。启动模式选项：

  • 按钮模式（浮动按钮）
  • 全屏模式
  • 仅按钮模式

按钮最高位置 - 设置按钮位置上限，以占总屏幕高度的百分比表示。例如，在垂直测量高度为四英寸的屏幕上，如果将此值设置为 75 (%)，则 DCP 按钮的上边缘无法来到距离屏幕顶部不到一英寸的位置。

底部最低位置 - 设置按钮位置下限，以占总屏幕高度的百分比表示。例如，在垂直测量高度为四英寸的屏幕上，如果将此值设置为 25 (%)，则 DCP 按钮的下边缘无法来到距离屏幕底部不到一英寸的位置。

拖动检测时间 - 在触发扫描器操作之前，DCP 应在屏幕轻触后应等待的等待时间（以毫秒计）。将 DCP 按钮拖动到新位置时，这可帮助阻止意外触发事件发生。

注意：使用摄像头作为扫描器时，快速触摸并释放 DCP 有时会启动取景器。要退出，请按“后退”按钮。

使用 DCP 进行扫描

要使用 DCP 扫描条码：

1.在 DCP 已启用的情况下，轻触并按住屏幕上为 DCP 指定的区域。当按住时，扫描光束（或摄像头取景器）将处于活动状态。

2.将扫描光束或摄像头十字线对准要扫描的条码。DCP 将使用在当前配置文件的条码输入插件中配置的首选项。

3.* 松开手指以停止扫描或关闭摄像头取景器。

注意：使用摄像头作为扫描器时，快速触摸并释放 DCP 控件有时会启动取景器。要退出，请按“后退”按钮。
