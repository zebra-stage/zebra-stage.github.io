DataWedge API 示例

DataWedge 7.0

概述

DataWedge API 示例应用程序演示直到版本 6.8（写作本文时的最新版本）的所有主要 DataWedge API 的功能。要利用此示例应用程序，需要对 DataWedge 有基本的了解。有关 DataWedge API 的更多信息以及有关此示例应用程序的深入了解，请参阅 Zebra 工程师 Darryn Campbell 编著的 DataWedge API - 优势和使用场景一文（此 DataWedge API 示例源自此文）。

注意：此应用程序仅以演示为目的。它按原样提供，可根据个人需求进行修改。

新增功能

示例应用程序中的新功能包括：

  • 新的 SET_REPORTING_OPTIONS API 提供使用 Intent 的报告功能的控制（在 DW v6.6 中引入）。
  • SWITCH_SCANNER API 现在支持使用新额外项的友好设备名称（在 DW v6.6 中引入）。
  • 新的 IMPORT_CONFIG API 允许使用 Intent（在 DW v6.7 中引入）导入配置和配置文件设置文件。
  • 新的 SET_IGNORE_DISABLED_PROFILES API 可配置 DataWedge 以避免切换到未启用的配置文件（在 DW v6.8 中引入）。
  • 新的 GET_IGNORE_DISABLED_PROFILES API 可返回 IGNORE_DISABLED_PROFILES 标记的状态（在 DW v6.8 中引入）。
  • 新的 SWITCH_SIMULSCAN_PARAMS API 可实现对 SimulScan 参数的运行时更改（在 DW v6.8 中引入）。

使用

安装 APK 之后，您会注意到按 DataWedge API 版本分类的 API 功能。要快速启动并运行，请启动 DataWedge API 示例应用程序。在 DataWedge 6.3 API 部分中的“创建配置文件”选项下，轻触发送。这会在 DataWedge 中创建“DW API 练习程序配置文件”，并将此应用程序关联到该配置文件。它可以针对 Intent 广播进行注册，以从 DataWedge 接收扫描的数据。

轻触发送按钮以对任何命令执行操作。发送命令之后，一个弹出消息会出现，指示发送的命令的状态。扫描条码之后，会在顶部填充数据以及扫描源和解码器信息。
