Licensing

Identity Guardian 2.1

Overview

Licensing requirements for Identity Guardian depend on the device type and specific functional needs. This section outlines the types of licenses available and when they are needed.

  • Zebra Enterprise-series devices: No license is required for basic Identity Guardian functions, including single sign-on (SSO) integration with Zebra apps, barcode scanning authentication, and combined barcode scanning and PIN entry authentication. An Identity Guardian license is required for advanced features such as:
    • Facial biometric authentication
    • SSO integration with third-party apps
    • Guardian Safe
    • Device API support
  • Zebra Professional-series devices: Most features require Identity Guardian licenses, depending on the presence of other Zebra product licenses on the device. Consider these scenarios:
    • With an Identity Guardian license but without a Mobility DNA (MDNA) license:
      • SSO integration with Zebra apps and facial biometric authentication are operational.
    • With a Device Tracker/Device Guardian license but without an Identity Guardian or MDNA license:
      • SSO integration with Zebra apps is functional.
      • If facial biometrics is enabled, Identity Guardian does not perform authentication or block device access.
    • Without Device Tracker/Device Guardian and Identity Guardian licenses:
      • If SSO is enabled, Identity Guardian does not perform authentication or block device access.

When using Identity Guardian in conjunction with Device Guardian, a Device Guardian license is required. Identity Guardian supports both Device Tracker and Device Guardian for Android 11 and Android 13 devices.

With Device Tracker rebranded to Device Guardian, starting with Android 14 and later, Identity Guardian exclusively supports Device Guardian. Therefore, a Device Tracker license is necessary for Android 13 and below, while a Device Guardian license is required for Android 14 and above.

This section discusses the acquisition and application of Identity Guardian licenses obtained from Zebra or through a Zebra reseller.

Available License Types:

  • Trial License (30-day evaluation period)
  • Term-based License:
    • 1-year term
    • 3-year term
    • 5-year term

Evaluation License

Zebra offers time-limited trial licenses to allow companies to evaluate the performance of Zebra solutions in their environment.

  • Customers, partners and distributors: Request through a Zebra account manager or sales engineer
  • Zebra account managers and sales engineers: Submit requests using SFDC forms
  • Zebra engineers and other Zebra employees and contractors: Submit a request in the Zebra ServiceNOW portal

I. Contact a Reseller

Zebra licenses are obtained through Zebra resellers or (in some cases) directly from Zebra.

To begin, select an option:

  • Find a Zebra Partner - form for submitting an inquiry via the web
  • How to Select a Channel Partner - explains the types of partners that engage with Zebra and some of their technologies and specialties
  • Partner Interaction Center - info for contacting Zebra's existing global partner network
  • Zebra Corporate Numbers and Links - broken down by global region
  • Global Marketing Contact Center - broken down by global region and country

II. Access Licensing System

After a licensing agreement is purchased, an email is sent to the license requester containing user credentials for accessing the Zebra Enterprise Software Licensing system.

Visit the Zebra licensing support page, register and log into the portal to:

  • Order licenses
  • Check status of existing orders
  • Assign licenses to devices or deployments
  • View inventory and assignments of current licenses

III. Locate URL

After procuring Identity Guardian licenses, a confirmation email is sent with the Cloud Licensing Service (CLS) URL. This URL is required to activate the license in the next step.


IV. Activate License

Licenses can be activated either manually or via mass deployment. Use the following Feature Name for activation: idguardian-feature-std

Manual

To activate the license on a device:

  1. In the Apps screen, launch Licensing Manager.
  2. Tap the + icon at the bottom right to add a license.
  3. In the "Activate License" screen, enter/select the following:
    • Select Activation Type: Feature activation
    • Enter Feature Name: idguardian-feature-std
    • Enter Partition Name: [Leave blank, no entry required]
    • Select License Source: [Opens "Select License Source" screen, see next step]
  4. In the "Select License Source" screen, enter/select the following and tap Submit:
    • Select License Source: Cloud/Local Server
    • Enter URL: [Copy/paste the CLS URL supplied by Zebra in the confirmation email]
    • Name: [Enter a name for the license source]
  5. Tap Activate.
  6. The license is displayed in the main screen.


Mass Deployment

There are 2 methods for deploying Identity Guardian licenses to devices:

  • OEMConfig via EMM or ZDNA Cloud
  • StageNow barcode scan or XML deployment via EMM or ZDNA Cloud

OEMConfig

OEMConfig is an administratrive tool used by compatible enterprise mobility management (EMM) platforms or Zebra DNA Cloud to configure Zebra Android devices via Managed Configurations. To deploy Identity Guardian licenses, set up License Configuration as follows:

  • Name: idguardian-feature-std
  • Server URL: [Enter the Cloud Licensing Server (CLS) URL provided by Zebra in the onboarding email.]
  • Server Friendly Name: [Enter a friendly name to associate with this server.]
  • Partition Name: [Leave blank; not required.]

StageNow

Zebra's StageNow tool can create a staging profile that is consumed by StageNow on the device or an Enterprise Mobility Management (EMM) system to deploy Identity Guardian licenses.

To create a Stagenow profile:

  1. Launch StageNow.

  2. Select Create new Profile from the left-hand pane.

  3. Select MX version 10.2 or higher - it must match the MX version on the device. Click Xpert Mode and click Create.

  4. Enter a profile name and click Start.

  5. Scroll down and click LicenseMgr. LicenseMgr is added to the right side. Click Add.

  6. Under LicenseMgr, select or enter the following:

    • License action type: Perform Zebra license action
    • Zebra license action: Activate Feature
    • Feature Name: idguardian-feature-std
    • License Source URL: [Copy/paste the CLS URL supplied by Zebra in the confirmation email]
    • License Source Friendly Name: [Enter a name identifier]

  7. Click Complete Profile. The StageNow profile creation is complete.

Choose one of the following deployment methods:

  • StageNow: Generate the barcode from the StageNow profile. Open StageNow client on the device and scan the barcode(s) generated.
  • EMM: Export XML from the StageNow profile. Do not edit the XML file to prevent unexpected behavior. Push the XML profile to the device via EMM (refer to your EMM guide) or Zebra DNA Cloud.
