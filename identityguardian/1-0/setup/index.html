Setup

Identity Guardian 1.0

Overview

Administrators set up Identity Guardian usage policies and controls through Managed Configurations using either Zebra DNA Cloud or Enterprise Mobility Management (EMM). This enables organizations to customize device restrictions and access. It also provides administrators dashboard visibility into usage details like user sign-in/sign-out events and usage times. This oversight helps enforce user accountability.

Identity Guardian simplifies application authentication with identity provider (IdP) integration. Users only need to sign in once using SSO. After that, SSO manages the rest of the application sign-ins, streamlining the process.

Requirements

Identity Guardian requirements:

  • Zebra devices running Android 11 or higher are supported.
  • For biometric authentication, a front-facing camera is required
  • Licenses:
    • An Identity Guardian license is required for advanced features including facial biometric authentication, Single Sign-On (SSO) support and device API support. See Mobility DNA licensing for information on how to obtain and apply the license.
    • Zebra Professional-series devices require a Mobility DNA Enterprise license for basic functionality

Installation

Download and install Identity Guardian from the Zebra support portal. Link will be added upon product release.

SSO

Identity Guardian integrates with Identity Providers (IdPs) simplifying authentication by only requiring users to log in once, and then leveraging Single Sign-On (SSO) to streamline the process. This single login grants users to gain access to multiple applications, eliminating the need to keep track of multiple logins for each app.

Integrations with both Microsoft Entra ID (formerly Azure Active Directory) and PingID platforms are available, with support provided for the OAUTH and OAUTH + OIDC authentication protocols.

Instructions to be provided upon product release.

ZDNA Cloud

Prior to device setup, configure Identity Guardian via Managed Configurations through Zebra DNA Cloud. This determines the method of device use (shared or personally assigned), user enrollment, and authentication method (facial biometric, passcode, and/or PIN).

See ZDNA Cloud documentation for a guide on how to configure Identity Guardian.
