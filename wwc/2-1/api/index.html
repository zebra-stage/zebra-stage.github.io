Wireless Developer Service APIs

Wireless Workstation Connect 2.1

Overview

Wireless Developer Service APIs allow custom apps running on Zebra Android devices to display content over the air on a monitor connected to a wireless recieving device such as the Zebra ZEC500.

initiate a connection with a wireless receiver such as the Zebra ZEC500 and share screen and input resources. When developing with the APIs, Zebra recommends observing the best practices and known behaviors described below to help ensure proper connection and operation.

Best Practices

  • Before calling the Connect API:
    • Call START_WIRELESS_DISPLAY_SCAN to determine the receiving device (e.g. ZEC500) to connect with.
    • Check the display properties of the intended receiver to confirm that CanConnect and IsAvailable properties are both true.

and you can call “Stop Scan” after establishing the connection, to minimize the resource usage.

While exit from the app, make sure to call the “DEINIT_DEV_SERVICE” API to clear the ZWDS. Otherwise, it will not be possible to other app to call “INIT_DEV_SERVICE” successfully.

Known Behavior

If it is kept device in idle longer time (more than 1 hr) ZWDS Api calls are not working as expected. It is needed disable and reenable the wi-fi for overcome the issue.

Calling of “DEINIT_DEV_SERVICE” API is not disconnecting existing connection session. If it is needed to disconnect the session, please call disconnect API before calling the “DEINIT_DEV_SERVICE” API.

If ZWDS runs in non-secure mode it will not validate any token send by test app and not showing any error messages related to secure tokens.

Once connection is established with ZEC500 device, if it is again call the API Connect. It will disconnect the existing connection.

APIs

INIT_DEV_SERVICE

Initializes the developer service for wireless communications with a ZEC500-type receiver. This API must be called before any other API calls are made.

Request: Intent Action

  • com.zebra.wirelessdeveloperservice.action.INIT_DEV_SERVICE

Supports the following key(s):

Key Type Description
CALLBACK_RESPONSE PendingIntent Object that receives the response
STATE_CHANGE_RCV_PKG String Package name to receive broadcast for state change

Response

Sends the following response(s) to the app:

Key Type Description
RESULT_CODE Integer 1: Success
0: Failure
RESULT_MESSAGE String Success: Success
Failure: Error Message

Set Proximity connection/disconnection broadcast on/off Intent SET_PROXIMITY_CONNECT will on/off the proximity connection threshold crossing broadcast with Maverick device. Once it is on ZWDS broadcast the Intent to app once Host device cross the connect/disconnect threshold. Request Intent Action com.zebra.wirelessdeveloperservice.action.SET_PROXIMITY_CONNECTION Add the following key to the Intent Key Type Description CALLBACK_RESPONSE PendingIntent PendingIntent object that receives the response PROXIMITY_CONNECT String ON – On the proximity connection broadcast OFF – Off the proximity connection broadcast PROXIMITY_DISCONNECT String ON- On the proximity disconnection broadcast OFF – Off the proximity disconnection broadcast CONNECT_THRESHOLD Integer Distance in feet (Max value is 30 feet) DISCONNECT_THRESHOLD Integer Distance in feet (Max value is 30 feet) Zebra Wireless Developer Service Design v 1.0.0 Response: The response is sent back to the app Key Type Description RESULT_CODE Integer 1: Success 0: Failure RESULT_MESSAGE String Success: Success Failure: Error Message

State Change Broadcasters ZWDS sends three types of broadcasts to customer app to update the current status. To receive ZWDS broadcast customer app should define the below ZWDS permission in its Manifest

<uses-permission android:name="com.zebra.permission.ZWDS_BROADCAST" />
