ImageAttributesDetector

AI Data Capture SDK 3.1

Overview

ImageAttributesDetector is an AI-driven component, specifically designed for Proof of Delivery (POD) workflows, that analyzes and images to detect and verify the authenticity and quality of images in real-time,. It provides an asynchronous, easy-to-use API to analyze critical image attributes - such as quality, object presence, and scene characteristics - directly within enterprise logistics and delivery applications.

  • ‚ö° Real-Time Processing - Uses asynchronous API for fast, non-blocking image analysis through a CompletableFuture.

  • üîß Flexible Configuration - Customize detection thresholds, metrics, and inference options to fit the business requirements for your workflow.

  • üìä Detailed Results - Provides comprehensive results including compliance status, confidence values, and metric details.

Performance Note: Optimized for mobile devices with GPU acceleration support through SNPE runtime.

Model Loading Options

There are two ways to load a model:

  • Default Model - The SDK automatically loads its built-in model, requiring no additional configuration. The default model is used when a Settings object is created without loading a custom model:

    // Uses default model
new Settings()

  • Custom Model - A custom model can be loaded from a file path for specialized use cases or when using an updated model. Supported file formats are: .tar.crypt, .tar, and .aar. The file must be readable and exist on the device. Usage:

    settings.loadModelFromFilePath(ModelType.TypeImageTagDetect, modelFile)

Developer Guide

This guide outlines the process for using ImageAttributesDetector, from initializing the SDK to processing image analysis results.

Step 1: Initialize SDK

This is a one-time setup step that activates the AI Data Capture SDK for the application and authenticates it with a license key. This operation should be performed once on a background thread, typically during application startup.

Sample Code:

    // Initialize SDK with context and license
    AIVisionSDK.initAIVisionSDK(context, licenseKey);

Step 2: Create and Configure Settings

Create a Settings object to define what the detector should look for. For each attribute to be analyzed (e.g., image clarity, package presence), an ImageAttributeMetricValue object is built. This object is used to enable the metric, set a custom threshold, and define the pass/fail condition before being added to the main settings configuration.

Best Practices:

  • At least one metric must be enabled.
  • Set custom values only when necessary; start with the defaults.
  • Use the toString() method on the Settings object for debugging purposes; see Debugging & Troubleshooting.

Sample Code:

    // Create settings instance
    ImageAttributesDetector.Settings settings =
    new ImageAttributesDetector.Settings();

    // Build metric configurations
    ImageAttributeMetricValue qualityMetric =
        new ImageAttributeMetricValue.Builder(
            ImageAttributeMetric.ImageQualityClear)
        .setValue(0.85f)              // Optional: custom threshold
        .setEnable(true)              // Enable this metric
        .setCondition(ConditionType.GT_THAN)  // Greater than check
        .build();

    ImageAttributeMetricValue packageMetric =
        new ImageAttributeMetricValue.Builder(
            ImageAttributeMetric.ImageTagPackageVisible)
        .setEnable(true)              // Uses default threshold 0.20
        .build();

    // Add metrics to settings
    List<ImageAttributeMetricValue> metrics = Arrays.asList(
        qualityMetric, packageMetric);
    settings.configureImageAttributeMetrics(metrics);

Step 3: Create Detector

With the settings configured, create an instance of the ImageAttributesDetector. This is an asynchronous operation that should be handled on a background thread (using an ExecutorService). The method returns a CompletableFuture for handling the successful creation in thenAccept or any initialization errors (like an invalid license or model issue) in exceptionally.

Important: At least one metric must be enabled in settings, otherwise an InvalidInputException is thrown.

Sample Code:

    // Create executor for async operations
    ExecutorService executor = Executors.newSingleThreadExecutor();

    // Get detector instance asynchronously
    try {
        CompletableFuture<ImageAttributesDetector> detectorFuture =
            ImageAttributesDetector.getImageAttributesDetector(
                settings,
                executor);

        detectorFuture.thenAccept(detector -> {
            Log.i("TAG", "Detector initialized successfully");
            // Detector is ready to use
            processImages(detector);

        }).exceptionally(ex -> {
            Log.e("TAG", "Initialization failed: " + ex.getMessage());
            return null;
        });

    } catch (AIVisionSDKException e) {
        // Handle SDK not initialized
        Log.e("TAG", "SDK Error: " + e.getMessage());
    } catch (InvalidInputException e) {
        // Handle invalid settings (null or no metrics enabled)
        Log.e("TAG", "Invalid Settings: " + e.getMessage());
    }

Step 4: Process Image Data

The detector.process() method analyzes an image, taking an ImageData object as input. This operation is asynchronous, handling threading internally and returning a CompletableFuture. The results are handled by attaching callbacks: thenAccept for a successful analysis, which provides a list of ImageAttributeResult objects for processing, and exceptionally to catch any processing errors.

Recommendations for processing:

  • Process images sequentially one at a time.
  • Handle asynchronous exceptions to manage processing errors.
  • Check results for compliance.
  • Log the confidence values to help with analysis and troubleshooting.

Sample Code:

    private void processImages(ImageAttributesDetector detector) {
        // Create ImageData from Bitmap
        ImageData imageData = new ImageData.fromBitmap(bitmap,0);

        // Process image
        CompletableFuture<List<ImageAttributeResult>> resultsFuture =
            detector.process(imageData);

        resultsFuture.thenAccept(results -> {
            for (ImageAttributeResult result : results) {
                String metricName = result.getMetric().toString();
                boolean isCompliant = result.isCompliant();
                Object value = result.getValue();

                Log.d("TAG", String.format(
                    "Metric: %s | Compliant: %s | Value: %s",
                    metricName, isCompliant, value));
            }
        }).exceptionally(ex -> {
            Log.e("TAG", "Processing failed: " + ex.getMessage());
            return null;
        });
    }

Step 5: Handle Results

Analyze results:

    for (ImageAttributeResult result : results) {
        switch (result.getMetric()) {
            case ImageQualityClear:
                if (result.isCompliant()) {
                    Log.i("TAG", "Image quality is acceptable");
                } else {
                    Log.w("TAG", "Image quality check failed");
                }
                break;
            case ImageTagPackageVisible:
                Float confidence = (Float) result.getValue();
                Log.d("TAG", "Package detection: " + confidence);
                break;
        }
    }

Step 6: Resource Management

To prevent memory leaks and ensure application stability, it is important to release all resources when the detector is no longer needed. Call dispose() within the appropriate life cycle method, such as onDestroy(). Also, the executor should be properly shut down. After disposal, a new instance must be created for further use; don't reuse disposed instances.

Sample Code:

    public class ImageAnalysisActivity extends AppCompatActivity {
        private ImageAttributesDetector detector;
        private ExecutorService executor;

        @Override
        protected void onCreate(Bundle savedInstanceState) {
            super.onCreate(savedInstanceState);
            setContentView(R.layout.activity_main);

            // Initialize resources
            executor = Executors.newSingleThreadExecutor();
            initializeDetector();
        }

        @Override
        protected void onPause() {
            super.onPause();
            // Optional: dispose if needed during pause
        }

        @Override
        protected void onDestroy() {
            super.onDestroy();

            // Always cleanup resources
            if (detector != null) {
                detector.dispose();
                detector = null;
            }

            if (executor != null) {
                executor.shutdown();
                try {
                    executor.awaitTermination(5, TimeUnit.SECONDS);
                } catch (InterruptedException e) {
                    executor.shutdownNow();
                }
            }
        }
    }

Complete Sample

The following source code shows a complete example of ....

    public class ImageAnalysisManager {
        private ImageAttributesDetector detector;
        private ExecutorService executor;

        public void initialize(Context context, String license) {
            // Initialize executor
            executor = Executors.newSingleThreadExecutor();

            // Initialize SDK
            AIVisionSDK.initAIVisionSDK(context, license);

            // Configure settings
            ImageAttributesDetector.Settings settings =
                new ImageAttributesDetector.Settings();

            List<ImageAttributeMetricValue> metrics = new ArrayList<>();
            metrics.add(new ImageAttributeMetricValue.Builder(
                ImageAttributeMetric.ImageQualityClear).setEnable(true).build());
            metrics.add(new ImageAttributeMetricValue.Builder(
                ImageAttributeMetric.ImageTagPackageVisible).setEnable(true).build());

            settings.configureImageAttributeMetrics(metrics);

            // Get detector
            try {
                ImageAttributesDetector.getImageAttributesDetector(settings, executor)
                    .thenAccept(det -> detector = det)
                    .exceptionally(ex -> {
                        Log.e("TAG", "Init failed", ex);
                        return null;
                    });
            } catch (Exception e) {
                Log.e("TAG", "Error", e);
            }
        }

        public void analyzeImage(Bitmap bitmap) {
            if (detector == null) return;

            detector.process(new ImageData(bitmap))
                .thenAccept(results -> processResults(results));
        }

        public void cleanup() {
            if (detector != null) detector.dispose();
            if (executor != null) executor.shutdown();
        }
    }
