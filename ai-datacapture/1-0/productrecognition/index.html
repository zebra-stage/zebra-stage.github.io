Product Recognition

AI Data Capture SDK 1.0

Overview

Product Recognition utilizes the Feature Extractor, Feature Storage, and Recognizer classes to identify products in images. Typically paired with the Localizer class, this helps build a comprehensive knowledge base that enhances inventory tracking and facilitates automated checkouts.

  • Feature Extractor - Optimized for retail product recognition, this class extracts key features from images to produce ‘embeddings’ or ‘descriptors.’
  • Feature Storage - These embeddings are stored in a vector database, allowing users to build a database of products for recognition.
  • Recognizer - This initialize the database to perform semantic searches, identifying the most similar embeddings within the knowledge base and predicting the identity of detected products.

Based on requirements, the Retail Shelf Object Localizer can first detect products within an image. Each detected product is then ‘cropped’ and individually submitted for recognition, streamlining the process for efficient and accurate identification.

Feature Extractor

The FeatureExtractor class is designed to generate a set of descriptors that are representative of items located within an image. This class provides functionalities to process both entire images and specific regions defined by bounding boxes, supporting various input formats and orientations.

Feature Extractor generates descriptors (embedding), which describes a feature and is a vector of floats, on a given image.

  • Input: Model and Image
  • Output: Descriptor

Initialization

The Feature Extractor object is initialized via its configuration structure. There are 2 mandatory fields that need to be set:

  • MODEL_ARCHIVE_FILEPATH = fully qualified file path to the .tar or .crypt archive from which the feature extractor model is read

  • MODEL_FILENAME = file name of a localizer model (no extension)

    import com.zebra.AI.Vision.FeatureExtractor;

FeatureExtractor.Settings settings = new FeatureExtractor.Settings(MODEL_ARCHIVE_FILEPATH, MODEL_FILENAME);

// Initialize FeatureExtractor object
FeatureExtractor featureExtractor = new FeatureExtractor(settings);

Usage

Generates a descriptor (embedding) on a given image.

IMAGE = Bitmap image

    float[] descriptors = featureExtractor.generateSingleDescriptor(IMAGE)

IMAGE = Bitmap image

    float[][] descriptors = featureExtractor.generateDescriptors(DETECTIONS, IMAGE)

generateDescriptors (BBox[ ] detections, Bitmap bmp, int orientation)

CompletableFuture FeatureExtractor.generateDescriptors (BBox[ ] detections, Bitmap bmp, int orientation) throws InvalidInputException

Description

Generate a set of descriptors for each of the items in an image determined by an array of bounding boxes.

Parameters

  • detections - array of bounding boxes determining the location of items in the image.
  • bmp - image where the items are present.
  • orientation - orientation angle which the image was taken from - the detection routine rotates the image by this angle (clockwise) before processing, then rotates the result (anticlockwise) to correct.

Exceptions

  • InvalidInputException

Throws InvalidInputException exception when Bitmap is passed as null or detections array is passed as null or orientation is not equal to anyone of the following:

  • 0 degree
  • 90 degree
  • 180 degree
  • 270 degree

Return Value

Array with a descriptor for each of the items defined by the bounding boxes.

Feature Storage

The FeatureStorage class is primarily used for updating the items the Recognizer can detect. It receives descriptors from the FeatureExtractor class and stores them in a database, adhering to a specific update strategy. The database can then be utilized to initialize a Recognizer with new detection capabilities.

Input: Descriptor & Class Label

Output: Database

Feature Storage (Learning block) has the main application of updating the set of items the Recognizer is able to detect. Descriptors generated by the FeatureExtractor can be fed into this class, which will store them in a DB, alongside the class name they represent, following a predefined update strategy. The resulting DB can then be used to initialize a Recognizer instance with new detection capabilities.

Initialization

The Feature Storage object is initialized via its configuration structure. There are 2 mandatory fields that need to be set. 

    import com.zebra.AI.Vision.FeatureStorage;

    FeatureStorage.Settings settings = new FeatureStorage.Settings();
    settings.resourceName = MODEL_ARCHIVE_FILEPATH;
    settings.dbSource = DATABASE;

    // Initialize FeatureStorage object
    FeatureStorage featureStorage = new FeatureStorage(settings);

Usage

    String[] classNames = featureStorage.getAvailableClassNames();

Add descriptors to the database:

DESCRIPTORS = list of descriptors (typically a result of FeatureExtractor generate descriptors call)

LABEL = identifier for the className that is being stored, for example it could be the SKU of a retail product

    featureStorage.addDescriptors(LABEL, DESCRIPTORS);

Recognizer

The Recognizer class provides methods for recognizing patterns and making recommendations based on input descriptors. It supports operations for finding top recognitions, generating recommendations, and managing index states.

The Recognizer class is designed to perform recognition tasks by finding the top-k recognitions that match a set of descriptors. It supports both synchronous and asynchronous operations for retrieval of recognition results and can handle loading and saving of indices and labels for persistent storage.
