Localizer

AI Data Capture SDK 1.0

Overview

The Localizer class provides functionality for object detection within images, commonly referred to as localization. It processes input images and outputs a list of bounding boxes that identify the locations of detected objects or barcodes, making it particularly useful for applications requiring the identification and localization of specific objects within an image. Each bounding box includes:

  • Coordinates of the bounding rectangle - Defines the position and size of the detected object.
  • Probability (Confidence) - Indicates the certainty of the detection.
  • Class - Specifies the type of object detected.

Types of detection supported:

  • Barcode Localizer - Detects 1D and 2D barcodes in images. After detection, the Barcode Decoder can be used to decode the barcodes. Users can specify the input size during initialization to accommodate different use cases: larger sizes improve detection of smaller barcodes but increase inference time, while smaller sizes enhance speed for images with larger barcodes. Supported barcode classes include LinearCode, DataMatrix, QRCode, Aztec, PDF417, DotCode, PostalCode, MaxiCode, CompositeCode, HanXinCode, 2D Code, MicroPDF417, MicroQRCode, GridMatrix, and ArUco.
  • Retail Shelf Object Localizer - Detects objects on grocery store shelves, including products, shelf labels, peg labels, and the shelves themselves, facilitating detailed analysis. It aids in detecting and categorizing items to improve inventory management and optimize product placement. The input size can be specified during initialization to optimize detection based on image distance; smaller input sizes are ideal for close-up, partial shelf images, while larger sizes generally yield better results for full shelf images, although with longer inference times.

The Localizer class can be configured using the Settings class, which allows for model parameters to be specified and, if necessary, overridden.

  • Input: Model and Image
  • Output: Bounding Boxes

Initialization

The Localizer object is initialized via its configuration structure. There are 2 mandatory fields that need to be set:

  • MODEL_ARCHIVE_FILEPATH = fully qualified file path to the .crypt archive from which the localizer model is read
  • MODEL_FILENAME = file name of a localizer model (no extension)

Optional: Users can override the default model input size (height and width) in inferencerOptions, as shown in the example below. Adjusting the input size impacts both accuracy and inference time. Smaller input sizes execute faster but may reduce accuracy, while larger input sizes may improve accuracy, especially for detecting smaller or distant objects. However, excessively large input sizes may lead to out-of-memory errors, potentially causing the application to crash at run-time. For ONNX models, specifying the height and width is mandatory, with the input sizes being in multiples of powers of two (e.g. 640).

    import com.zebra.AI.Vision.Localizer;

    Localizer.Settings settings = new Localizer.Settings(MODEL_ARCHIVE_FILEPATH, MODEL_FILENAME);
    # Optional model input size parameters
    settings.inferencerOptions.defaultDims.height = 1600
    settings.inferencerOptions.defaultDims.width  = 1600

    // Initialize Localizer object
    Localizer localizer = new Localizer(settings);

If required, advanced options are available; see Settings Class for more information.

Usage

Detect objects on a given image and output a collection of bounding boxes for each detected object.

IMAGE = Bitmap image

    CompletableFuture <BBox[]> future = localizer.detect(IMAGE); // Return type is com.zebra.AI.Vision.BBox

APIs

Localizer Constructor

Constructs a new Localizer object by applying the specified settings and loading the detection model.

  • Constructor:
    • Localizer.Localizer (Settings settings)
  • Parameters:
    • settings (Settings) - Configuration settings for initializing the Localizer.
  • Return Value:
    • Localizer object
  • Exceptions:
    • FileNotFoundException - Thrown if the model file is not found in the archive.
    • IOException - Thrown if the model archive is corrupted.


detect (Bitmap bmp, int orientation)

Runs the model on an input image, optionally including a depth map, to calculate bounding boxes. The total number of supplied channels (primary image + depth) must match the model's input channel requirements.

  • Method:
    • CompletableFuture<BBox> com.zebra.AI.Vision.Localizer.detect (Bitmap bmp, int orientation)
  • Parameters:
    • bmp (Bitmap) - The input image for detection.
    • orientation (int) - The orientation angle of the input image. The routine rotates the image clockwise by this angle for processing, then applies an anticlockwise rotation to the results for correction.
  • Return Value:
    • CompletableFuture with BBox[ ] as output, where each BBox contains detectedObjects, locations, probability and class of objects.
  • Exceptions:
    • InvalidInputException - Thrown if Bitmap is null or orientation is not 0, 90, 180 or 270 degrees.


detect (Bitmap bmp)

Executes the model on an input image, optionally including a depth map, to calculate bounding boxes. The total number of input channels (primary image + depth) must match the model's required input channels.

  • Constructor:
    • CompletableFuture<BBox> Localizer.detect (Bitmap bmp)
  • Parameters:
    • bmp (Bitmap) - The input image on which detection is performed.
  • Return Value:
    • CompletableFuture with BBox[ ] as output. BBox contains detectedObjects, locations, probability and class of objects.
  • Exceptions:
    • InvalidInputException thrown when Bitmap is passed as null.


getModelVersion()

Get version string associated with model.

  • Method:
    • String Localizer.getModelVersion()
  • Return Value:
    • A string containing the model version.


getDetectedClasses()

Retrieves the list of names representative of the classes the model can detect.

  • Method:
    • String[ ] Localizer.getDetectedClasses()
  • Return Value:
    • An array of strings with names of classes.


Dispose()

Releases all internal resources used by the Localizer object. This method should be called manually to free resources.

  • Method:
    • void Localizer.dispose()
